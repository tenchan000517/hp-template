# 事例ページ 詳細設計

**URL**: /case（一覧）、/case/[slug]（詳細）

---

## このページの目的

### 専門家①（マーケティング）の視点

**マーケティング目標:**
- 「本当にできるのか」という疑念を払拭する最終証拠
- 課題→解決→成果の流れで、自社への適用可能性を示す
- 量ではなく質で勝負（代表的な3-5件を深掘り）
- 問い合わせへの最後のひと押し

**コンバージョン設計:**
- 事例を読んだ後は、問い合わせが主要な選択肢
- 「この事例と似た課題がある」と思わせる
- クライアント名は実名が理想（許可を得て）

### 専門家②（心理学）の視点

**心理的ゴール:**
- **社会的証明**: 「他の会社も依頼している」という安心感
- **類似性の認知**: 「自分の会社と似ている」という自己投影
- **成功のビジョン**: 「依頼すればこうなれる」という期待

**活用する心理効果:**
1. ストーリーへの没入: 課題から成果への物語に引き込む
2. 具体性の法則: 抽象的な実績より、具体的な事例が記憶に残る
3. 成功体験の疑似体験: 他者の成功を通じて自分の成功をイメージ

### 専門家③（デザイン）の視点

**デザイン上の役割:**
- このテンプレートで最も「実績」を見せるページ
- 写真を効果的に使用するが、ギャラリー的にはしない
- 各事例が「作品」として成立する見せ方

**視覚的ゴール:**
- 事例写真が主役、テキストは補助
- 余白の中に事例が浮かぶ構成
- 一覧では抑制、詳細で展開

---

## ユーザーの状態

**このページに来る前の状態:**
- サービス内容を理解している
- 「本当にできるのか」を確認したい
- 自社との類似性を探している

**このページで感じてほしいこと:**
- 「確かな実績がある」
- 「自分の会社と似た課題を解決している」
- 「この会社に依頼したい」

**このページから離れる時の状態:**
- 問い合わせページへ: 「相談したい」（最も理想的）
- 他の事例へ: 「もっと見たい」
- 離脱: 「今ではないが、候補として覚えておく」

## セクション2: 事例一覧

### なぜこのセクションが必要か

**専門家①（マーケティング）:**
一覧では「選ぶ」ためのミニマルな情報のみ。詳細は各事例ページに委ねる。サムネイル画像とタイトルで、興味を引く事例を探させる。

**専門家②（心理学）:**
選択肢は3-6件に絞る。多すぎると選べない（選択のパラドックス）。少なすぎると実績不足に見える。各事例のサムネイルで「業種」や「成果物」を瞬時に把握させる。

**専門家③（デザイン）:**
グリッドレイアウトだが、均等ではなく変化をつける。最初の事例を大きく、残りを小さくするなど、視覚的な優先順位を設計。

### レイアウト

**構成:**
最初の事例を大きく、残りを2カラムで表示。

```
+--------------------------------------------------+
|                                                  |
|  +--------------------------------------------+  |
|  |                                            |  |
|  |        [事例1: メイン事例]                  |  |
|  |        大きなサムネイル                     |  |
|  |                                            |  |
|  +--------------------------------------------+  |
|  クライアント名                                  |
|  プロジェクト概要                                |
|                                                  |
|  +--------------------+  +--------------------+  |
|  | [事例2]            |  | [事例3]            |  |
|  |                    |  |                    |  |
|  +--------------------+  +--------------------+  |
|  クライアント名          クライアント名          |
|                                                  |
|  +--------------------+  +--------------------+  |
|  | [事例4]            |  | [事例5]            |  |
|  |                    |  |                    |  |
|  +--------------------+  +--------------------+  |
|  クライアント名          クライアント名          |
|                                                  |
+--------------------------------------------------+
```

**なぜこのレイアウトか:**
- 最も見せたい事例を大きく目立たせる
- 残りは均等に並べ、選びやすくする
- グリッドだが、サイズの変化でリズムを作る

**代替案を採用しない理由:**
- すべて均等: 優先順位がつかない
- マソンリーレイアウト: カジュアルすぎる
- カルーセル: 一覧性が低い

### コンテンツ要素

**要素1: メイン事例（最初の1件）**

サムネイル画像:
- サイズ: 幅100%、アスペクト比 16:9
- ホバー時: 透明度0.9に変化

クライアント名:
- サイズ: 18px、游ゴシック Medium、#333333
- 画像から24px下

プロジェクト概要:
- サイズ: 14px、游ゴシック、#666666
- クライアント名から8px下
- 1行に収める（超過は...で省略）

**要素2-5: サブ事例（残り4件）**

サムネイル画像:
- サイズ: 幅100%（2カラムの各列幅）、アスペクト比 4:3
- ホバー時: 透明度0.9に変化

クライアント名:
- サイズ: 16px、游ゴシック Medium、#333333
- 画像から16px下

### デザイン詳細

**背景:** #FFFFFF

**余白:**
- メイン事例の下余白: 80px
- 2カラムグリッドのガター: 32px
- 各行間の余白: 64px

**特徴的な表現:**
- 画像に影やボーダーなし
- ホバーは透明度変化のみ（控えめ）
- カーソルはpointerに変更

### アニメーション

**使用する/しない:** 使用しない

**理由:**
一覧ページで要素が動くと煩雑。静止した状態で全体を俯瞰させる。

### SP（スマートフォン）での変化

**レイアウト変更:**
- すべて1カラム
- メイン事例とサブ事例のサイズ差をなくす

**サイズ変更:**
- 各事例間の余白: 48px

# 事例詳細ページ

## セクション構成

1. **ヒーロー**: 大きな事例写真
2. **プロジェクト概要**: 基本情報
3. **課題**: Before（依頼前の状態）
4. **アプローチ**: プロセス（何をしたか）
5. **成果**: After（どうなったか）
6. **クライアントの声**: 証言（任意）
7. **他の事例へ**: ナビゲーション
8. **お問い合わせ導線**: CTA

---

## セクション1: ヒーロー

### なぜこのセクションが必要か

**専門家①（マーケティング）:**
事例詳細の最初は、成果物のビジュアルで引き込む。テキストより先に「見た目の印象」で興味を喚起。

**専門家②（心理学）:**
視覚は最も処理速度が速い感覚。美しい成果物を見せることで、ポジティブな第一印象を形成（ハロー効果）。

**専門家③（デザイン）:**
全幅ではなく、左右に余白を残した配置。画像が「展示」されている感覚。

### レイアウト

**構成:**

```
+--------------------------------------------------+
|                                                  |
|                                                  |
|      +------------------------------------+      |
|      |                                    |      |
|      |      [メインビジュアル]             |      |
|      |      成果物または現場の写真         |      |
|      |                                    |      |
|      +------------------------------------+      |
|                                                  |
+--------------------------------------------------+
```

### コンテンツ要素

**要素1: メイン画像**
- 位置: 左右各5%のマージン
- サイズ: 幅90%、アスペクト比 16:9 または 3:2
- 画質: 高解像度（2400px幅程度）

### デザイン詳細

**背景:** #FFFFFF
**余白:** 上 120px（ヘッダーから）、下 80px

### アニメーション

**使用する/しない:** 使用しない

---

## セクション2: プロジェクト概要

### なぜこのセクションが必要か

**専門家①（マーケティング）:**
基本情報を整理して提示。クライアント名、業種、プロジェクト期間、担当領域など。類似性を判断するための情報。

**専門家②（心理学）:**
「自分と似ているか」を判断するための客観情報。業種や規模感で自己投影を促す。

**専門家③（デザイン）:**
左に概要情報、右にタイトルという非対称配置。表組みではなく、ラベル+値のシンプルな形式。

### レイアウト

**構成:**

```
+--------------------------------------------------+
|                                                  |
|   Client        株式会社〇〇                      |
|   Industry      製造業                            |
|   Year          2023                              |
|   Scope         戦略設計 / ブランディング          |
|                                                  |
|   ─────────────────────────────────────          |
|                                                  |
|   〇〇株式会社様 リブランディングプロジェクト      |
|                                                  |
|   創業50年を迎える老舗製造業の、                  |
|   次の50年に向けたブランド刷新を担当しました。    |
|                                                  |
+--------------------------------------------------+
```

### コンテンツ要素

**要素1: 基本情報（4項目）**

ラベル:
- サイズ: 12px、Cormorant Garamond、#999999
- 幅: 100px固定

値:
- サイズ: 15px、游ゴシック、#333333
- ラベルの右に配置

項目間余白: 16px

**要素2: 区切り線**
- 幅: 100%、高さ: 1px、色: #E0E0E0
- 基本情報から48px下

**要素3: プロジェクトタイトル**
- サイズ: 28px（PC）/ 22px（SP）、游明朝、#1A1A1A
- 区切り線から48px下

**要素4: リード文**
- サイズ: 16px、游ゴシック、#333333
- タイトルから24px下
- 行間: 1.9
- 最大幅: 680px

### デザイン詳細

**背景:** #FFFFFF
**余白:** 上80px、下120px
**左右マージン:** 10%

## セクション3: 課題（Before）

### なぜこのセクションが必要か

**専門家①（マーケティング）:**
「課題」から始めることで、読み手が自社の課題と照らし合わせる。共感ポイントを作り、続きを読ませる。

**専門家②（心理学）:**
課題を詳細に描写することで、「自分も同じ」という認識を生む。問題を共有することで、解決への期待が高まる。

**専門家③（デザイン）:**
テキスト中心だが、課題を視覚化する写真やグラフを1点添えても良い。ただし装飾ではなく、内容に直結するもののみ。

### レイアウト

**構成:**

```
+--------------------------------------------------+
|                                                  |
|   Challenge                                      |
|   課題                                            |
|                                                  |
|   創業当時から変わらないロゴとビジュアル           |
|   アイデンティティは、社員の高齢化とともに         |
|   「古い会社」という印象を与えていました。         |
|                                                  |
|   特に採用面で苦戦しており、                      |
|   若い世代に「働きたい」と思われる                 |
|   ブランドイメージへの刷新が急務でした。          |
|                                                  |
|                             [Before写真]         |
|                                                  |
+--------------------------------------------------+
```

### コンテンツ要素

**要素1: セクションタイトル**
- 英語: 14px、Cormorant Garamond、#999999
- 日本語: 24px、游明朝、#1A1A1A

**要素2: 課題テキスト**
- サイズ: 16px、游ゴシック、#333333
- 行間: 2.0
- 段落間: 24px
- 最大幅: 640px

**要素3: Before写真（任意）**
- 位置: テキストの右下、または下
- サイズ: 幅300-400px
- キャプション: 「リブランディング前のロゴ」等

### デザイン詳細

**背景:** #F5F5F5
**余白:** 上160px、下160px
**左右マージン:** 15%

**専門家②（心理学）:**

**専門家③（デザイン）:**

## セクション5: 成果（After）

### なぜこのセクションが必要か

**専門家①（マーケティング）:**
成果を具体的に示す最重要セクション。可能であれば数値で示す。ビジュアルで変化を見せる。

**専門家②（心理学）:**
Before→Afterの変化は最も説得力がある形式。視覚的な変化は特に印象に残る。

**専門家③（デザイン）:**
成果物の写真を複数点、ギャラリー形式で展示。ただしスライダーではなく、スクロールで見られる形式。
