# 理念・代表メッセージページ 詳細設計

**URL**: /philosophy

---

## このページの目的

### 専門家①（マーケティング）の視点

**マーケティング目標:**
- 代表者の人格・価値観で「この人に任せたい」を引き出す
- 価格ではなく「人」で選ばれる土台を築く
- 「なぜこの会社が存在するのか」という根源的な問いに答える
- 紹介・口コミで来た人に「噂通りの人だ」と確信させる

**コンバージョン設計:**
- このページ自体での問い合わせは期待しない
- 事例ページまたはサービスページへの自然な遷移
- 「この人の仕事を見たい」という欲求の喚起

### 専門家②（心理学）の視点

**心理的ゴール:**
- **類似性の法則**: 「この代表者は自分と似た価値観を持っている」
- **専門家への信頼転移**: 顔と言葉を見せることで、組織全体への信頼が生まれる
- **ストーリーへの没入**: 創業の物語で感情的なつながりを形成

**活用する心理効果:**
1. 人格の一貫性: 言葉と行動が一致していることへの信頼
2. 脆弱性の開示: 完璧ではない過去を語ることで親近感
3. 原体験の共有: 「なぜこの仕事をしているか」への共感

### 専門家③（デザイン）の視点

**デザイン上の役割:**
- このテンプレートで最も「人間味」を出すページ
- ただしMinimalの原則は崩さない
- 代表者の写真が主役。テキストは最小限に

**視覚的ゴール:**
- 余白の中に、代表者の存在感が浮かび上がる
- 写真1枚で人格を伝える
- 縦書きを効果的に使い、日本的な「間」を表現

## ユーザーの状態

**このページに来る前の状態:**
- TOPページで「この会社は他と違う」と感じた
- 「どんな人がやっているのか」という好奇心
- 紹介で来た場合、すでに代表者名を知っている可能性

**このページで感じてほしいこと:**
- 「この人の考え方に共感する」
- 「この人なら信頼できる」
- 「この人の仕事を見てみたい」

**このページから離れる時の状態:**
- 事例ページへ: 「実際の仕事を確認したい」
- サービスページへ: 「具体的に何ができるのか知りたい」
- 問い合わせページへ: 「この人に相談したい」（少数だが高品質）

## セクション構成

理念ページは以下の5セクションで構成する:

1. **ページタイトル**: シンプルな見出しのみ
2. **代表者ポートレート**: 大きな写真1枚
3. **代表メッセージ本文**: 創業の原点と理念
4. **プロフィール**: 経歴と略歴
5. **次への導線**: 事例またはサービスへ

## セクション1: ページタイトル

### なぜこのセクションが必要か

**専門家①（マーケティング）:**
「理念」という言葉だけでは何のページかわかりにくい。「Philosophy」や「代表メッセージ」を副題として添え、このページの内容を明確にする。ただし、装飾は最小限に。

**専門家②（心理学）:**
ユーザーは「今どこにいるか」を常に確認したい（認知的な安心感）。明確なページタイトルがあることで、迷子になる不安を解消。

**専門家③（デザイン）:**
TOPページと同様、余白を大きく取った見出し。ページ上部の50%以上を余白とし、贅沢な空間使いを継続。

### レイアウト

**構成:**
画面上部に大きな余白、その下に日本語タイトルと英語サブタイトル。

```
+--------------------------------------------------+
|                                                  |
|                                                  |
|                                                  |
|                                                  |
|   Philosophy                                     | ← 英語（小さく）
|   代表メッセージ                                   | ← 日本語タイトル（大きく）
|                                                  |
+--------------------------------------------------+
```

**なぜこのレイアウトか:**
- 英語を上に小さく、日本語を下に大きく配置（逆の順序だと「英語サイト」的になる）
- 左寄せにすることで、続くコンテンツとの視線の流れを作る
- 中央揃えにしないことで、AIデザイン的な無難さを回避

### コンテンツ要素

**要素1: 英語タイトル**

- 位置: 左から10%、上から55%
- サイズ: 14px
- フォント: Cormorant Garamond
- 字間: 0.2em
- 色: #999999

**要素2: 日本語タイトル**

- 位置: 英語タイトルの16px下
- サイズ: 48px（PC）/ 32px（SP）
- フォント: 游明朝
- 字間: 0.1em
- 色: #1A1A1A

### デザイン詳細

**余白:**

- 下余白: 120px

**特徴的な表現:**
- 装飾なし
- 線やアンダーラインなし
- ただのテキストが、余白によって「作品」になる

### アニメーション

**理由:**
ページ遷移時のフェードのみ。タイトルは最初から表示されている状態が自然。

## セクション2: 代表者ポートレート

### なぜこのセクションが必要か

**専門家①（マーケティング）:**
「顔が見える」ことで信頼性が格段に上がる。特に高単価サービスでは「誰がやるのか」が重要。スタジオ撮影の高品質な写真で、プロフェッショナリズムを示す。

**専門家②（心理学）:**
人間は顔に強く反応する（顔認識は脳の専用領域が処理）。代表者の顔を見せることで、組織全体を「人格化」できる。目が合う写真は特に信頼を高める。

**専門家③（デザイン）:**
このテンプレートで最も大きな写真を使用するセクション。ただし「ヒーローイメージ」的な全面表示ではなく、余白の中に浮かぶように配置。写真自体がアートとして成立する構図を求める。

### レイアウト

**構成:**
画面の右半分に大きな縦長写真。左半分は余白のまま。

```
+--------------------------------------------------+
|                                                  |
|                              +----------------+  |
|                              |                |  |
|                              |   [代表者      |  |
|                              |    ポートレート] |  |
|                              |                |  |
|                              |                |  |
|                              +----------------+  |
|                                                  |
+--------------------------------------------------+
```

**なぜこのレイアウトか:**
- 右側に写真を置くことで、左の余白が「呼吸」の空間になる
- 縦長（3:4または2:3）の写真は、人物のポートレートとして最も自然
- 全面に広げないことで、写真が「飾られている」感覚を出す

**代替案を採用しない理由:**
- 全幅ヒーローイメージ: 押し付けがましい。AIデザインの典型
- 左に写真: 視線が右に流れるため、次のテキストを左に置くと不自然
- 正方形写真: SNS的になりすぎる

### コンテンツ要素

**要素1: 代表者ポートレート写真**

- 位置: 画面右側、右端から5%マージン
- サイズ: 幅 画面幅の45%（最大600px）
- アスペクト比: 3:4（縦長）
- 配置: 垂直方向は上寄せ

撮影指示:
- バストアップ〜ウエストアップ
- 背景: 無地（白またはグレー）または仕事場
- 表情: 微笑みすぎない、真剣さを感じる表情
- 服装: 業種に応じた正装またはスマートカジュアル
- ライティング: 自然光またはストロボのソフトライト

**要素2: 氏名（写真のキャプションとして）**

- 位置: 写真の下、8px余白
- 構成: 役職 + 氏名
- サイズ: 13px
- フォント: 游ゴシック
- 色: #666666
- 書式: 「代表取締役　山田 太郎」

### デザイン詳細

**背景:**
- 色: #FFFFFF

**余白:**
- 上余白: 120px（タイトルセクションから）
- 下余白: 160px
- 左余白（写真まで）: 55%（左半分が余白）

**特徴的な表現:**
- 写真に枠線なし
- 影なし
- 写真自体のクオリティで勝負

### アニメーション

**使用する/しない:** 使用しない

**理由:**
写真は静止した状態で見せる。フェードインやズームは「演出過剰」になる。

### SP（スマートフォン）での変化

**レイアウト変更:**
- 中央配置
- 幅: 80%

**サイズ変更:**
- 上余白: 80px
- 下余白: 80px

### このセクションでの心理効果

**ユーザーが感じること:**
- 「この人がこの会社の代表か」
- 「しっかりした印象の人だ」
- 「話を聞いてみたい」

## セクション3: 代表メッセージ本文

### なぜこのセクションが必要か

**専門家①（マーケティング）:**
代表者の言葉を通じて、会社の価値観と独自性を伝える。これは「セールス」ではなく「共感の創出」。読んだ人が「自分のための会社だ」と感じれば成功。

**専門家②（心理学）:**
物語（ナラティブ）は人間の記憶に残りやすい。単なる理念の羅列ではなく、「なぜこの仕事をしているのか」という原体験を語ることで、感情的なつながりを形成。

**専門家③（デザイン）:**
長文のテキストだが、読みやすさを最優先。1行の文字数を制限し、行間を広く取り、余白の中に文字が浮かぶレイアウト。見出しと本文の強弱をはっきりつける。

### レイアウト

**構成:**
左側に縦書きの大見出し、右側に横書きの本文。

```
+--------------------------------------------------+
|                                                  |
|  本    |                                         |
|  質    |   私がこの仕事を始めたのは、            |
|  を    |   一つの違和感からでした。               |
|  、    |                                         |
|  か    |   20年前、ある大企業のプロジェクトに     |
|  た    |   参画した時のこと...                    |
|  ち    |                                         |
|  に    |   （本文が続く）                         |
|  。    |                                         |
|        |                                         |
|        |                                         |
+--------------------------------------------------+
```

**なぜこのレイアウトか:**
- 縦書きの見出しは日本的な「品格」を表現
- 左に見出し、右に本文という構成は新聞のコラムを想起させ、「読み物」としての格を与える
- 縦書き見出しが本文に並走することで、視線が自然に下へ流れる

**代替案を採用しない理由:**
- 全て横書き: 普通すぎる。他サイトと差別化できない
- 中央揃え: 長文を中央揃えにすると読みにくい
- 写真と交互配置: この段階では言葉に集中させたい

### コンテンツ要素

**要素1: 縦書き大見出し**

- 位置: 左から10%
- サイズ: 36px（PC）/ 24px（SP）
- フォント: 游明朝
- 書字方向: 縦書き（vertical-rl）
- 色: #1A1A1A
- 高さ: 本文の高さに合わせる（または固定高さ）

**要素2: 本文**

- 位置: 左から25%〜右から15%（幅60%程度）
- サイズ: 16px
- フォント: 游ゴシック
- 行間: 2.0
- 字間: 0.02em
- 色: #333333
- 段落間余白: 32px

本文の構成（推奨）:

```
第1段落: 導入（なぜこの話をするのか）
第2段落: 原体験（どんな経験がきっかけだったか）
第3段落: 気づき（その経験から何を学んだか）
第4段落: 決意（だからこの仕事をしている）
第5段落: 約束（お客様に対するスタンス）
```

**要素3: 署名**

- 位置: 本文の最後、右寄せ
- 構成: 役職 + 氏名
- サイズ: 14px
- フォント: 游明朝（氏名部分）
- 色: #333333

### デザイン詳細

**背景:**
- 色: #F5F5F5（極薄いグレー）
- 理由: 写真セクションからの区切りを明確に。また、長文を読む際の目の疲れを軽減

**余白:**
- 上余白: 160px
- 下余白: 160px
- 縦書き見出し左マージン: 10%
- 本文左マージン: 25%
- 本文右マージン: 15%

**特徴的な表現:**
- 縦書きと横書きの混在
- 装飾なし、純粋にタイポグラフィのみ
- 長文だが、余白によって圧迫感なし

### アニメーション

**使用する/しない:** テキストのみフェードイン

**具体的な動き:**
- 縦書き見出しと本文がそれぞれ独立してフェードイン
- ただし、同時にスタート（時差なし）
- opacity 0→1、y 20px→0
- 0.6秒、cubic-bezier(0.4, 0, 0.2, 1)

### SP（スマートフォン）での変化

**レイアウト変更:**
- 縦書き見出しは解除し、横書きの通常見出しに変更
- 見出し→本文の縦積みレイアウト

**サイズ変更:**
- 見出し: 24px（横書き）
- 本文: 15px
- 行間: 1.9
- 左右パディング: 24px

**省略要素:**
- 縦書き表現はSPでは完全に削除

### このセクションでの心理効果

**ユーザーが感じること:**
- 「この人は本物だ」
- 「表面的なことを言っていない」
- 「自分の価値観と合う」

## セクション4: プロフィール

### なぜこのセクションが必要か

**専門家①（マーケティング）:**
経歴・資格は「外部からの権威づけ」。メッセージで人格を見せた後に、客観的な実績を示すことで信頼を補強。ただし、羅列にならないよう、厳選した情報のみ。

**専門家②（心理学）:**
人間は「肩書き」に弱い（権威性バイアス）。大学名、資格、受賞歴などは、理性的な検証なしに信頼を高める。ただし、多すぎると「見せびらかし」になるため、3-5項目に絞る。

**専門家③（デザイン）:**
経歴は「表」形式になりがちだが、Authority Minimalでは表を使わない。テキストの連続で、あくまで「読み物」として見せる。

### レイアウト

**構成:**
中央揃えで、縦に並ぶシンプルなリスト形式。

```
+--------------------------------------------------+
|                                                  |
|                     Profile                      | ← 小さく
|                    プロフィール                   | ← 中サイズ
|                                                  |
|               山田 太郎 / Taro Yamada            |
|                                                  |
|               1975年　東京都生まれ                |
|               1998年　〇〇大学 建築学科 卒業      |
|               2003年　△△事務所 設立             |
|               2015年　□□賞 受賞                 |
|                                                  |
+--------------------------------------------------+
```

**なぜこのレイアウトか:**
- 中央揃えにすることで、「履歴書」的な堅さを和らげる
- 年号→内容のシンプルな構成で読みやすい
- 項目を絞ることで、重要な情報だけが印象に残る

### コンテンツ要素

**要素1: セクションタイトル**

- 英語「Profile」: 14px、Cormorant Garamond、#999999
- 日本語「プロフィール」: 24px、游明朝、#1A1A1A
- 間隔: 8px
- 中央揃え

**要素2: 氏名**

- 日本語 / 英語 の形式
- サイズ: 20px
- フォント: 游ゴシック Medium
- 色: #333333
- 中央揃え
- タイトルから48px下

**要素3: 経歴リスト**

- 形式: 年号　内容（全角スペース区切り）
- サイズ: 15px
- フォント: 游ゴシック
- 色: #333333
- 行間: 2.2（リスト間に余裕）
- 氏名から32px下
- 中央揃え（各行は左揃えに見えるが、ブロック全体は中央）

経歴の推奨項目数: 4-6行
- 生年・出身
- 最終学歴
- 創業または入社
- 主要な受賞・資格（1-2個）

### デザイン詳細

**背景:**
- 色: #FFFFFF（白に戻す）

**余白:**
- 上余白: 160px
- 下余白: 160px

**特徴的な表現:**
- 表組みを使わない
- 箇条書きの・も使わない
- 純粋なテキストの羅列

### アニメーション

**使用する/しない:** 使用しない

**理由:**
経歴は「事実」の羅列。演出は不要。

### SP（スマートフォン）での変化

**レイアウト変更:**
- 変更なし（中央揃え維持）

**サイズ変更:**
- 日本語タイトル: 20px
- 氏名: 18px
- 経歴: 14px

## セクション5: 次への導線

### なぜこのセクションが必要か

**専門家①（マーケティング）:**
理念ページを読み終えたユーザーを、次のアクションへ導く。「この人の仕事を見たい」という気持ちに応えて事例ページへ、または「何ができるか知りたい」に応えてサービスページへ。

**専門家②（心理学）:**
ページ終端で離脱されないよう、次の行動を提案。ただしAuthority Minimalでは「控えめな提案」。強制ではなく、選択肢を示すだけ。

**専門家③（デザイン）:**
フッター直前に、余白の中に2つのリンクを配置。カード型ではなく、テキストリンク形式。

### レイアウト

**構成:**
左右に2つの選択肢を並べる。

```
+--------------------------------------------------+
|                                                  |
|                                                  |
|      Case Studies          Services             |
|      事例を見る →           サービスを見る →     |
|                                                  |
|                                                  |
+--------------------------------------------------+
```

**なぜこのレイアウトか:**
- 2つの選択肢を対等に並べ、ユーザーの関心に応じて選ばせる
- カードやボタンではなく、テキストリンクで控えめに
- 左右の配置で、選択の意思決定を促す

### コンテンツ要素

**要素1-2: ナビゲーションリンク（2つ）**

各リンクの構成:
- 英語: 14px、Cormorant Garamond、#999999
- 日本語: 18px、游ゴシック Medium、#333333 + 「→」
- 間隔: 8px
- ホバー時: 日本語テキストに下線

配置:
- 左のリンク: 左から20%
- 右のリンク: 右から20%
- 垂直方向: セクション中央

### デザイン詳細

**背景:**
- 色: #F5F5F5

**余白:**
- 上余白: 120px
- 下余白: 120px

### アニメーション

**使用する/しない:** 使用しない

### SP（スマートフォン）での変化

**レイアウト変更:**
- 縦積みに変更
- 中央揃え
- 2つのリンク間の余白: 48px

## ページ全体のフロー設計

### 視線の流れ

```
1. タイトル: このページの内容を認識
   ↓
2. ポートレート: 代表者の顔を認識
   ↓
3. メッセージ本文: 考え方・価値観を理解
   ↓
4. プロフィール: 客観的な経歴で権威を確認
   ↓
5. 導線: 次のページを選択
```

### スクロール体験

- タイトルセクション: 1画面目（スクロールなしで見える）
- ポートレート: 1.5画面目（少しスクロールで全体が見える）
- メッセージ本文: 2-4画面目（じっくり読む）
- プロフィール: 5画面目
- 導線: 6画面目

## 実装時の注意点

### 写真について

- ファイルサイズ: 200KB以下を目標（品質とのバランス）
- フォーマット: WebP優先、フォールバックにJPEG
- 遅延読み込み: FV以外はlazy loading
- alt属性: 「代表取締役 山田太郎のポートレート写真」等

### 縦書きの実装

```css
.vertical-heading {
  writing-mode: vertical-rl;
  text-orientation: upright;
  white-space: nowrap;
}
```

IE非対応だが、2024年以降は考慮不要。

### 長文の読みやすさ

- 1行の最大文字数: 40文字程度
- これ以上長いと視線の移動が大きくなり、読みにくい
- max-width: 680px 程度で制限
