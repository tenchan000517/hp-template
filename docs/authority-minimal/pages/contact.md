# お問い合わせページ 詳細設計

**URL**: /contact

---

## このページの目的

### 専門家①（マーケティング）の視点

**マーケティング目標:**
- サイト全体の最終ゴール（コンバージョンポイント）
- 「真剣な問い合わせ」のみを集める（冷やかし排除）
- 問い合わせのハードルを適度に保つ（高すぎず低すぎず）
- 送信後の期待値を適切に設定する

**コンバージョン設計:**
- フォーム項目を多めにし、質の高いリードを獲得
- 「お気軽に」ではなく「ご相談」というトーン
- 送信完了後は明確な次ステップを提示

### 専門家②（心理学）の視点

**心理的ゴール:**
- **コミットメントの獲得**: フォーム入力という行動で意思を固める
- **不安の解消**: 「送信したらどうなるか」を明示
- **選択の正当化**: 「この会社に問い合わせて良かった」と思わせる

**活用する心理効果:**
1. 一貫性の原理: フォーム入力 → 相談 → 契約への流れ
2. 返報性: 丁寧な対応を約束 → 信頼を返したくなる
3. 認知的閉合: 送信完了で「一歩進んだ」満足感

### 専門家③（デザイン）の視点

**デザイン上の役割:**
- フォームは機能的要素だが、品格を損なわない
- 入力欄は大きく、余白を十分に
- 送信ボタンは控えめだが確実に認識できる

**視覚的ゴール:**
- フォーム要素もMinimalデザインの一部として統合
- 装飾を排し、入力に集中できる環境
- 完了後のサンクスメッセージも同じトーンで

## ユーザーの状態

**このページに来る前の状態:**
- 会社について十分に理解した
- 「相談したい」という意思が固まっている
- または「まず話を聞いてみたい」という軽い気持ち

**このページで感じてほしいこと:**
- 「入力しやすい」
- 「送信しても売り込まれなさそう」
- 「きちんと対応してもらえそう」

**このページから離れる時の状態:**
- 送信完了: 「第一歩を踏み出せた」
- 離脱: 「今ではないが、いつでも戻れる」

## セクション構成

お問い合わせページは以下の4セクションで構成する:

1. **ページタイトル**: シンプルな見出し + 導入文
2. **フォーム本体**: 入力フィールド群
3. **送信ボタン**: 控えめだが明確なCTA
4. **補足情報**: 対応時間・注意事項

## セクション2: フォーム本体

### なぜこのセクションが必要か

**専門家①（マーケティング）:**
フォーム項目は「多すぎず少なすぎず」。Authority Minimalでは意図的に項目を多めにし、真剣な問い合わせのみを獲得。項目が多いことで「ちゃんと読んでくれそう」という期待も生まれる。

**専門家②（心理学）:**
入力項目は「質問に答える」行為。一問一答形式で、圧迫感を与えずに情報を引き出す。必須/任意を明確にし、「とりあえず送れる」状態も許容。

**専門家③（デザイン）:**
フォーム要素は通常「事務的」になりがちだが、余白とフォントサイズで品格を維持。入力欄は大きめに、ラベルはグレーで控えめに。

### レイアウト

**構成:**
縦に並ぶ一問一答形式。

```
+--------------------------------------------------+
|                                                  |
|   お名前 *                                        |
|   +--------------------------------------------+ |
|   |                                            | |
|   +--------------------------------------------+ |
|                                                  |
|   会社名                                          |
|   +--------------------------------------------+ |
|   |                                            | |
|   +--------------------------------------------+ |
|                                                  |
|   メールアドレス *                                |
|   +--------------------------------------------+ |
|   |                                            | |
|   +--------------------------------------------+ |
|                                                  |
|   電話番号                                        |
|   +--------------------------------------------+ |
|   |                                            | |
|   +--------------------------------------------+ |
|                                                  |
|   ご相談内容 *                                    |
|   ○ 新規プロジェクトのご相談                     |
|   ○ 既存サイトの改善                             |
|   ○ ブランディングについて                       |
|   ○ その他                                       |
|                                                  |
|   プロジェクトの概要 *                            |
|   +--------------------------------------------+ |
|   |                                            | |
|   |                                            | |
|   |                                            | |
|   +--------------------------------------------+ |
|                                                  |
|   ご予算                                          |
|   [ 選択してください ▼ ]                         |
|                                                  |
|   ご希望の時期                                    |
|   [ 選択してください ▼ ]                         |
|                                                  |
+--------------------------------------------------+
```

**なぜこのレイアウトか:**
- 1カラムで縦に並べ、スクロールで自然に入力
- 項目間に十分な余白を取り、圧迫感を排除
- 入力欄は横幅いっぱいで、入力しやすく

**代替案を採用しない理由:**
- 2カラム: 複雑に見え、SPで崩れる
- ステップ形式: オーバースペック
- チャットボット形式: Authority Minimalのトーンに合わない

### フォーム項目

| 項目名 | タイプ | 必須/任意 | 補足 |
|--------|--------|----------|------|
| お名前 | text | 必須 | placeholder: 山田 太郎 |
| 会社名 | text | 任意 | 個人の方は「個人」と記載可 |
| メールアドレス | email | 必須 | 自動バリデーション |
| 電話番号 | tel | 任意 | ハイフンなしでも可 |
| ご相談内容 | radio | 必須 | 4択、その他含む |
| プロジェクトの概要 | textarea | 必須 | 5行以上推奨 |
| ご予算 | select | 任意 | 未定を含む選択肢 |
| ご希望の時期 | select | 任意 | 未定を含む選択肢 |

**ご相談内容の選択肢:**
1. 新規プロジェクトのご相談
2. 既存サイト・ブランドの改善
3. ブランディング・戦略について
4. その他

**ご予算の選択肢:**
1. 選択してください（初期値）
2. 50万円未満
3. 50万円〜100万円
4. 100万円〜300万円
5. 300万円〜500万円
6. 500万円以上
7. 未定・相談したい

**ご希望の時期の選択肢:**
1. 選択してください（初期値）
2. なるべく早く
3. 1ヶ月以内
4. 3ヶ月以内
5. 半年以内
6. 時期は未定

### コンテンツ要素

**ラベル:**
- サイズ: 14px、游ゴシック、#333333
- 必須マーク「*」: 同サイズ、#C0392B（赤すぎない赤）
- 位置: 入力欄の上、8px余白

**入力欄（text / email / tel）:**
- 高さ: 56px
- パディング: 0 16px
- ボーダー: 1px solid #E0E0E0
- 角丸: 0（直線的）
- フォント: 16px、游ゴシック、#333333
- フォーカス時: ボーダー色 #333333

**テキストエリア（textarea）:**
- 高さ: 200px
- パディング: 16px
- その他は入力欄と同じ

**ラジオボタン:**
- カスタムスタイル（デフォルトを上書き）
- サイズ: 20px × 20px
- 選択時: 中央に#333333の円
- ラベルサイズ: 15px

**セレクトボックス:**
- 高さ: 56px
- 外観はテキスト入力と統一
- 矢印: 右端にシンプルな▼

### デザイン詳細

**背景:** #FFFFFF

**余白:**
- 各項目間: 40px
- ラベル〜入力欄: 8px
- 左右マージン: 20%（PC）/ 24px（SP）
- フォーム全体の最大幅: 600px、中央揃え

**特徴的な表現:**
- 角丸なし（シャープなエッジ）
- 影なし
- フォーカス時のボーダー変化のみ

### バリデーション

**リアルタイムバリデーション:**
- メールアドレス: 形式チェック
- 必須項目: 空欄チェック

**エラー表示:**
- メッセージ: 入力欄の下に赤文字（14px、#C0392B）
- 入力欄ボーダー: #C0392Bに変化

**エラーメッセージ例:**
- 「お名前を入力してください」
- 「有効なメールアドレスを入力してください」
- 「ご相談内容を選択してください」
- 「プロジェクトの概要を入力してください」

### アニメーション

## セクション3: 送信ボタン

### なぜこのセクションが必要か

**専門家①（マーケティング）:**
送信ボタンは最終アクション。目立たせすぎると押し売り感が出る。Authority Minimalでは、控えめだが確実に認識できるデザイン。

**専門家②（心理学）:**
ボタンのラベルは「送信する」ではなく「相談する」「ご依頼を送信」など、行動の意味を明確に。「送信」だけでは機械的。

**専門家③（デザイン）:**
派手なボタンではなく、ボーダーのみのシンプルなボタン。ホバー時に反転。

### レイアウト

```
+--------------------------------------------------+
|                                                  |
|   [ ] 個人情報保護方針に同意する                  |
|                                                  |
|              [送信する]                          |
|                                                  |
+--------------------------------------------------+
```

### コンテンツ要素

**要素1: プライバシーポリシー同意チェックボックス**
- サイズ: 20px × 20px
- ラベル: 15px、游ゴシック、#333333
- リンク「個人情報保護方針」: 下線付き、ホバーで色変化

**要素2: 送信ボタン**
- サイズ: 幅 200px、高さ 56px
- フォント: 15px、游ゴシック Medium
- 色: #333333
- ボーダー: 1px solid #333333
- 背景: transparent
- ホバー時: 背景 #333333、テキスト #FFFFFF
- トランジション: 0.2s ease

### デザイン詳細

**背景:** #FFFFFF
**余白:** フォームから80px下、セクション下120px
**配置:** 中央揃え

**無効時（バリデーションエラー）:**
- opacity: 0.5
- cursor: not-allowed

## ページ全体のフロー設計

### 視線の流れ

```
1. タイトル + 導入: このページの目的を認識
   ↓
2. フォーム入力: 一問一答で回答
   ↓
3. 同意 + 送信: 最終確認
   ↓
4. サンクス: 完了の確認
```

### 期待される行動

1. フォーム送信（最優先）
2. 電話での問い合わせ（急ぎの場合）
3. 離脱（今ではない）
