# チームページ 詳細設計

**URL**: /team

---

## このページの目的

### 専門家①（マーケティング）の視点

**マーケティング目標:**
- 「誰がやるのか」という不安の解消
- チーム全体の専門性・多様性を示す
- 「この人たちに任せたい」という安心感の創出
- 個人ではなくチームとしての信頼構築

**コンバージョン設計:**
- 直接的なCVは期待しない
- 会社全体への信頼を補強するページ
- 問い合わせ前の最終確認として機能

### 専門家②（心理学）の視点

**心理的ゴール:**
- **人格化**: 組織を「顔のある人々」として認識させる
- **専門性の分散**: 代表者だけでなく、チーム全体が専門家
- **親近感**: 「一緒に仕事をする人たち」としてのイメージ形成

**活用する心理効果:**
1. 単純接触効果: 顔を見せることで親近感
2. 社会的証明: 「この人たちが選んだ会社」という暗示
3. 能力の証明: 経歴・資格での権威づけ

### 専門家③（デザイン）の視点

**デザイン上の役割:**
- メンバーの「人となり」を感じさせるページ
- 写真が主役だが、ギャラリーではなく「紹介」として見せる
- 統一感のある写真スタイルでプロフェッショナリズムを表現

**視覚的ゴール:**
- 余白の中にメンバーが並ぶ静かな構成
- 写真は同じトーン、同じ構図で統一
- テキストは最小限、写真で語る

---

## ユーザーの状態

**このページに来る前の状態:**
- 会社について一通り理解した
- 「どんな人が働いているのか」を知りたい
- または、特定のメンバー名で検索して来た

**このページで感じてほしいこと:**
- 「しっかりした人たちがいる」
- 「専門性が高そう」
- 「この人たちと仕事をしたい」

**このページから離れる時の状態:**
- 問い合わせページへ: 「相談したい」
- 会社概要ページへ: 会社全体の情報を確認したい
- 離脱: 十分な情報を得た

---

## セクション構成

チームページは以下の3セクションで構成する:

1. **ページタイトル**: シンプルな見出し + 導入文
2. **メンバー一覧**: 全メンバーの紹介
3. **お問い合わせ導線**: 控えめなCTA

## セクション2: メンバー一覧

### なぜこのセクションが必要か

**専門家①（マーケティング）:**
メンバー全員を平等に紹介するか、代表者を強調するかは会社の方針による。Authority Minimalでは、代表者を最初に大きく、残りを小さく配置する「ピラミッド型」を採用。

**専門家②（心理学）:**
人数は3-8名程度が認知しやすい。多すぎると「覚えられない」、少なすぎると「人手が足りなさそう」。代表者を最初に置くことで、組織の求心力を示す。

**専門家③（デザイン）:**
写真のトーン、構図、背景を統一することで、チームとしての一体感を表現。個人プロフィールページへのリンクは設けず、一覧で完結。

### レイアウト

**構成:**
代表者を大きく、他メンバーを2カラムのグリッドで配置。

```
+--------------------------------------------------+
|                                                  |
|                +----------------+                |
|                |                |                |
|                |   [代表者]      |                |
|                |   大きな写真    |                |
|                |                |                |
|                +----------------+                |
|                山田 太郎                          |
|                Taro Yamada                        |
|                代表取締役 / ブランドストラテジスト  |
|                                                  |
|   +----------------+       +----------------+    |
|   |                |       |                |    |
|   |   [メンバー1]   |       |   [メンバー2]   |    |
|   |                |       |                |    |
|   +----------------+       +----------------+    |
|   鈴木 花子                 佐藤 一郎             |
|   Hanako Suzuki             Ichiro Sato          |
|   デザイナー                ストラテジスト         |
|                                                  |
|   +----------------+       +----------------+    |
|   |                |       |                |    |
|   |   [メンバー3]   |       |   [メンバー4]   |    |
|   |                |       |                |    |
|   +----------------+       +----------------+    |
|   田中 次郎                 高橋 三郎             |
|   Jiro Tanaka               Saburo Takahashi     |
|   プロジェクトマネージャー   エンジニア            |
|                                                  |
+--------------------------------------------------+
```

**なぜこのレイアウトか:**
- 代表者を最初に大きく配置し、組織の中心を明確に
- 他メンバーは2カラムで均等に紹介
- 写真とテキストの配置を統一し、整然とした印象

**代替案を採用しない理由:**
- 全員同サイズ: 代表者の特別感がなくなる
- カルーセル: 一覧性が低い
- 詳細ページへのリンク: 情報過多。一覧で完結させる

### コンテンツ要素

**代表者（1名）**

写真:
- サイズ: 幅400px（PC）/ 幅80%（SP）
- アスペクト比: 3:4（縦長）
- 配置: 中央

氏名（日本語）:
- サイズ: 20px、游ゴシック Medium、#333333
- 写真から24px下
- 中央揃え

氏名（英語）:
- サイズ: 13px、Cormorant Garamond、#999999
- 日本語から8px下
- 中央揃え

肩書き:
- サイズ: 14px、游ゴシック、#666666
- 英語から12px下
- 中央揃え

**その他メンバー（2-7名）**

写真:
- サイズ: 2カラムグリッド、各幅48%（ガター4%）
- アスペクト比: 3:4（縦長）

氏名（日本語）:
- サイズ: 16px、游ゴシック Medium、#333333
- 写真から16px下

氏名（英語）:
- サイズ: 12px、Cormorant Garamond、#999999
- 日本語から4px下

肩書き:
- サイズ: 13px、游ゴシック、#666666
- 英語から8px下

### デザイン詳細

**背景:** #FFFFFF

**余白:**
- 代表者の下余白: 100px
- メンバー行間: 64px
- セクション下余白: 120px
- 左右マージン: 15%

**特徴的な表現:**
- 写真は全員同じ背景色（白またはグレー）で統一
- 写真に枠線や影なし
- 名前と肩書きのみ、経歴詳細は記載しない

### 写真撮影の統一ルール

**統一すべき要素:**
- 背景: 白または薄いグレー（全員同じ）
- 構図: バストアップ、正面やや斜め
- ライティング: 自然光またはソフトライト
- 服装: 統一感はあるが個性を残す（例: 全員白シャツ）
- 表情: 微笑み、しかし笑いすぎない

**避けるべきこと:**
- 背景がバラバラ
- 解像度や色味が異なる
- 撮影時期が明らかに違う
- スマホの自撮り

### アニメーション

**使用する/しない:** 使用しない

**理由:**
人物写真がフェードインすると演出過剰。静止した状態で、落ち着いて見てもらう。

### SP（スマートフォン）での変化

**レイアウト変更:**
- 代表者: 幅80%
- その他メンバー: 1カラム（幅80%）

**サイズ変更:**
- 代表者氏名: 18px
- メンバー氏名: 15px
- メンバー間余白: 48px

   ↓

## バリエーション

### メンバー数による調整

**3名の場合:**
- 代表者を中央大、残り2名を左右に小さく配置
- または全員同サイズで3カラム

**5名以上の場合:**
- 代表者を大きく、残りを2カラムグリッド
- 行数が増えても問題なし

**代表者1名のみの場合:**
- チームページ自体を設けず、理念ページに統合
- または「About」ページとして会社概要と統合

### 経歴を載せるか

**載せない（推奨）:**
- シンプルで、Authority Minimalの原則に沿う
- 詳細は問い合わせ後に伝える

**載せる場合:**
- 2-3行の短い経歴
- 氏名の下に追加
- フォント: 13px、游ゴシック、#666666

## 実装時の注意点

### データ構造

```json
{
  "members": [
    {
      "id": "yamada",
      "nameJa": "山田 太郎",
      "nameEn": "Taro Yamada",
      "title": "代表取締役 / ブランドストラテジスト",
      "image": "/images/team/yamada.jpg",
      "isRepresentative": true
    },
    {
      "id": "suzuki",
      "nameJa": "鈴木 花子",
      "nameEn": "Hanako Suzuki",
      "title": "デザイナー",
      "image": "/images/team/suzuki.jpg",
      "isRepresentative": false
    }
  ]
}
```

### 画像要件

- サイズ: 800px × 1067px（3:4）推奨
- 形式: WebP優先、フォールバックにJPEG
- 容量: 各100KB以下
- 命名: 名前のローマ字（yamada.jpg）

### アクセシビリティ

- alt属性: 「[氏名]のポートレート写真」
- 画像がない場合のフォールバック: イニシャルを表示
