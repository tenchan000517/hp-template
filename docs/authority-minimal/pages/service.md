# サービスページ 詳細設計

**URL**: /service

---

## このページの目的

### 専門家①（マーケティング）の視点

**マーケティング目標:**
- 「何ができるか」ではなく「何を解決するか」を伝える
- 価格ではなく価値で判断させる土台を作る
- 競合との比較を無効化する独自性の提示
- 問い合わせへの心理的ハードルを下げる（ただし急かさない）

**コンバージョン設計:**
- 直接の問い合わせよりも、事例ページへの遷移を優先
- 「このサービスで何ができたか」を見せることで説得力を増す
- 料金は掲載しない（問い合わせ後に個別提案）

### 専門家②（心理学）の視点

**心理的ゴール:**
- **問題の言語化**: 「自分の課題を言葉にしてもらえた」という感覚
- **解決可能性の認知**: 「この会社なら解決できそう」という期待
- **リスク低減**: 「まず相談だけでも」と思える安心感

**活用する心理効果:**
1. フレーミング効果: 課題を「問題」ではなく「機会」として提示
2. 社会的証明: 「多くの企業が同じ課題を抱えている」という暗示
3. 権威性: 独自のフレームワークや方法論の提示

### 専門家③（デザイン）の視点

**デザイン上の役割:**
- サービス内容を「読ませる」ページ
- しかし長文の羅列ではなく、視覚的なリズムを持たせる
- 各サービスに独自性を感じさせつつ、統一感を維持

**視覚的ゴール:**
- 余白とタイポグラフィで情報を整理
- サービス間の区切りを余白のみで表現
- 写真やアイコンに頼らない、言葉の力で勝負

---

## ユーザーの状態

**このページに来る前の状態:**
- 理念ページで代表者への信頼を形成済み
- または TOPページから直接、「何をする会社か」を知りたい
- 「自分の課題を解決できるか」を確認したい

**このページで感じてほしいこと:**
- 「自分の課題を理解してくれている」
- 「独自の方法論を持っている」
- 「実際の事例を見てみたい」

**このページから離れる時の状態:**
- 事例ページへ: 「実際にどう解決したのか見たい」
- 問い合わせページへ: 「相談したい」（高確率で成約に繋がる）

---

## セクション構成

サービスページは以下の5セクションで構成する:

1. **ページタイトル**: シンプルな見出し
2. **サービス哲学**: このサービスの根底にある考え方
3. **サービス詳細**: 3つのサービスカテゴリ
4. **プロセス**: 依頼から完了までの流れ
5. **事例への導線**: 実績を見せる誘導

## セクション1: ページタイトル

### なぜこのセクションが必要か

**専門家①（マーケティング）:**
ユーザーが「サービスページにいる」ことを明確に認識させる。タイトルだけでなく、一文の導入を添えることで、ページの意図を伝える。

**専門家②（心理学）:**
「今どこにいるか」の認知的安心感。また、導入文でユーザーの課題意識に寄り添う言葉を入れることで、読み進める動機を作る。

**専門家③（デザイン）:**
他ページと統一したタイトルセクションのフォーマット。余白を贅沢に使い、ページの品格を示す。

### レイアウト

**構成:**

```
+--------------------------------------------------+
|                                                  |
|                                                  |
|                                                  |
|   Services                                       |
|   私たちの仕事                                    |
|                                                  |
|         本質的な課題に向き合い、                  |
|         持続的な価値を生み出す。                  |
|                                                  |
+--------------------------------------------------+
```

### コンテンツ要素

**要素1: 英語タイトル**
- 位置: 左から10%、上から50%
- サイズ: 14px、Cormorant Garamond、#999999

**要素2: 日本語タイトル**
- 位置: 英語の16px下
- サイズ: 48px（PC）/ 32px（SP）、游明朝、#1A1A1A

**要素3: 導入文**
- 位置: 日本語タイトルから48px下
- サイズ: 18px（PC）/ 16px（SP）、游ゴシック、#333333
- 最大幅: 500px
- 行間: 1.8

### デザイン詳細

**背景:** #FFFFFF
**余白:** 上 ビューポート50%（最低300px）、下 120px

**使用する/しない:** 使用しない

- 導入文: 16px

## セクション2: サービス哲学

### なぜこのセクションが必要か

**専門家①（マーケティング）:**
サービスの詳細を説明する前に、「なぜこのサービスを提供しているのか」を伝える。競合との差別化は「何をするか」ではなく「なぜするか」で生まれる。

**専門家②（心理学）:**
人は「Why」に共感する（ゴールデンサークル理論）。機能説明の前に理念を伝えることで、その後の情報をポジティブに解釈する心理的土台を作る。

**専門家③（デザイン）:**
短いテキストを大きな余白の中に配置。引用のような扱いで、言葉の重みを出す。

### レイアウト

**構成:**
中央に1つの短い段落。

```
+--------------------------------------------------+
|                                                  |
|                                                  |
|       私たちは「答え」を売りません。              |
|       クライアントと共に「問い」を立て、          |
|       本質に迫ることで、                          |
|       表面的な対処ではなく、                      |
|       根本的な変化を実現します。                  |
|                                                  |
|                                                  |
+--------------------------------------------------+
```

**なぜこのレイアウトか:**
- 中央配置で、視線を自然に集める
- 5行程度の短文で、読み飛ばされない
- 余白に囲まれることで、言葉が「浮かび上がる」

### コンテンツ要素

**要素1: 哲学テキスト**
- 位置: 中央揃え、垂直も中央
- サイズ: 20px（PC）/ 17px（SP）
- フォント: 游明朝
- 行間: 2.2
- 色: #333333
- 最大幅: 520px

### デザイン詳細

**背景:** #F5F5F5（薄いグレー）
**余白:** 上 160px、下 160px

**特徴的な表現:**
- 引用符なし
- 装飾なし
- 言葉だけで勝負

### アニメーション

**使用する/しない:** テキストのみフェードイン
- opacity 0→1、y 20px→0
- 0.6秒、cubic-bezier(0.4, 0, 0.2, 1)

### SP（スマートフォン）での変化

- サイズ: 17px
- 上下余白: 100px
- 左右パディング: 32px

## セクション3: サービス詳細

### なぜこのセクションが必要か

**専門家①（マーケティング）:**
具体的なサービス内容を伝えるメインセクション。ただし「機能の羅列」ではなく「解決する課題」を軸に説明。各サービスが連携していることも示す。

**専門家②（心理学）:**
3つに絞ることで、選択のパラドックスを回避。「多すぎて選べない」ではなく「この3つで完結する」という安心感。各サービスの冒頭に「こんな課題はありませんか」を入れ、自分事化を促す。

**専門家③（デザイン）:**
3つのサービスを均等に並べるのではなく、1つずつ縦に展開。各サービス間に十分な余白を取り、情報が混ざらないようにする。

### レイアウト

**構成:**
3つのサービスを縦に配置。各サービスは左右交互に配置して視覚的リズムを作る。

```
+--------------------------------------------------+
|                                                  |
|   01                                             |
|   Strategy                                       |
|   戦略設計                                        |
|                                                  |
|       課題を正しく定義し、                        |
|       進むべき方向を明確にする。                  |
|                                                  |
|       ・現状分析と課題の構造化                    |
|       ・ゴール設定と戦略立案                      |
|       ・実行計画の策定                            |
|                                                  |
+--------------------------------------------------+
|                                                  |
|                                           02     |
|                                       Design     |
|                                      デザイン     |
|                                                  |
|              戦略を具体的なかたちにする。         |
|              美しさと機能を両立させる。           |
|                                                  |
|                   ・ビジュアルアイデンティティ    |
|                   ・ウェブサイト・アプリ設計      |
|                   ・プロダクトデザイン            |
|                                                  |
+--------------------------------------------------+
|                                                  |
|   03                                             |
|   Branding                                       |
|   ブランディング                                  |
|                                                  |
|       一貫した価値を届け続ける                    |
|       仕組みを構築する。                          |
|                                                  |
|       ・ブランド戦略策定                          |
|       ・コミュニケーション設計                    |
|       ・社内浸透プログラム                        |
|                                                  |
+--------------------------------------------------+
```

**なぜこのレイアウトか:**
- 左→右→左の交互配置で、視線に動きを作る
- 番号（01, 02, 03）で順序性と体系性を示す
- 各サービスは「概念説明」→「具体項目」の2層構造

**代替案を採用しない理由:**
- 3カラム配置: 情報が薄まる。詳細を伝えられない
- カード型: AIデザインの典型。囲むと安っぽくなる
- アイコン付き: 装飾過剰。言葉で伝える

### コンテンツ要素

**各サービスの構成要素:**

**要素1: サービス番号**
- サイズ: 14px、Cormorant Garamond、#CCCCCC
- 形式: 「01」「02」「03」

**要素2: 英語名**
- サイズ: 14px、Cormorant Garamond、#999999
- 番号の16px下

**要素3: 日本語名**
- サイズ: 28px（PC）/ 22px（SP）、游明朝、#1A1A1A
- 英語の8px下

**要素4: 概念説明**
- サイズ: 16px、游ゴシック、#333333
- 日本語名から32px下
- 最大幅: 400px
- 行間: 1.9

**要素5: 具体項目**
- サイズ: 15px、游ゴシック、#666666
- 概念説明から24px下
- 形式: 「・」で始まるリスト
- 項目間余白: 12px

### デザイン詳細

**背景:** #FFFFFF

**余白:**
- セクション上余白: 160px
- 各サービス間の余白: 200px
- セクション下余白: 160px

**配置詳細:**
- サービス1（戦略）: 左から10%
- サービス2（デザイン）: 右から10%（右寄せ）
- サービス3（ブランディング）: 左から10%

**特徴的な表現:**
- 境界線なし
- 余白のみで分離
- 番号がうっすらと見える（#CCCCCC）

### アニメーション

**使用する/しない:** 各サービスブロックにフェードイン
- 各ブロックが画面内に入った時にフェードイン
- 3つ同時ではなく、スクロールに応じて順次

### SP（スマートフォン）での変化

**レイアウト変更:**
- すべて左寄せ（交互配置は解除）
- 左パディング: 24px

**サイズ変更:**
- 日本語名: 22px
- 概念説明: 15px
- 具体項目: 14px
- サービス間余白: 120px

## セクション4: プロセス

### なぜこのセクションが必要か

**専門家①（マーケティング）:**
「どう進むのか」の不透明感は問い合わせのハードルになる。プロセスを可視化することで、依頼後のイメージを持たせ、安心感を与える。

**専門家②（心理学）:**
未知への恐怖を軽減。ステップが明確だと「自分にもできそう」と感じる。また、プロセスの存在自体が「体系化されている＝プロフェッショナル」という印象を与える。

**専門家③（デザイン）:**
ステップを横並びではなく、縦に流れるタイムライン形式で表現。ただし一般的なタイムラインデザイン（丸い点と線）は使わず、番号と余白のみで表現。

### レイアウト

**構成:**
縦に流れる4つのステップ。左に番号、右に内容。

```
+--------------------------------------------------+
|                                                  |
|   Process                                        |
|   進め方                                          |
|                                                  |
|   ─────────────────────────────────────          |
|                                                  |
|   01        ヒアリング                            |
|             課題と目標を丁寧にお聞きします。       |
|             この段階で費用は発生しません。         |
|                                                  |
|   02        ご提案                                |
|             課題を整理し、解決の方向性と           |
|             概算費用をご提示します。               |
|                                                  |
|   03        実行                                  |
|             定期的なコミュニケーションを取りながら、|
|             プロジェクトを進行します。             |
|                                                  |
|   04        完了・継続                            |
|             成果を確認し、必要に応じて             |
|             継続的なサポートを行います。           |
|                                                  |
+--------------------------------------------------+
```

**なぜこのレイアウトか:**
- 縦の流れが「時間の経過」を自然に表現
- 左の番号が「今どこか」を明確に
- 説明文は2行程度に抑え、読み飛ばしを防止

### コンテンツ要素

**要素1: セクションタイトル**
- 英語「Process」: 14px、Cormorant Garamond、#999999
- 日本語「進め方」: 28px、游明朝、#1A1A1A

**要素2: 区切り線**
- 幅: 100%
- 太さ: 1px
- 色: #E0E0E0
- タイトルから48px下

**要素3: 各ステップ**

番号:
- サイズ: 14px、Cormorant Garamond、#CCCCCC
- 位置: 左から10%

ステップ名:
- サイズ: 18px、游ゴシック Medium、#333333
- 位置: 左から20%

説明文:
- サイズ: 15px、游ゴシック、#666666
- ステップ名の16px下
- 行間: 1.8

**ステップ間余白:** 64px

### デザイン詳細

**背景:** #F5F5F5

**余白:**
- 上余白: 160px
- 下余白: 160px

**特徴的な表現:**
- タイムラインの「線」を使わない
- 番号と余白だけでステップを表現
- ミニマルなプロセス表示

### アニメーション

**使用する/しない:** 使用しない

### SP（スマートフォン）での変化

**レイアウト変更:**
- 番号とステップ名を縦に積む
- 番号 → ステップ名 → 説明文

**サイズ変更:**
- ステップ名: 16px
- 説明文: 14px
- ステップ間余白: 48px

## セクション5: 事例への導線

### なぜこのセクションが必要か

**専門家①（マーケティング）:**
サービス説明だけでは「本当にできるのか」の疑念が残る。実際の事例を見せることで、説得力を完結させる。問い合わせより先に事例を見てもらう。

**専門家②（心理学）:**
言葉だけの説明より、具体的な成果を見せる方が信頼性が高い（社会的証明）。「まず事例を見てから判断しよう」という安心感。

**専門家③（デザイン）:**
CTAセクションだが、派手なボタンではなく、テキストリンク形式。事例への誘導を優先し、問い合わせは控えめに。

### レイアウト

**構成:**
中央に誘導テキストと、その下に2つのリンク（事例と問い合わせ）。

```
+--------------------------------------------------+
|                                                  |
|                                                  |
|          これまでの仕事をご覧ください。           |
|                                                  |
|           [事例を見る →]    [相談する →]          |
|                                                  |
|                                                  |
+--------------------------------------------------+
```

**なぜこのレイアウトか:**
- 事例を左（先に目に入る位置）、問い合わせを右に配置
- 「まず事例を見て」という優先順位を暗示
- 問い合わせは選択肢として提示するが、強制しない

### コンテンツ要素

**要素1: 誘導テキスト**
- サイズ: 20px、游明朝、#333333
- 中央揃え

**要素2: リンク（2つ）**
- サイズ: 14px、游ゴシック、#666666
- ホバー時: 下線出現
- 間隔: 48px

### デザイン詳細

**背景:** #FFFFFF

**余白:**
- 上余白: 120px
- 下余白: 120px

### アニメーション

**使用する/しない:** 使用しない

### SP（スマートフォン）での変化

**レイアウト変更:**
- リンクを縦に積む
- リンク間余白: 32px

## ページ全体のフロー設計

### 視線の流れ

```
1. タイトル: ページの意図を認識
   ↓
2. サービス哲学: 「なぜ」を理解
   ↓
3. サービス詳細: 「何を」を理解（3つ）
   ↓
4. プロセス: 「どう進むか」を理解
   ↓
5. 導線: 事例または問い合わせへ
```

### 期待される行動

**最優先:** 事例ページへ遷移
**次点:** 問い合わせページへ遷移

---

## 実装時の注意点

### 料金について

- このページに料金情報は掲載しない
- 「費用は個別にご提案」というスタンス
- プロセスのヒアリング段階で「費用は発生しません」と明記

### サービス数について

- 3つを基本とする
- 2つでも可（4つ以上は情報過多）
- サービス数はsite.jsonで動的に管理
