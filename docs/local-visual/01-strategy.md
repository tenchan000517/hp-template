# Local Visual テンプレート 全体戦略

## 1-1. このテンプレートの独自価値

### 専門家①（WEBマーケティング権威）の視点

#### Standard テンプレートとの決定的な違い

Standard テンプレートは「情報を網羅的に伝える」設計であり、ユーザーがページを読み進めることで理解を深める構造になっている。一方、Local Visual は「**3秒で信頼させ、30秒で電話させる**」設計である。

具体的な違い：

| 観点 | Standard | Local Visual |
|-----|----------|--------------|
| 情報設計 | テキスト→画像→CTA | 画像→数字→CTA→テキスト |
| ファーストビュー | キャッチコピー中心 | 施工写真＋地域名＋電話番号 |
| 信頼構築手段 | 会社情報・沿革 | 施工事例の写真点数・ビフォーアフター |
| CTA設計 | ページ下部に1箇所 | 各セクション末尾＋固定ヘッダー |
| 数字の使い方 | 補足情報として配置 | ファーストビューで大きく配置 |

#### このテンプレートでしか実現できないコンバージョン戦略

1. **地域名の視覚的強調**
   - ファーストビューの左上に地域名を大きく配置（例：「岡崎市」が32px以上）
   - 地域検索ユーザーは「自分の地域か」を最初に確認する行動パターンがある
   - この確認を0.5秒以内に完了させることで離脱を防ぐ

2. **実績数字の3点セット**
   - 「創業XX年」「施工実績XXX件」「地域密着XX年」を必ずファーストビューに配置
   - 3つの数字があることで「たまたま1つ良い数字がある」ではなく「総合的に信頼できる」印象を与える
   - 数字は48px以上、単位は24px以下で強弱をつける

3. **電話番号のダブル露出**
   - ヘッダー固定＋ファーストビュー右下に電話番号
   - 緊急性の高いユーザー（水漏れ、屋根損傷など）は電話を最優先する
   - 「今すぐ電話」ボタンはオレンジ系で他要素から浮かせる

4. **施工事例のグリッド密度**
   - PC: 3カラム、SP: 2カラム
   - 1画面に6〜9件の事例サムネイルが見えることで「件数が多い」印象を視覚的に与える
   - Standard の1カラム大型カードでは実現できない「量感」を演出

#### 地域密着ビジネスに特化した設計

1. **対応エリアの視覚化**
   - 地図画像＋市町村リストの両方を配置
   - 「自分の地域が対応範囲内か」を地図でも文字でも確認できる
   - 車での移動時間目安を併記することで「本当に来てくれる」安心感

2. **Googleマップ連携の強化**
   - 住所クリックで Google Maps が開く
   - 「ここから◯分」の距離感を自動計算するリンク
   - MEO対策として構造化データに地域情報を含める

3. **地域実績の可視化**
   - 施工事例に地域タグを必須化（例：「岡崎市 戸建て 外壁塗装」）
   - 地域フィルター機能で「自分の地域の実績」だけを表示可能
   - 「◯◯市での施工実績XX件」を動的に表示

### 専門家②（行動心理学権威）の視点

#### このテンプレートが活用する心理的メカニズム

1. **近接性バイアス（Proximity Bias）**
   - 人は物理的に近い存在に対して親近感と信頼感を持つ
   - 地域名を大きく表示することで「同じ地域の仲間」という心理的ポジションを獲得
   - 「全国対応」よりも「◯◯市専門」の方が信頼される理由はここにある

2. **社会的証明（Social Proof）の地域限定版**
   - 「全国で100万件の実績」より「この地域で500件の実績」の方が強い
   - 地域住民は「自分と同じ立場の人」の選択を参考にする
   - 施工事例に「◯◯町 M様邸」と表記することで「同じ町内の人も利用している」と伝える

3. **可用性ヒューリスティック（Availability Heuristic）**
   - 人は思い出しやすい情報を重要だと判断する
   - 施工事例の写真を大量に見せることで「この会社は実績が多い」という印象を形成
   - 写真は文字より10倍記憶に残りやすい（Picture Superiority Effect）

4. **ピークエンドの法則（Peak-End Rule）**
   - ユーザーは体験の「ピーク」と「終わり」を最も記憶する
   - ファーストビュー（ピーク）で強い印象を与え、CTA（終わり）で行動を促す
   - 中間コンテンツは「ピークの印象を補強する」役割に徹する

5. **損失回避（Loss Aversion）の活用**
   - 「今すぐお電話ください」より「本日中のお問い合わせで◯◯」
   - ただし、地域密着ビジネスでは過度な緊急性演出は逆効果
   - 「いつでもご相談ください」という安心感とのバランスが重要

#### 「地域」という要素がもたらす心理効果

1. **内集団バイアス（In-group Bias）**
   - 同じ地域に住む人を「仲間」と認識する心理
   - 「岡崎市で創業」「地元出身の職人」などの情報が信頼を増幅
   - 方言や地域イベントへの言及も効果的

2. **認知的流暢性（Cognitive Fluency）**
   - 馴染みのある地名は処理しやすく、好印象につながる
   - 「◯◯駅から車で10分」「◯◯高校の近く」など、ローカルランドマークを使う
   - ユーザーが頭の中で場所をイメージできることが重要

3. **単純接触効果（Mere Exposure Effect）**
   - 繰り返し地域名を見ることで親近感が増す
   - ページ内で地域名を5回以上自然に登場させる設計
   - ただし「◯◯市◯◯市◯◯市」という不自然な連呼は逆効果

#### 写真が信頼構築に果たす役割

1. **視覚的真実性（Visual Truth）**
   - 写真は「証拠」として機能する
   - テキストの「丁寧な仕事をします」より、写真の「整った現場」が100倍説得力がある
   - ビフォーアフター写真は「変化の証拠」として最強のコンテンツ

2. **顔の効果（Face Effect）**
   - 人の顔が写っている写真は視線を集める
   - 職人の顔、代表者の顔があることで「実在する人間」という信頼感
   - 作業着姿の職人写真は「本物の職人」という印象を強化

3. **環境の代理評価（Environmental Proxy）**
   - 現場写真の「整理整頓」「養生の丁寧さ」を見て仕事の質を推測する
   - 汚い現場写真は絶対に使わない
   - 晴れた日の明るい写真を基本とする（曇天・雨天は避ける）

4. **一貫性の原理（Consistency Principle）**
   - 施工事例のクオリティが揃っていることで「安定した品質」を連想
   - 写真の明るさ、構図、サイズを統一する
   - 1枚だけ低品質な写真があると全体の印象が下がる

### 専門家③（WEBデザイナー）の視点

#### このテンプレートのビジュアルアイデンティティ

1. **「堅実さ」と「活力」の両立**
   - 地域密着ビジネスに求められるのは「安心感」
   - しかし「古臭い」「活気がない」印象は避けたい
   - 解決策：色使いは保守的に、写真の選定で活力を表現

2. **「プロフェッショナル」と「親しみやすさ」の両立**
   - 技術力を示すシャープな写真
   - 人柄を示す柔らかな人物写真
   - この2種類の写真を交互に配置することでバランスを取る

3. **「情報量」と「余白」の緊張関係**
   - 地域密着ビジネスは伝えたい情報が多い（サービス、エリア、実績、会社情報）
   - しかし情報過多はユーザーを疲れさせる
   - 解決策：セクションごとに「情報密度」を変える。写真セクション→テキストセクション→写真セクションのリズム

#### 「Visual Impact」表現の具体化

1. **写真の支配率**
   - ファーストビュー：写真が画面の70%以上を占める
   - 施工事例セクション：写真が80%以上、テキストは最小限
   - 会社概要セクション：写真50%、テキスト50%

2. **写真の見せ方バリエーション**
   - フルブリード（画面端まで広がる）：ファーストビュー、セクション背景
   - マスク処理（斜めカット、曲線カット）：サービス紹介
   - グリッド配置：施工事例一覧
   - オーバーレイ：テキストを写真の上に配置（透過黒背景）

3. **写真とテキストの関係性**
   - 写真の上にテキストを置く場合：黒オーバーレイ40-60%
   - 写真の横にテキストを置く場合：写真が大きく、テキストはコンパクトに
   - 写真の下にテキストを置く場合：キャプション程度の最小限

#### AIデザインとの差別化ポイント

1. **均一性の排除**
   - AI は「全セクション同じレイアウト」を生成しがち
   - 本テンプレートは意図的に各セクションのレイアウトを変える
   - ただし「変化のパターン」は設計段階で決めておき、無秩序にはしない

2. **カードコンテナの多用を避ける**
   - AI は全てをカードで囲いたがる
   - 本テンプレートでは「必要な時だけカード」を原則とする
   - 写真はカードなしで直接配置する場合が多い

3. **紫・青のグラデーションを使わない**
   - AI が生成しがちな「紫→青」「ピンク→紫」グラデーションは禁止
   - 地域密着ビジネスには不適切（テック企業っぽくなる）
   - 使用する場合は「白→ベージュ」「グレー→白」程度の控えめなもの

4. **過剰なアニメーションを避ける**
   - AI は全要素にフェードインを付けたがる
   - 本テンプレートでは「注目させたい要素だけ」にアニメーション
   - 静止している要素の方が多いことで、動く要素が際立つ

5. **写真のサイズに強弱をつける**
   - AI は全ての写真を同じサイズで並べがち
   - 本テンプレートでは「メイン写真」と「サブ写真」でサイズを変える
   - ギャラリーでも1枚だけ大きくするなどのアクセントをつける

## 1-2. カラー戦略

### メインカラーの推奨系統と心理効果

#### 推奨1：ダークネイビー系（#1a2744 〜 #2d3a4f）

**心理効果**：
- 信頼感、誠実さ、専門性を連想させる
- 建設業、製造業、士業に最適
- 日本人は「紺色」を好む傾向がある（学生服、ビジネススーツの影響）

**使用箇所**：
- ヘッダー背景
- フッター背景
- ファーストビューのテキストオーバーレイ
- セクション見出しのテキスト色

**注意点**：
- 真っ黒（#000000）は使わない。重すぎる
- 純粋な紺（#000080）も使わない。古臭い
- 彩度を少し落とした「ダークネイビー」がベスト

#### 推奨2：ディープグリーン系（#1a3c34 〜 #2d5a4f）

**心理効果**：
- 安心感、自然、環境配慮を連想させる
- 造園業、木造住宅、エコリフォームに最適
- 「長く続いている会社」という印象

**使用箇所**：
- ヘッダー背景
- アクセント色として（ボタン以外）
- 会社概要セクションの背景

#### 推奨3：ウォームグレー系（#4a4a48 〜 #6b6b68）

**心理効果**：
- 洗練、モダン、高品質を連想させる
- 精密加工、設計事務所、ハイエンドリフォームに最適
- 写真を邪魔しない

**使用箇所**：
- テキスト色（見出し）
- 区切り線
- 背景のアクセント

### アクセントカラーの使いどころ

#### 電話番号・CTAボタン：オレンジ系（#e67e22 〜 #f39c12）

**心理効果**：
- 行動喚起、緊急性、温かみ
- 赤ほど攻撃的でなく、黄色ほど軽くない
- 「押しても大丈夫」という安心感

**具体的な使い方**：
- 電話番号のテキスト色（#e67e22）
- 「今すぐお電話」ボタンの背景（#f39c12）
- ホバー時に少し暗くする（#d35400）

**使用ルール**：
- 1画面（ビューポート）内に2箇所まで
- 電話番号とCTAボタン以外には使わない
- 濫用するとうるさくなり、逆効果

#### サブアクセント：ライトブルー系（#3498db 〜 #5dade2）

**心理効果**：
- 清潔感、透明性、信頼
- 水回り工事、クリーニング系に最適

**具体的な使い方**：
- リンクテキストのホバー色
- 選択中のタブの下線
- 図解のアイコン色

### 背景色のバリエーションとリズム

#### 白（#ffffff）
- 基本の背景色
- 写真が映える
- 連続3セクションまで使用可

#### オフホワイト（#f8f9fa）
- 白と交互に使用
- セクションの区切りを視覚化
- 情報量が多いセクションに使用（読みやすくなる）

#### ライトベージュ（#f5f3f0）
- 温かみが必要なセクション
- 代表挨拶、会社概要など「人」が主役のセクション
- 木造住宅、造園など自然素材系に最適

#### ダークセクション（#1a2744 等）
- 1ページ内に1〜2箇所
- ファーストビュー、CTA前のセクション
- 緊張感を与え、次のアクションを促す

#### 背景色のリズム例（TOPページ）

```
セクション1: 写真フルブリード（写真が背景）
セクション2: 白
セクション3: オフホワイト
セクション4: 写真フルブリード
セクション5: 白
セクション6: ダークネイビー（CTA）
```

### 避けるべき色の組み合わせ

1. **紫 + 青のグラデーション**
   - 理由：テック企業、スタートアップの印象
   - 地域密着ビジネスには不適切

2. **ピンク + 紫**
   - 理由：女性向けサービス、エンタメの印象
   - 建設・製造業には不適切

3. **黒 + 金**
   - 理由：高級路線すぎる、近寄りがたい
   - 地域密着の「親しみやすさ」と矛盾

4. **赤の多用**
   - 理由：警告色、攻撃的
   - CTAボタンに赤は使わない（押すのをためらう）

5. **パステルカラーの多用**
   - 理由：カジュアルすぎる、信頼感が薄い
   - 建設・製造業の「堅実さ」と矛盾

6. **彩度が高すぎる色の組み合わせ**
   - 理由：目が疲れる、安っぽく見える
   - 彩度は60-70%程度に抑える

## 1-3. タイポグラフィ戦略

### 見出しのサイズ階層

#### PC（デスクトップ）

| 要素 | サイズ | font-weight | 用途 |
|-----|-------|-------------|------|
| 地域名 | 32px | 700 | ファーストビュー左上 |
| メインキャッチ | 56px | 900 | ファーストビュー中央 |
| 実績数字 | 72px | 900 | ファーストビュー内 |
| 数字の単位 | 24px | 400 | 実績数字の右横 |
| 電話番号 | 36px | 700 | ヘッダー、ファーストビュー |
| H2（セクション見出し） | 40px | 700 | 各セクションの見出し |
| H3（小見出し） | 24px | 600 | セクション内の区切り |
| 本文 | 16px | 400 | 説明テキスト |
| キャプション | 14px | 400 | 写真の説明 |

#### SP（スマートフォン）

| 要素 | サイズ | font-weight | 用途 |
|-----|-------|-------------|------|
| 地域名 | 20px | 700 | ファーストビュー左上 |
| メインキャッチ | 28px | 900 | ファーストビュー中央 |
| 実績数字 | 40px | 900 | ファーストビュー内 |
| 数字の単位 | 16px | 400 | 実績数字の右横 |
| 電話番号 | 24px | 700 | ヘッダー、タップエリア |
| H2（セクション見出し） | 24px | 700 | 各セクションの見出し |
| H3（小見出し） | 18px | 600 | セクション内の区切り |
| 本文 | 15px | 400 | 説明テキスト |
| キャプション | 13px | 400 | 写真の説明 |

### 電話番号の強調方法

1. **サイズによる強調**
   - 周囲のテキストより1.5〜2倍のサイズ
   - PC: 36px、SP: 24px が基準

2. **色による強調**
   - オレンジ系（#e67e22）で他要素から浮かせる
   - 背景がダークの場合は白（#ffffff）

3. **アイコンによる強調**
   - 電話アイコンを番号の左に配置
   - アイコンサイズは数字の0.8倍程度

4. **分割表記**
   - 「0120-XXX-XXX」のようにハイフンで区切る
   - 視認性と記憶しやすさが向上

5. **配置場所**
   - ヘッダー右端（PC）
   - ヘッダー中央（SP）
   - ファーストビュー右下
   - 各セクション末尾のCTA

### 日本語特有の考慮点

#### 字間（letter-spacing）

- 見出し：0.05em 〜 0.1em
- 本文：0em 〜 0.03em
- 日本語は詰めすぎると読みにくい
- 英数字混在時は自動調整されないので注意

#### 行間（line-height）

- 見出し：1.2 〜 1.4
- 本文：1.8 〜 2.0
- 日本語は漢字が密集するため、英語より広めの行間が必要

#### 約物の処理

- 「」『』（）の前後に余計なスペースが入らないよう注意
- 半角と全角の混在を避ける

#### 禁則処理

- 行頭に句読点が来ないよう注意
- 自動で処理されない場合は手動で調整

### 「Visual Impact」表現におけるタイポグラフィの役割

1. **数字を目立たせる**
   - このテンプレートでは「数字」が最も重要な情報
   - 数字専用のスタイル：特大サイズ + 細い単位テキスト
   - 例：「500件」→「500」が72px、「件」が24px

2. **コントラストの最大化**
   - 白背景に黒文字、ダーク背景に白文字
   - グレー文字は補足情報にのみ使用
   - 写真の上に置く場合は必ずオーバーレイを入れる

3. **タイポグラフィで「静」を作る**
   - 写真が「動」を担当
   - テキストは整然と配置し、視線の休憩地点を作る
   - ゴチャゴチャした手書き風フォントは使わない

4. **フォントファミリー**
   - 見出し：Noto Sans JP（700, 900）
   - 本文：Noto Sans JP（400）
   - 数字強調：Inter（tabular-nums）
   - 統一感を優先し、装飾フォントは使わない

## 1-4. 余白とリズム戦略

### セクション間の余白の考え方

#### 基本原則

余白には「呼吸」の役割がある。密度の高いコンテンツの後には大きな余白、軽いコンテンツの後には小さな余白を置く。

#### セクション間余白の基準（PC）

| 前のセクション | 後のセクション | 余白サイズ |
|--------------|--------------|----------|
| 写真フルブリード | テキスト主体 | 80px |
| テキスト主体 | 写真フルブリード | 60px |
| テキスト主体 | テキスト主体 | 100px |
| 写真グリッド | テキスト主体 | 80px |
| ダークセクション | 白背景セクション | 0px（背景色で区切り）|
| 白背景 | オフホワイト背景 | 0px（背景色で区切り）|

#### セクション間余白の基準（SP）

- PC の 60% 程度
- 最小 40px、最大 60px
- スクロールが長くなりすぎないよう調整

### 「Visual Impact」表現における余白の意味

1. **写真を引き立てる余白**
   - 写真の周囲に十分な余白を取ることで、写真が「作品」として見える
   - ギャラリーでは写真間に16〜24pxのギャップ
   - 写真とテキストの間は32px以上

2. **数字を強調する余白**
   - 実績数字の周囲は他のテキストより広い余白
   - 数字の上下に48px以上の空間
   - 数字同士の間は64px以上（PC）

3. **CTAを際立たせる余白**
   - CTAボタンの上に80px以上の余白
   - ボタン自体のパディングは上下16px、左右48px
   - 「他の要素から離れている」ことで重要性を示す

### 写真と余白のバランス

#### フルブリード写真の場合

- 上下左右の余白：0px（画面端まで）
- 写真内のテキスト配置：写真の端から80px以上内側

#### グリッド写真の場合

- 写真間のギャップ：16〜24px
- グリッド全体の左右余白：コンテナ幅（max-width: 1200px）に準拠
- グリッド上下の余白：48〜64px

#### インライン写真（テキスト横）の場合

- 写真とテキストの間：32〜48px
- テキスト側の余白：通常のパラグラフ余白

### 均一な余白を避ける具体的な方法

1. **意図的な非対称**
   - 上余白と下余白を意図的に変える
   - 例：上80px、下40px → 「次のセクションへの期待感」を作る
   - 例：上40px、下80px → 「このセクションの余韻」を残す

2. **コンテンツタイプによる変化**
   - 写真主体セクション：余白小さめ（写真の力で区切る）
   - テキスト主体セクション：余白大きめ（読みやすさ確保）

3. **セクション内の余白変化**
   - 見出しの下：24px
   - パラグラフ間：16px
   - 要素グループ間：48px
   - 全て同じ32pxにしない

4. **視覚的重心の調整**
   - 写真が左にある場合、右のテキストは上寄せ
   - 写真が右にある場合、左のテキストは中央寄せ
   - 「バランスを取る」のではなく「意図的にずらす」

5. **呼吸のリズム**
   - 3セクションごとに「大きな余白セクション」を挟む
   - 密→密→疎→密→密→疎 のリズム
   - ユーザーがスクロールに疲れないための設計

## 1-5. アニメーション戦略

### どこにアニメーションを使い、どこに使わないか

#### アニメーションを使う箇所（限定的）

1. **ファーストビューの要素出現**
   - 写真：即座に表示（アニメーションなし）
   - テキスト：0.3秒後にフェードイン + 上からスライド
   - 目的：写真が主役であることを示す

2. **実績数字のカウントアップ**
   - 0 → 目標値 へ1.5秒でカウントアップ
   - 目的：数字に視線を集める、「動いている会社」の印象

3. **CTAボタンのホバー**
   - 背景色が0.2秒で変化
   - スケールが1.02倍に
   - 目的：クリッカブルであることを示す

4. **施工事例のホバー**
   - 写真がスケール1.05倍に（overflow: hiddenでクリップ）
   - 0.3秒のトランジション
   - 目的：インタラクティブであることを示す

5. **スクロール連動（限定的）**
   - 施工事例グリッドの各アイテムが順番にフェードイン
   - 間隔：0.1秒ずつずらす
   - 目的：「まだ見えていないコンテンツがある」ことを示す

#### アニメーションを使わない箇所

1. **ヘッダー・フッター**
   - 常に静止
   - ユーザーが操作方法を覚えやすい

2. **テキストコンテンツ**
   - 本文、説明文は静止
   - 読みやすさを優先

3. **メイン写真**
   - フルブリード写真は静止
   - 動くと安っぽく見える

4. **フォーム要素**
   - 入力フィールドは静止
   - 操作の確実性を優先

5. **地図・住所**
   - 静止
   - 情報を正確に読み取らせる

### 「Visual Impact」表現におけるアニメーションの役割

1. **静止画の力を活かす**
   - 写真そのものは動かない
   - 周囲の要素を少しだけ動かすことで、写真が際立つ
   - 「動かないことのインパクト」を意識

2. **視線誘導**
   - 最初に見せたい要素を先にフェードイン
   - 次に見せたい要素を0.3秒後にフェードイン
   - ユーザーの視線をコントロール

3. **信頼感とのバランス**
   - 過剰なアニメーションは「軽い」印象
   - 地域密着ビジネスには「落ち着き」が必要
   - アニメーションは「味付け」程度に抑える

### 過剰にならない基準

1. **1画面内でアニメーションする要素は3つまで**
   - 例：見出し、数字、CTAボタン のみ
   - それ以外は静止

2. **アニメーションの長さは0.3秒以内**
   - それ以上長いと「待たされている」感覚
   - ただし数字カウントアップは1.5秒（演出として許容）

3. **スクロールアニメーションは1回のみ**
   - 表示された要素が再度非表示になってアニメーションするのはNG
   - 一度表示されたら静止

4. **ローディングアニメーションは使わない**
   - 画像の遅延読み込みはプレースホルダー（色ブロック）で対応
   - スピナーやスケルトンは使わない

5. **パララックスは使わない**
   - 背景と前景の速度差をつけるエフェクト
   - 酔いやすい、表示崩れのリスク
   - 地域密着ビジネスには不要

### 具体的なアニメーションパターン（3種類）

#### パターン1：フェードスライドイン（上から）

```typescript
// Framer Motion 設定
const fadeSlideIn = {
  initial: { opacity: 0, y: -20 },
  animate: { opacity: 1, y: 0 },
  transition: { duration: 0.3, ease: "easeOut" }
}
```

**使用箇所**：
- セクション見出し
- ファーストビューのテキスト

**タイミング**：
- ビューポートに入った瞬間
- 遅延なし

#### パターン2：カウントアップ

```typescript
// 数字カウントアップの設定
const countUp = {
  from: 0,
  to: targetValue,
  duration: 1.5,
  startOnView: true,
  enableScrollSpy: true,
  scrollSpyOnce: true
}
```

**使用箇所**：
- 実績数字（創業年数、施工件数、地域密着年数）

**タイミング**：
- ビューポートに入った瞬間に開始
- 1回のみ

#### パターン3：ホバースケール

```typescript
// ホバー時のスケール変化
const hoverScale = {
  whileHover: { scale: 1.02 },
  transition: { duration: 0.2, ease: "easeInOut" }
}
```

**使用箇所**：
- CTAボタン
- 施工事例サムネイル（写真内部がスケール、外枠は固定）
- カード型要素

**注意点**：
- 写真のスケールはoverflow: hiddenで切り抜く
- ボタンのスケールは1.02が限度（それ以上は大げさ）
