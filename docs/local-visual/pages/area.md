# 対応エリア ページ詳細設計

## ページ基本情報

**URL**: /area

**meta title**: 対応エリア｜〇〇市・△△市・××町の外壁塗装・リフォーム【会社名】

**meta description**: 【会社名】の対応エリア一覧。〇〇市を中心に、△△市、××町など周辺地域もカバー。地域密着だからこそスピーディーな対応が可能。対応エリア外でもまずはご相談ください。

---

## このページの目的

### 専門家①（マーケティング）の視点

**コンバージョン目標**：
1. 「自分の地域が対応範囲内」の確認
2. 問い合わせへの誘導
3. 地域別の施工事例への誘導

**戦略**：
- 地域名を網羅的に掲載し、MEO/SEO効果を最大化
- 地図と文字リストの両方で対応エリアを可視化
- 「対応エリア外でも相談可能」でカバー範囲を広げる
- 地域ごとの施工実績数を表示し、社会的証明を強化

### 専門家②（心理学）の視点

**心理的ゴール**：
1. 「自分の地域に対応している」確信
2. 「地元に強い会社」という親近感
3. 「すぐ来てくれる」安心感

**活用する心理メカニズム**：
- 近接性バイアス：地域名の大量露出で「地元感」を強化
- 具体性効果：「〇〇市での施工実績XX件」という具体的数字
- 損失回避：「対応エリア外でも相談可能」で機会損失を防ぐ

### 専門家③（デザイン）の視点

**デザイン上の役割**：
- 地図ビジュアルで視覚的にエリアを把握
- 地域名リストは整理されたグリッドで
- 各地域への「入り口」としての機能

## セクション構成

### セクション1: ページヘッダー

#### なぜこのセクションが必要か

**専門家①（マーケティング）**：
- ページ内容を即座に伝える
- 対応エリアの中心地を明示

**専門家②（心理学）**：
- 「正しいページにいる」確認
- 地域名の露出で親近感

**専門家③（デザイン）**：
- 他の下層ページと統一感
- 写真で地域感を演出

#### レイアウト

**構成**：
高さ300px程度の帯状セクション。背景には地域の風景写真または会社外観写真を配置（ぼかし処理）。

```
┌──────────────────────────────────────────────────────────┐
│ (背景: 地域風景写真 + ぼかし + オーバーレイ)             │
│                                                          │
│              「対応エリア」（H1）                         │
│         〇〇市を中心に、周辺地域もカバーしています       │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

**なぜこのレイアウトか**：
- 他の下層ページ（サービス、施工事例）と統一感
- 地域の風景写真で「地元感」を演出
- サブタイトルで中心エリアを明示

#### コンテンツ要素

**ページタイトル（H1）**：
- テキスト：「対応エリア」
- サイズ：48px（PC）、32px（SP）
- 色：白

**サブタイトル**：
- テキスト：「〇〇市を中心に、周辺地域もカバーしています」
- サイズ：18px
- 色：白の80%

#### デザイン詳細

**背景**：
- 地域のランドマーク写真、または会社外観写真
- filter: blur(4px)
- 黒オーバーレイ 60%

**余白**：
- 上：120px（ヘッダー高さ含む）
- 下：60px

#### アニメーション

**使用しない**

#### SP（スマートフォン）での変化

- 高さ：200px
- タイトル：32px

### セクション2: 対応エリアマップ

#### なぜこのセクションが必要か

**専門家①（マーケティング）**：
- 視覚的に対応エリアを把握させる
- 「広い範囲をカバーしている」印象
- 地図はクリックで詳細に飛べる

**専門家②（心理学）**：
- 地図は直感的に理解しやすい
- 色分けで「中心エリア」と「周辺エリア」を区別
- 自分の位置をイメージしやすい

**専門家③（デザイン）**：
- オリジナル地図イラストでサイトの品質を示す
- Googleマップ埋め込みではなく、デザイン性重視
- 下のリストセクションへの導入

#### レイアウト

**構成**：
白背景に、中央に大きな地図イラストを配置。地図上に主要地域名をプロット。

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│              ┌─────────────────────────────┐            │
│              │                             │            │
│              │    [地図イラスト]           │            │
│              │                             │            │
│              │   ・〇〇市（中心）          │            │
│              │   ・△△市                  │            │
│              │   ・××町                  │            │
│              │   ・□□市                  │            │
│              │                             │            │
│              │    対応エリアが色付け       │            │
│              │                             │            │
│              └─────────────────────────────┘            │
│                                                          │
│              本社から車で約30分圏内が主な対応エリア      │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

**なぜこのレイアウトか**：
- 地図が中央にあることで視線を集める
- 対応エリアを視覚的に「面」として認識させる
- 距離感を示す文言で「本当に来てくれる」安心感

**代替案を採用しない理由**：
- Googleマップ埋め込み：デザイン性が低い、読み込み遅延
- 地域名リストのみ：視覚的インパクト不足

#### コンテンツ要素

**地図イラスト**：
- スタイル：フラットデザインの地図イラスト
- サイズ：幅80%（PC）、幅100%（SP）
- 色分け：
  - 中心エリア（〇〇市）：メインカラー濃い
  - 周辺エリア：メインカラー薄い
  - 対応外：グレー
- ピンまたはドット：各地域の中心に配置
- クリック/タップ：該当地域セクションにスクロール

**補足テキスト**：
- 「本社から車で約30分圏内が主な対応エリアです」
- 「記載のない地域もお気軽にご相談ください」

#### デザイン詳細

**背景**：
- 白（#ffffff）

**余白**：
- 上：80px
- 下：48px
- 地図下の補足テキストまで：32px

**特徴的な表現**：
- 地図イラストはSVGで軽量かつ高品質
- ホバーで地域名がツールチップ表示（PC）
- 地域をクリックすると下のリストにスクロール

#### アニメーション

**使用する**：
- 地図：ページ読み込み時にフェードイン
- 地域ピン：順番にポップアップ（0.1秒ずつ遅延）

**理由**：
- 地図への注目を集める
- 「広い範囲」の印象を強化

#### SP（スマートフォン）での変化

**レイアウト変更**：
- 地図幅：100%（横スクロール可能に）
- またはピンチズーム対応

**サイズ変更**：
- 地図高さ：300px 程度

## ページ全体の補足

### SEO観点での地域名の使い方

このページは地域SEO/MEOの要となるページ。以下を意識：

1. **H1**: 「対応エリア」
2. **H2**: 「メイン対応エリア：〇〇市」「周辺対応エリア」など
3. **本文中**: 各地域名を自然に含める
4. **meta description**: 主要地域名を含める
5. **構造化データ**: LocalBusiness の serviceArea を設定

### 構造化データ例

```json
{
  "@context": "https://schema.org",
  "@type": "LocalBusiness",
  "name": "会社名",
  "areaServed": [
    {
      "@type": "City",
      "name": "〇〇市"
    },
    {
      "@type": "City",
      "name": "△△市"
    }
  ]
}
```

### 地域別施工事例へのリンク設計

各地域カードから施工事例ページへのリンク：
- URL例：`/works?area=〇〇市`
- 施工事例ページでクエリパラメータを受け取り、フィルタリング表示
