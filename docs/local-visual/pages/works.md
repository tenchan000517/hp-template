# 施工事例 ページ詳細設計

## ページ基本情報

**URL**: /works（一覧）、/works/[slug]（詳細）

**meta title（一覧）**: 施工事例｜〇〇市の外壁塗装・リフォーム【会社名】

**meta description（一覧）**: 【会社名】の施工事例一覧。〇〇市を中心に500件以上の施工実績。外壁塗装、屋根工事、リフォームのビフォーアフター写真を多数掲載。

---

## このページの目的

### 専門家①（マーケティング）の視点

**コンバージョン目標**：
1. 写真で仕事の質を証明し、問い合わせへ誘導
2. 「自分と同じような案件がある」と感じさせる
3. 地域・サービス別にフィルタリングし、ニーズにマッチさせる

**戦略**：
- 施工事例はサイト内で最も強力な営業ツール
- ビフォーアフター写真で「変化」を可視化
- 地域タグと工事種別タグで探しやすく
- 詳細ページでお客様の声・担当者コメントを追加し説得力を強化

### 専門家②（心理学）の視点

**心理的ゴール**：
1. 「この会社に頼めば、こうなる」という結果のイメージ
2. 「自分の地域でも実績がある」という安心感
3. 「丁寧な仕事をしてくれそう」という品質期待

**活用する心理メカニズム**：
- 社会的証明：多くの事例 = 多くの人が選んでいる
- 類似性バイアス：「自分と同じ地域・同じような家」の事例に共感
- ビジュアル真実性：写真は「証拠」として機能する
- 変化への期待：ビフォーアフターで「自分の家も変わる」想像

### 専門家③（デザイン）の視点

**デザイン上の役割**：
- サイト内で最も写真密度が高いページ
- 写真のクオリティと見せ方がサイト全体の印象を決める
- ギャラリー形式で「量感」と「質」を両立

## セクション構成（一覧ページ）

### セクション1: ページヘッダー

#### なぜこのセクションが必要か

**専門家①（マーケティング）**：
- 施工事例ページであることを即座に伝える

**専門家②（心理学）**：

**専門家③（デザイン）**：
- 他の下層ページと統一感
- 写真が多いページの導入として

#### レイアウト

**構成**：
高さ300px程度の帯状セクション。背景には施工完了後の外観写真を配置。

```
┌──────────────────────────────────────────────────────────┐
│ (背景: 施工完了写真 + ぼかし + オーバーレイ)             │
```

**なぜこのレイアウトか**：
- 施工写真を背景に使うことで「写真が主役」のページを予告
- 総件数を大きく表示し、実績をアピール

#### コンテンツ要素

**ページタイトル（H1）**：
- テキスト：「施工事例」
- サイズ：48px（PC）、32px（SP）
- 色：白

**サブタイトル**：
- テキスト：「これまでの施工実績をご紹介します」

**実績数バッジ**：
- テキスト：「総施工件数: 500件以上」

#### デザイン詳細

**背景**：
- 施工完了後の美しい外観写真
- filter: blur(4px)
- 黒オーバーレイ 50%

**余白**：

### セクション2: フィルター＆ソート

#### なぜこのセクションが必要か

**専門家①（マーケティング）**：
- ユーザーが自分に関係のある事例を見つけやすくする
- 地域・サービス別で検索できることでSEO効果も
- フィルタリングされた件数を表示

**専門家②（心理学）**：
- 「選択できる」コントロール感
- 自分に関連する事例を見つけたときの「これだ」感
- 情報過多を防ぎ、認知負荷を軽減

**専門家③（デザイン）**：
- フィルターUIはシンプルで直感的に
- 選択中のフィルターを視覚的に強調
- 結果件数をリアルタイム表示

#### レイアウト

**構成**：
白背景に、フィルターボタン群 + ソートオプション。

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│   地域: [すべて] [〇〇市] [△△市] [××町] ...           │
│                                                          │
│   工事種別: [すべて] [外壁塗装] [屋根工事] [リフォーム]  │
│                                                          │
│   並び替え: [新しい順 ▼]                                │
│                                                          │
│   該当件数: 150件                                        │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

**なぜこのレイアウトか**：
- 2段のフィルター（地域、工事種別）で複合絞り込み
- タグ形式のボタンで直感的操作
- 件数表示でフィードバック

**代替案を採用しない理由**：
- ドロップダウン：一度に1つしか見えない、選択肢がわかりにくい
- 左サイドバー：SPでの使い勝手が悪い

#### コンテンツ要素

**地域フィルター**：
- オプション：すべて、〇〇市、△△市、××町、□□市、その他
- スタイル：タグ形式ボタン、選択中はメインカラー背景
- 複数選択不可（1つのみ）

**工事種別フィルター**：
- オプション：すべて、外壁塗装、屋根工事、リフォーム、防水工事、その他
- スタイル：同上
- 複数選択不可

**ソートオプション**：
- オプション：新しい順、古い順
- スタイル：ドロップダウン

**該当件数**：
- テキスト：「該当件数: XXX件」
- フィルター変更時にリアルタイム更新

#### デザイン詳細

**背景**：
- 白（#ffffff）

**余白**：
- 上：32px
- 下：32px
- フィルター行間：16px

**特徴的な表現**：
- フィルターボタン：
  - 未選択：白背景、グレー枠線
  - 選択中：メインカラー背景、白テキスト
  - ホバー：背景薄く色付け
- 件数：オレンジ色で強調

#### アニメーション

**使用する**：
- フィルター選択時：下のグリッドがフェードで切り替わり

#### SP（スマートフォン）での変化

**レイアウト変更**：
- フィルターは横スクロール可能なタグ列
- ソートはドロップダウン維持

**サイズ変更**：
- タグボタン：パディング小さめ
- 横スクロール領域：-webkit-overflow-scrolling: touch

**代替案を採用しない理由**：
- 1カラム大型カード：量感が出ない、スクロールが長い
- スライダー：一度に1件しか見えない

- 写真が1.05倍にズーム（overflow: hiddenでクリップ）

- リンク先：/works/[slug]

**ページネーション/もっと見る**：
- 初期表示：12件
- 「もっと見る」ボタンで12件ずつ追加読み込み
- または ページネーション（1, 2, 3, ... , 10）

**背景**：
- オフホワイト（#f8f9fa）

**余白**：
- 上：48px
- 下：80px
- グリッドギャップ：24px（PC）、16px（SP）
- グリッド左右：コンテナ幅

**特徴的な表現**：
- 写真は角丸8px
- 地域タグ：黒オーバーレイ（rgba(0,0,0,0.7)）+ 白テキスト、フォントサイズ12px、パディング4px 8px
- 工事種別：18px、ダークネイビー、font-weight 600
- ホバーオーバーレイ：メインカラー 70%透過

#### アニメーション

**使用する**：
- 初期表示時：各カードが順番にフェードイン（0.05秒ずつ遅延）
- ホバー時：写真ズーム + オーバーレイ出現（0.3秒）
- 「もっと見る」クリック時：追加カードがフェードイン

**理由**：
- 順番にフェードインで「まだある」感覚
- ホバーでインタラクティブ感

#### SP（スマートフォン）での変化

**レイアウト変更**：
- 3カラム → 2カラム
- ギャップ：16px

**サイズ変更**：
- カードサイズ：画面幅の45%程度

**省略要素**：
- ホバーエフェクトは無効（タップで遷移）

### セクション1: ビフォーアフター（メインビジュアル）

#### なぜこのセクションが必要か

**専門家①（マーケティング）**：
- ビフォーアフターは最強のコンテンツ
- 「変化」を見せることで効果を証明
- 最初に見せて興味を引く

**専門家②（心理学）**：
- 変化の視覚化で「自分の家も変わる」想像
- 対比効果で「良くなった」が際立つ

**専門家③（デザイン）**：
- 2分割レイアウトで対比を明確に
- スライダーまたは横並びで見せる

#### レイアウト

**構成**：
白背景に、ビフォー写真とアフター写真を横並び（またはスライダー）で表示。

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│   ┌──────────────────────────────────────────────────┐  │
│   │                                                   │  │
│   │  [BEFORE]           │           [AFTER]          │  │
│   │                     │                            │  │
│   │  施工前の写真       │     施工後の写真           │  │
│   │                     │                            │  │
│   └──────────────────────────────────────────────────┘  │
│                                                          │
│   または スライダー形式（中央のバーをドラッグで比較）    │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

**表示オプション**：
1. **横並び表示**：左にBefore、右にAfter、中央に分割線
2. **スライダー比較**：1枚の写真エリアで、スライダーバーを左右にドラッグしてBefore/Afterを比較

**なぜこのレイアウトか**：
- 対比を一目で把握できる
- スライダー形式は「同じ角度」での比較が効果的
- 横並びは一覧性が高い

#### コンテンツ要素

**Before写真**：
- ラベル：「BEFORE」（写真左上にオーバーレイ）
- 説明：「施工前」

**After写真**：
- ラベル：「AFTER」（写真左上にオーバーレイ）
- 説明：「施工後」

**注意点**：
- Before/Afterは同じアングル、同じ時間帯で撮影
- 天候が同じ（できれば晴天）
- 構図が揃っていること

#### デザイン詳細

**背景**：
- 白（#ffffff）

**余白**：
- 上：48px
- 下：48px

**特徴的な表現**：
- Before/Afterラベル：メインカラー背景、白テキスト
- 横並びの場合：gap 16px、中央に縦線（2px、グレー）
- スライダーの場合：中央バーはメインカラー、ドラッグアイコン付き

#### アニメーション

**使用する（スライダー形式の場合）**：
- ページ読み込み時、スライダーバーが中央から右に一瞬動いて戻る
- 「ドラッグできる」ことを示すヒント

#### SP（スマートフォン）での変化

**レイアウト変更**：
- 横並び → 縦並び（Beforeが上、Afterが下）
- スライダー形式は維持可能

```typescript
type WorkCase = {
  slug: string;                    // URLスラッグ

  beforeImage: string;             // Before写真パス
  afterImage: string;              // After写真パス

}
```
