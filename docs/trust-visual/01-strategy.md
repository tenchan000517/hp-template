# Trust Visual テンプレート 全体戦略

## 1-1. このテンプレートの独自価値

### 👤 専門家①（WEBマーケティング30年の権威）の視点

#### Standard テンプレートとの決定的な違い

Standard テンプレートは「企業情報を整理して伝える」ことが目的であり、既に一定の認知度がある企業向けである。一方、Trust Visual は**「知名度ゼロからでも信頼を獲得する」**ことに特化している。

| 観点 | Standard | Trust Visual |
|------|----------|--------------|
| 前提条件 | 企業名を知っている/紹介で来訪 | 初めて知った/検索で発見 |
| 主な説得手法 | 論理的説明（テキスト中心） | 社会的証明（数字・写真・ロゴ） |
| ファーストビュー | 会社のビジョン・理念 | 実績数字 + 取引先ロゴ |
| 信頼の源泉 | 会社の歴史・理念 | 第三者からの評価 |
| 情報の密度 | 中程度（読ませる設計） | 高め（見せる設計） |
| 写真の役割 | 雰囲気づくり | 証拠としての提示 |

#### このテンプレートでしか実現できないコンバージョン戦略

**「第三者証言による信頼の即時獲得」戦略**

知名度のない企業が「自分たちは素晴らしい」と言っても信頼されない。Trust Visual では、**顧客・取引先・数字という「第三者の声」を全面に押し出す**ことで、自社PRの胡散臭さを回避する。

1. **数字による権威付け**: ファーストビューで「取引実績500社」「顧客満足度98%」「創業35年」等の数字を大きく表示。数字は客観的に見えるため、初対面でも信頼されやすい
2. **取引先ロゴによる信頼の借用**: 有名企業のロゴを見せることで「あの会社が取引しているなら安心」という心理を活用。ロゴは文字より認知が速い
3. **顔写真付きお客様の声**: 匿名の感想ではなく、顔写真・会社名・役職付きで掲載。実在の人物からの推薦は最強の社会的証明
4. **実績写真のギャラリー化**: 納品物・施工事例・作業風景を写真で大量に見せることで「実際にやっている」という信頼を視覚的に獲得

#### 「社会的証明」をどう戦略的に配置するか

**ページごとの社会的証明の配置設計**:

| ページ | 社会的証明の種類 | 配置タイミング | 心理的効果 |
|--------|----------------|--------------|-----------|
| TOP | 実績数字（4つ）+ 取引先ロゴ | ファーストビュー直下 | 「多くの企業が選んでいる」という安心感 |
| TOP | お客様の声（3件サマリー） | 中盤 | 「実際に満足している人がいる」 |
| 選ばれる理由 | 各理由に紐づく数字 | 理由の説明内 | 理由の裏付けとしての説得力 |
| 実績紹介 | 写真ギャラリー | 全面 | 「実際にこれだけやっている」 |
| お客様の声 | 顔写真付き詳細インタビュー | 全面 | 「この人も、この人も推薦している」 |
| 会社概要 | 資格・認証バッジ | 後半 | 「公的機関も認めている」 |

**重要原則**: 社会的証明は「早く、多く、具体的に」見せる。ページ後半に配置するのは効果が薄い。

### 👤 専門家②（日本の行動心理学の権威）の視点

#### このテンプレートが活用する心理的メカニズム

**1. 社会的証明（Social Proof）**

人間は不確実な状況において「他者の行動」を手がかりにする。特に日本人は集団主義的傾向が強く、「みんなが選んでいる」「多くの企業が導入している」という情報に敏感に反応する。

Trust Visual での活用:
- 取引実績の数（「500社以上が導入」）
- 有名企業のロゴ一覧
- 顔写真付きのお客様の声
- 「業界シェアNo.1」等の順位表示

**2. 権威性バイアス（Authority Bias）**

人間は権威ある存在からの情報を無批判に受け入れやすい。Trust Visual では「数字」「資格」「認証」を権威として活用する。

具体的な活用:
- ISO認証、業界団体の認定バッジ
- 「創業○年」という歴史
- メディア掲載実績
- 専門資格保有者の人数

**3. ハロー効果（Halo Effect）**

一つの良い印象が、他の要素の評価にも影響を与える。有名企業のロゴが並んでいると「この会社も一流なのだろう」という推論が働く。

Trust Visual での活用:
- 取引先ロゴは必ず「知名度の高い順」に配置
- 大企業ロゴを左側（視線の起点）に配置
- ロゴの数は多すぎず8-12社程度（多すぎると安っぽく見える）

**4. 同調行動（Conformity）**

日本人は特に「周囲と同じ選択をしたい」という欲求が強い。「多くの企業が選んでいる」という情報は、選択の正当化材料となる。

Trust Visual での活用:
- 「〇〇業界で導入実績No.1」
- 「お客様満足度98%」
- 「リピート率95%」

**5. 具体性効果（Concreteness Effect）**

抽象的な情報より具体的な情報の方が記憶に残り、信頼される。「多くの」より「523社」、「高品質」より「不良率0.01%」の方が信頼される。

Trust Visual での活用:
- 全ての実績を具体的な数字で表現
- 「約500社」ではなく「523社」と表記
- 小数点や端数を含めることで「正確にカウントしている」印象を与える

#### 「信頼」はどのような要素から構築されるか

心理学的な信頼構築モデルを Trust Visual に適用:

**信頼の3要素**:

| 要素 | 定義 | Trust Visual での表現 |
|------|------|---------------------|
| 能力（Ability） | 期待される成果を出せる力 | 実績写真、数字、資格 |
| 誠実さ（Integrity） | 約束を守る姿勢 | 顔写真付きお客様の声、社員紹介 |
| 善意（Benevolence） | 顧客の利益を考える姿勢 | 丁寧な説明、サポート体制の可視化 |

**初対面での信頼構築順序**:

1. **まず能力を示す**（実績数字・取引先ロゴ）
2. **次に誠実さを示す**（顔が見える顧客の声・社員紹介）
3. **最後に善意を示す**（サポート体制・アフターフォロー）

この順序が重要。能力を示す前に善意を強調すると「口だけ」と思われる。

#### 日本人特有の「みんなが選んでいる」への反応

日本人は個人主義の欧米と比較して、以下の特徴がある:

1. **リスク回避傾向が強い**: 「失敗したくない」という気持ちが強く、「他社も使っている」という情報は安心材料になる
2. **「出る杭は打たれる」文化**: 独自の選択より、多数派と同じ選択を好む傾向がある
3. **「お墨付き」への信頼**: 有名企業や公的機関が認めているという事実に弱い

Trust Visual はこれらの心理特性を最大限に活用:
- 取引先ロゴは「大企業」「有名企業」を優先的に掲載
- 「業界No.1」「シェアNo.1」等のNo.1表記を活用（根拠があれば）
- ISO認証、業界団体認定等の「お墨付き」を目立つ位置に配置

### 👤 専門家③（人間最強のWEBデザイナー）の視点

#### このテンプレートのビジュアルアイデンティティ

**「確かな実在感」**

Trust Visual のビジュアルアイデンティティは「確かな実在感」である。抽象的なイメージや装飾ではなく、**「実際の写真」「具体的な数字」「実在する人物」**によって画面を構成する。

具体的な表現:
- **写真は大きく、高解像度で**: 写真が小さいと「隠したいことがある」という印象を与える
- **人物の顔は必ず見せる**: 匿名・シルエットは信頼を下げる
- **数字は画面の主役に**: 96px以上の超大型フォントで表示
- **余白は「引き算」ではなく「呼吸」**: Minimal とは異なり、情報密度は高めだが、適切な呼吸を確保

#### 「Visual Impact」表現で信頼をどう表現するか

**Visual Impact = 視覚的なインパクトで信頼を伝える**

これは「派手に」「目立つように」という意味ではない。**「一目で信頼できると感じる」**視覚設計である。

具体的なアプローチ:

**1. 数字のタイポグラフィ**
- 実績数字は96-120pxの超大型で表示
- 数字部分のみウェイトを900（Black）に
- 数字の色はメインカラー、単位はサブカラーで対比
- カウントアップアニメーションで動的なインパクト

**2. 写真のスケール感**
- 人物写真は顔が識別できるサイズ（最小300px幅）
- 実績写真は画面幅の50%以上を使用
- 写真の重なり・オーバーラップで奥行きを演出

**3. ロゴの整列と統一感**
- 取引先ロゴはグレースケール化して統一感を出す
- 高さを揃え、等間隔で配置
- 背景は薄いグレー（#f8f8f8）で浮かせる

**4. 認証バッジの権威ある配置**
- バッジは原色のまま（権威性を維持）
- 背景は白、周囲に十分な余白
- 「ISO9001認証」等のテキストを添える

#### AIデザインとの差別化ポイント

AIが生成しがちな「信頼感を表現したサイト」の特徴:
1. **アイコンで表現**: 盾のアイコン、チェックマークを多用
2. **青一色**: 信頼＝青という安直な連想
3. **抽象的なイラスト**: 握手するシルエット、上昇するグラフ
4. **均一なカードデザイン**: 全ての情報を同じサイズのカードに
5. **ストックフォト感**: 笑顔のビジネスパーソン、握手する手

**Trust Visual での対抗策**:

1. **アイコンは使わない**: 実際の写真・数字で表現。「信頼」をアイコンで表す必要はない
2. **青以外の選択肢**: ダークグリーン（#1e3a2f）やダークネイビー（#1a2744）等、深みのある色を推奨
3. **実写のみ使用**: イラストやストックフォトは一切禁止。自社で撮影した写真のみ
4. **情報の優先順位を可視化**: 重要な数字は超大型、補足情報は小さく。均一にしない
5. **オリジナル写真の「生感」**: スタジオ撮影より現場感のある写真を重視

## 1-2. カラー戦略

### メインカラーの推奨系統と心理効果（信頼感を与える色）

**推奨①: ダークグリーン系（#1e3a2f 〜 #2d4a3e）**

| 色 | HEX | 心理効果 | 使用場所 |
|----|-----|---------|---------|
| ディープフォレスト | #1e3a2f | 安定、成熟、信頼 | メインテキスト、ヘッダー |
| モスグリーン | #3d5a4c | 落ち着き、誠実 | サブ見出し、ホバー時 |
| セージ | #7a9988 | 柔和、親しみ | サブテキスト、区切り線 |

**なぜダークグリーンか**:
- 青は「信頼」の定番だが、差別化できない
- 緑は「成長」「安定」「誠実」を連想させる
- 日本の伝統色（深緑、利休鼠）との親和性が高い
- 建設・製造・士業等、堅実さが求められる業種に適合

**推奨②: ダークネイビー系（#1a2744 〜 #2a3f5f）**

| 色 | HEX | 心理効果 | 使用場所 |
|----|-----|---------|---------|
| ミッドナイト | #1a2744 | 専門性、権威、信頼 | メインテキスト、ヘッダー |
| ディープブルー | #2a3f5f | 安心、落ち着き | サブ見出し |
| スチールブルー | #5a7a9a | 知性、冷静 | サブテキスト |

**なぜこの紺か**:
- 純粋な青（#0066FF等）は企業サイトの定番すぎる
- 黒に近い紺は「格式」「歴史」を感じさせる
- 金融・コンサル・IT等、知性を売りにする業種に適合

### アクセントカラーの使いどころ

**推奨: 温かみのあるアクセント**

| 系統 | HEX | 使用場所 | 理由 |
|------|-----|---------|------|
| テラコッタ | #c45c3e | CTAボタン | 行動を促す温かみ |
| ゴールド | #b8860b | 数字のハイライト | 価値・成果の強調 |
| コーラル | #e07a5f | ホバー時の変化 | 注目を集めるが派手すぎない |

**アクセントカラーの使用ルール**:
1. **CTAボタンに最優先で使用**: 他の要素と差別化
2. **実績数字の単位部分に使用可**: 「500**社**」の「社」部分
3. **それ以外には使用禁止**: 見出し、アイコン、背景には使わない

### 背景色のバリエーションとリズム

| 色 | HEX | 使用場所 | 目的 |
|----|-----|---------|------|
| ピュアホワイト | #FFFFFF | メインコンテンツ、写真の背景 | 写真を際立たせる |
| オフホワイト | #f8f8f8 | 取引先ロゴセクション、区切り | 領域の分割 |
| ウォームグレー | #f0ebe6 | お客様の声の引用部分 | 温かみと区別 |
| ライトグレー | #e8e8e8 | 表組み、情報整理エリア | 情報の階層化 |
| メインカラー | （上記参照） | ヘッダー、フッター、CTAセクション | 締めと対比 |

**リズムの作り方**:
```
白 → オフホワイト → 白 → メインカラー（ダーク）→ 白 → オフホワイト → メインカラー
```
このように交互にすることで、スクロール時の視覚的リズムを生む。

### 避けるべき色の組み合わせ

1. **純粋な青（#0066FF, #0000FF等）**: 差別化できない。銀行・大企業サイトと同化
2. **紫系全般**: AIデザインの典型。また「胡散臭さ」を連想させるリスク
3. **蛍光色・ネオン系**: 「信頼」と正反対のイメージ
4. **黒（#000000）+ 赤**: 警告・危険を連想。信頼構築に不適
5. **パステルカラー多用**: 軽すぎる印象。堅実さが伝わらない
6. **グラデーション**: 特に青→紫、ピンク→オレンジは完全に禁止

## 1-3. タイポグラフィ戦略

### 見出しのサイズ階層

| 階層 | PC サイズ | SP サイズ | ウェイト | 用途 |
|------|----------|----------|---------|------|
| 数字（実績） | 96-120px | 56-72px | Black (900) | 実績数字の表示 |
| H1 | 56px | 36px | Bold (700) | ページタイトル |
| H2 | 40px | 28px | Bold (700) | セクション見出し |
| H3 | 24px | 20px | SemiBold (600) | サブ見出し |
| 本文 | 17px | 16px | Regular (400) | 通常テキスト |
| キャプション | 14px | 13px | Regular (400) | 補足、写真説明 |

**なぜ実績数字を最大にするか**:
- Trust Visual の核心は「数字で信頼を獲得する」こと
- 数字は視覚的に最も目立つべき要素
- ページを開いた瞬間に数字が目に入ることが重要

### 数字の強調方法（実績数字を目立たせる）

**数字専用のタイポグラフィルール**:

```css
.achievement-number {
  font-size: 96px;
  font-weight: 900;
  font-feature-settings: "tnum"; /* 等幅数字 */
  letter-spacing: -0.02em; /* やや詰める */
  line-height: 1.0;
  color: var(--main-color);
}

.achievement-unit {
  font-size: 24px;
  font-weight: 600;
  color: var(--accent-color);
  margin-left: 4px;
}
```

**カウントアップアニメーション時の考慮**:
- 数字が変化するため、等幅フォント（Tabular Numbers）を使用
- 変化中にガタつかないよう、min-width を設定

**数字の表記ルール**:
- 3桁ごとにカンマ区切り（1,234社）
- 小数点は1桁まで（98.7%）
- 「約」は使わない。具体的な数字を（523社、98.2%）

### 日本語特有の考慮点

**字間（letter-spacing）**:
- 数字見出し: -0.02em（詰めて密度を上げる）
- 日本語見出し: 0.05em（漢字の可読性向上）
- 本文: 0（デフォルト）

**行間（line-height）**:
- 数字見出し: 1.0（塊として見せる）
- 日本語見出し: 1.4
- 本文: 1.8（日本語は横幅が広いため）
- お客様の声引用: 2.0（読みやすさ重視）

**フォントファミリー**:
```css
/* 数字用（等幅） */
font-family: "Inter", "Helvetica Neue", sans-serif;

/* 日本語用 */
font-family: "Noto Sans JP", "Hiragino Kaku Gothic ProN", Meiryo, sans-serif;
```

**数字と日本語の混植時**:
- 数字は欧文フォント（Inter推奨）
- 数字の垂直位置を微調整（vertical-align: -0.05em 程度）

### 「Visual Impact」表現におけるタイポグラフィの役割

Trust Visual では、タイポグラフィ自体が**ビジュアル要素**として機能する。

**1. 数字のスケール感**:
- 96px以上の数字は「画像」と同等の視覚的インパクト
- 数字だけでセクションの主役になれる

**2. コントラストによる視線誘導**:
- 超大型の数字 → 中型の見出し → 小型の説明文
- この落差が視線を自然に誘導

**3. 数字+写真の組み合わせ**:
- 数字を写真にオーバーレイする表現も有効
- 例: 作業風景写真の上に「500」という数字を半透明で配置

## 1-4. 余白とリズム戦略

### セクション間の余白の考え方

**基本原則: 情報密度は高いが、呼吸はある**

Trust Visual は Minimal 系テンプレートと異なり、情報密度は高い。しかし「ぎゅうぎゅう詰め」にはしない。**各情報ブロックの周囲に十分な余白を確保**する。

**セクション間の余白設計**:

| パターン | 上余白 | 下余白 | 使用場面 |
|---------|--------|--------|---------|
| 標準 | 120px | 120px | 通常のセクション間 |
| 強調前 | 160px | 100px | 重要セクションの前（呼吸を持たせる） |
| 連続 | 80px | 80px | 関連性の高いセクション間 |
| 写真ギャラリー | 60px | 60px | 写真が主役の場合（余白より写真を優先） |
| CTAセクション | 100px | 140px | 最後のプッシュ（下は余韻を残す） |

### 「Visual Impact」表現における余白の意味

**余白 = 情報の「額縁」**

美術館で絵画が額縁に入っているように、数字や写真という「作品」に対して余白という「額縁」を与える。

具体的な適用:
- **実績数字の周囲**: 数字の上下左右に最低60pxの余白
- **お客様の声**: 引用部分の左右に40pxのパディング
- **写真ギャラリー**: 写真間のギャップは16-24px（狭め。写真を多く見せたい）

**余白による強弱**:
- 重要な情報 → 周囲の余白を大きく（120px+）
- 補足情報 → 周囲の余白は小さめ（40-60px）

### 情報密度と余白のバランス

**Trust Visual の情報密度マップ**:

```
[密] 取引先ロゴセクション（ロゴが横に8-12個並ぶ）
  ↓
[疎] 実績数字セクション（4つの数字のみ、余白大）
  ↓
[密] お客様の声（写真+テキストが複数）
  ↓
[疎] CTA（ボタン1つのみ、余白大）
```

このリズムにより:
- ユーザーの目が休まるタイミングを作る
- 重要なCTAに視線を集中させる
- スクロール時の体験に変化を与える

### 均一な余白を避ける具体的な方法

**1. セクションの重要度で余白を変える**

```css
/* ❌ AIデザインでよくある均一な余白 */
.section { padding: 80px 0; }

/* ✅ Trust Visualのリズム */
.hero { padding: 0 0 80px; }
.stats { padding: 120px 0; }          /* 最重要：大きく */
.logos { padding: 60px 0; }           /* 補助：小さく */
.testimonials { padding: 100px 0; }
.cta { padding: 100px 0 140px; }      /* 下を大きく */
```

**2. 写真とテキストの余白差**

写真と写真の間: 16-24px（写真を多く見せたい）
写真とテキストの間: 32-40px（読みやすさ確保）
テキストとテキストの間: 24-32px

**3. 数字の周囲は特別扱い**

実績数字は「絵」として扱い、周囲に十分な余白を確保:
- 数字の上: 80px
- 数字の下（単位含む）: 24px
- 数字間（横並び時）: 60px以上

## 1-5. アニメーション戦略

### どこにアニメーションを使い、どこに使わないか

**使用する場面**:
1. **実績数字のカウントアップ**: Trust Visual の最重要アニメーション
2. **スクロールによるセクション出現時**（フェードイン）
3. **取引先ロゴの横スクロール**（無限ループ、オプション）
4. **CTAボタンのホバー時**
5. **写真ギャラリーのホバー時**（軽いズーム）

**使用しない場面**:
1. **ページ遷移時**: 即座に表示。ローダーのみ許可
2. **お客様の声の顔写真**: 動くと落ち着きがなくなる
3. **テキストの1文字ずつ出現**: 読みにくくなる
4. **パララックス**: 写真がずれると「証拠」としての信頼性が下がる
5. **3D回転、複雑な変形**: 派手すぎて胡散臭くなる

### 数字のカウントアップ演出

**カウントアップは Trust Visual の象徴的アニメーション**

```tsx
// 数字カウントアップ
const countUp = {
  initial: {
    opacity: 0,
    // 数値は別途 useMotionValue で管理
  },
  whileInView: {
    opacity: 1,
  },
  transition: {
    duration: 0.3,
  },
  viewport: {
    once: true,
    margin: "-100px"
  }
};

// 数値のカウントアップは framer-motion の useMotionValue + useTransform を使用
// 0 → 目標値 を 2秒かけて easeOut で変化
```

**カウントアップの設計**:
- 開始値: 0
- 終了値: 実際の数値（523, 98.7 等）
- 所要時間: 2秒
- イージング: easeOut（最初速く、最後ゆっくり）
- トリガー: 画面の中央付近に到達した時
- 発火回数: 1回のみ（once: true）

**小数点を含む数字の場合**:
- 整数部分のみカウントアップ
- または、小数点以下は固定表示（98.7% なら 0→98 とカウントし、.7 は最初から表示）

### 過剰にならない基準

**3秒ルール**: ページを開いて3秒以内に全てのビューポート内アニメーションが完了していること

**1回ルール**: 同じ要素が2回以上アニメーションしないこと（ループを除く）

**意味ルール**: 全てのアニメーションに「なぜ動くか」の理由があること
- カウントアップ → 「積み重ね」「成長」を表現
- フェードイン → 「登場」を自然に見せる
- ホバーズーム → 「興味を持っている」ことへのフィードバック

**禁止アニメーション**:
- バウンス（跳ね返り）: 安っぽくなる
- スプリング: 過剰な動き
- パルス（点滅）: 広告感が出る
- 回転: 意味がない
- スケール変化（大→小→大）: 落ち着きがなくなる

### 具体的なアニメーションパターン（3種類）

**パターン1: フェードイン + マイクロスライド**

```tsx
// 使用場面: セクション出現時
const fadeInUp = {
  initial: { opacity: 0, y: 30 },
  whileInView: { opacity: 1, y: 0 },
  transition: { duration: 0.6, ease: "easeOut" },
  viewport: { once: true, margin: "-80px" }
};
```

- 移動距離は30px（Trust Visual は Minimal より少し大きめ）
- 0.6秒で完了
- `margin: "-80px"` で画面に入る少し前から発火準備

**パターン2: カウントアップ（数字専用）**

```tsx
// 使用場面: 実績数字
// framer-motion の animate + useMotionValue を組み合わせ
const [count, setCount] = useState(0);

useEffect(() => {
  if (inView) {
    const controls = animate(0, targetValue, {
      duration: 2,
      ease: "easeOut",
      onUpdate: (value) => setCount(Math.round(value))
    });
    return () => controls.stop();
  }
}, [inView, targetValue]);
```

- 0から目標値まで2秒
- easeOut で後半ゆっくり
- 表示は整数にまとめる（Math.round）

**パターン3: ホバーズーム（写真専用）**

```tsx
// 使用場面: 写真ギャラリー、実績写真
const photoHover = {
  whileHover: {
    scale: 1.03,
    transition: { duration: 0.3 }
  }
};
```

- スケールは1.03（3%拡大）が限界。それ以上は派手すぎる
- 写真のコンテナに `overflow: hidden` を設定し、はみ出さないように

## 設計原則まとめ

### このテンプレートの3つの柱

1. **第三者証明の全面展開**: 自社PRではなく、顧客・取引先・数字に語らせる
2. **数字のビジュアル化**: 数字を96px以上の超大型で表示し、視覚的インパクトに
3. **実写による信頼**: イラストやストックフォトを排除し、自社撮影写真のみ使用
