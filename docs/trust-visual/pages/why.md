# 選ばれる理由ページ 詳細設計

## 基本情報

**URL**: `/why`

**ファイル**: `src/app/why/page.tsx`

---

## このページの目的

### 👤 専門家①（マーケティング）視点

**マーケティング目標**:
1. TOPページで興味を持ったユーザーに、選ばれる理由を論理的に説明する
2. 各理由を具体的な数字・事実で裏付け、信頼性を高める
3. 理由を読んだ後、実績または問い合わせへの遷移を促す

**KPI指標**:
- ページ完読率: 60%以上（最後のCTAまで到達）
- 実績ページへの遷移率: 25%以上
- お問い合わせページへの遷移率: 15%以上

### 👤 専門家②（心理学）視点

**心理的ゴール**:
1. 「なぜこの会社が選ばれているのか」という疑問に答え、認知的クロージャーを与える
2. 各理由に数字を添えることで、「根拠がある」という安心感を提供
3. 理由を3つに絞ることで、記憶に残りやすくする（マジカルナンバー3）

**活用する心理メカニズム**:
- **具体性効果**: 抽象的な理由ではなく、具体的な数字で裏付け
- **認知的流暢性**: 3つに絞ることで処理しやすく、好意的に評価される
- **権威性バイアス**: ISO認証、業界資格等の公的な裏付け

### 👤 専門家③（デザイン）視点

**デザイン上の役割**:
1. 各理由を「大きな見出し + 写真 + 説明 + 根拠数字」の統一フォーマットで表現
2. 数字を目立たせ、「根拠付き」であることを視覚的に伝える
3. 写真を効果的に使い、「言葉だけではない」信頼感を演出

## ユーザーの状態

**このページに来る前の状態**:
- TOPページで「選ばれる理由を詳しく見る」をクリック
- 「なぜこの会社が選ばれているのか」に興味を持っている
- 数字やロゴで「信頼できそう」と感じているが、詳細を確認したい

**このページで感じてほしいこと**:
- 「なるほど、こういう理由で選ばれているのか」
- 「それぞれの理由に根拠となる数字がある」
- 「実際の写真もあるし、嘘ではなさそう」

**このページから離れる時の状態**:
- 実績紹介へ遷移: 「具体的な実績を見てみたい」
- お問い合わせへ遷移: 「この会社に相談してみたい」
- TOPへ戻る: 「他の情報も確認したい」

## セクション1: ページヒーロー（Page Hero）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
ページの目的を明確にし、「このページを読む価値がある」と感じさせる。TOPからの流れを引き継ぎ、期待感を高める。

**👤 専門家②（心理学）**:
ページ冒頭で「これから3つの理由を説明する」と予告することで、認知的準備を促す。構造化された情報は処理しやすく、信頼される。

**👤 専門家③（デザイン）**:
背景に薄いパターンまたは写真を敷き、「新しいページに来た」感を演出。ページタイトルを大きく表示し、一目で何のページかわかるようにする。

### レイアウト

**構成**:
- 高さ: 40vh（画面の40%）
- 背景: 薄い幾何学パターン または オフホワイト
- ページタイトル: 中央配置、56px
- リード文: タイトルの24px下、中央配置、17px

**なぜこのレイアウトか**:
- TOPページより低い高さ（40vh）にすることで、すぐにコンテンツへ入れる
- 中央配置で「このページのテーマ」を明確に伝える
- 装飾は控えめにし、本題（理由の説明）に早く入る

**代替案を採用しない理由**:
- ❌ 100vhのフルスクリーン: 本題への到達が遅れる
- ❌ 背景に大きな写真: 理由の説明が本題なので、ヒーローで主張しすぎない

### コンテンツ要素

**ページタイトル**:
- テキスト: 「選ばれる理由」
- サイズ: 56px（PC）、36px（SP）
- ウェイト: Bold（700）
- 色: メインカラー
- 配置: 中央揃え

**リード文**:
- テキスト: 「500社以上の企業様に選ばれ続ける理由。それは、35年かけて積み上げてきた「信頼」の証です。」
- サイズ: 17px（PC）、15px（SP）
- 色: サブテキストカラー
- 配置: 中央揃え、最大幅600px

### デザイン詳細

**背景**:
- 色: オフホワイト（#f8f8f8）または薄い幾何学パターン
- 理由: 白からの変化でセクションを区切りつつ、主張しすぎない

**余白**:
- 上余白: 80px（ヘッダー分考慮）
- 下余白: 60px

### アニメーション

**使用する**: タイトルのフェードイン

```tsx
const titleFade = {
  initial: { opacity: 0, y: 20 },
  animate: { opacity: 1, y: 0 },
  transition: { duration: 0.6 }
};
```

### SP（スマートフォン）での変化

**サイズ変更**:
- ページタイトル: 36px
- リード文: 15px
- 高さ: 30vh

## セクション2: 理由①「35年の経験と実績」

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
「歴史がある」ことは信頼の源泉。創業年数と累計実績数で「長く続いている＝信頼されている」ことを証明する。

**👤 専門家②（心理学）**:
日本人は「老舗」「歴史」に対する信頼が厚い。35年続いているという事実は、「多くの試練を乗り越えてきた」「お客様に支持され続けてきた」という推論を促す。

**👤 専門家③（デザイン）**:
左側に写真（創業当時〜現在の変遷、または会社の歴史を感じさせるカット）、右側に説明と数字を配置。写真で「実在する歴史」を視覚化する。

### レイアウト

**構成**:
- 背景色: 白（#FFFFFF）
- 左側50%: 写真エリア（会社の歴史を感じさせる写真）
- 右側50%: テキストエリア（理由番号 + タイトル + 説明 + 根拠数字）

**テキストエリアの内部構成**:
- 理由番号: 「01」（96px、アクセントカラー、Light）
- タイトル: 「35年の経験と実績」（32px、メインカラー、Bold）
- 説明文: 3-5行の詳細説明（17px）
- 根拠数字: 2-3個の数字をバッジ風に表示

**なぜこのレイアウトか**:
- 左右分割で写真とテキストを同時に見せ、「視覚的証拠 + 論理的説明」を両立
- 理由番号を大きく表示し、「構造化されている」ことを示す
- 根拠数字をバッジ風にすることで、「事実に基づいている」ことを強調

**代替案を採用しない理由**:
- ❌ テキストのみ: 「歴史」を言葉だけで伝えても実感がわかない
- ❌ 縦並び: 情報密度が下がり、スクロール量が増える

**写真エリア**:
- 内容: 会社の歴史を感じさせる写真

- サイズ: 左側50%幅、高さは自然な縦横比で
- 処理: 角丸なし、フルブリード（左端まで広げる）

**理由番号**:

- サイズ: 96px（PC）、64px（SP）

- 配置理由: 大きく表示することで「構造化」を示す

**タイトル**:

## セクション3: 理由②「徹底した品質管理」

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
BtoBでは「品質」が最重要の判断基準。ISO認証、検査体制、不良率等の具体的な数字で「品質に自信がある」ことを証明する。

**👤 専門家②（心理学）**:
「ISO認証」「不良率0.01%」等の公的・客観的な指標は、権威性バイアスを活用できる。第三者機関が認めた品質は、自社の主張より信頼される。

**👤 専門家③（デザイン）**:
理由①とは左右を逆にし、右側に写真、左側にテキストを配置。交互レイアウトによりリズムを作り、スクロール時の単調さを防ぐ。

### レイアウト

**構成**:
- 背景色: オフホワイト（#f8f8f8）で理由①と区別
- 左側50%: テキストエリア（理由番号 + タイトル + 説明 + 根拠数字 + 認証バッジ）
- 右側50%: 写真エリア（品質管理の様子、検査機器等）

**なぜこのレイアウトか**:
- 理由①と左右を逆にすることで、視覚的なリズムを作る
- 背景色を変えることで、新しいセクションに入ったことを示す
- 認証バッジ（ISO等）を追加し、公的な裏付けを視覚化

**代替案を採用しない理由**:
- ❌ 理由①と同じレイアウト: 単調になり、スクロール時に退屈に感じる
- ❌ 背景色を変えない: セクションの区切りがわかりにくい

### コンテンツ要素

**写真エリア**:
- 内容: 品質管理の様子を示す写真
  - 案①: 検査機器での測定風景
  - 案②: 品質管理担当者の真剣な表情
  - 案③: 検査データを確認する様子
- 配置: 右側50%幅

**理由番号**:
- テキスト: 「02」
- スタイル: 理由①と同じ

**タイトル**:
- テキスト: 「徹底した品質管理」

**説明文**:
- 内容例:
  「ISO9001認証を取得し、体系的な品質管理体制を構築しています。
  全数検査を基本とし、出荷前には熟練の検査員が最終チェックを行います。
  不良率は0.01%以下を維持しており、納品後のクレームもほぼゼロです。」

**根拠数字（バッジ）**:

| 数字 | 単位 | 説明 |
|------|------|------|
| 0.01 | %以下 | 不良率 |
| 100 | % | 全数検査 |
| 3 | 名 | 専任検査員 |

**認証バッジ**:
- 内容: ISO9001、その他業界認証のロゴ
- サイズ: 高さ48px
- 配置: 根拠数字の下、32px余白を空けて

### デザイン詳細

**背景**:
- 色: オフホワイト（#f8f8f8）

**余白**:
- 上余白: 120px
- 下余白: 120px
- 左側テキストエリアの内側余白: 上0、右60px、下0、左10%

**認証バッジのスタイル**:

## セクション4: 理由③「迅速・柔軟な対応」

「困った時に助けてくれる」という安心感は、信頼構築において重要。具体的な対応時間や実例を示すことで、「本当に対応してくれそう」と感じさせる。

**👤 専門家③（デザイン）**:
理由①と同じレイアウト（左：写真、右：テキスト）に戻し、交互のリズムを維持。写真は「スピード感」「コミュニケーション」を感じさせるカットを使用。

- 右側50%: テキストエリア

**なぜこのレイアウトか**:
- 理由①→②→③で「左右左」の交互レイアウトにより、リズムを作る
- 背景色も「白→オフホワイト→白」の交互で視覚的変化

### コンテンツ要素

**写真エリア**:

  - 案①: 電話対応中の社員（笑顔）

## セクション5: CTA（Call to Action）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
3つの理由を読み終えたユーザーは「この会社は信頼できそう」と感じている状態。このタイミングで問い合わせへ誘導する。

**👤 専門家②（心理学）**:
理由を読んで納得したユーザーは、行動を起こす準備ができている。「選ばれる理由がわかった。では、実際に問い合わせてみよう」という流れを作る。

**👤 専門家③（デザイン）**:
TOPページのCTAと同じデザインを使用し、一貫性を保つ。ただし、このページでは「実績を見る」へのリンクも併記し、選択肢を提供する。

### レイアウト

**構成**:
- 背景色: メインカラー（ダーク系）
- 見出し: 白文字、中央配置
- CTAボタン2つ: 「お問い合わせ」（アクセントカラー）+「実績を見る」（白枠）

**なぜこのレイアウトか**:
- 「お問い合わせ」が主ボタン、「実績を見る」が副ボタンとして、選択肢を提供
- まだ問い合わせる決心がついていないユーザーには、実績ページへの導線を用意

### コンテンツ要素

**見出し**:
- テキスト: 「お気軽にご相談ください」
- サイズ: 40px（PC）、28px（SP）
- 色: 白

**サブテキスト**:
- テキスト: 「ご質問・お見積もりは無料です。まずはお話をお聞かせください。」
- サイズ: 17px
- 色: 白（opacity 0.8）

**CTAボタン（主）**:
- テキスト: 「お問い合わせ」
- スタイル: アクセントカラー背景、白文字
- リンク先: `/contact`

**CTAボタン（副）**:
- テキスト: 「実績を見る」
- スタイル: 透明背景、白枠、白文字
- リンク先: `/works`
