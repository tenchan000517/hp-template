# サービスページ 詳細設計

## 基本情報

**URL**: `/service`

**ファイル**: `src/app/service/page.tsx`

---

## このページの目的

### 👤 専門家①（マーケティング）視点

**マーケティング目標**:
1. 提供サービスの全体像を明確に伝える
2. 各サービスの特徴と対応範囲を具体的に説明する
3. 「自分のニーズに合うサービスがある」と感じさせ、問い合わせへ誘導する

**KPI指標**:
- ページ完読率: 55%以上
- お問い合わせへの遷移率: 12%以上
- 平均滞在時間: 90秒以上

### 👤 専門家②（心理学）視点

**心理的ゴール**:
1. サービス内容を明確に理解させ、「何を頼めるか」を明確にする
2. 対応範囲の広さを示し、「色々なことを相談できる」と思わせる
3. 各サービスに数字や実績を添え、信頼性を高める

**活用する心理メカニズム**:
- **認知的流暢性**: 情報を整理して提示することで「わかりやすい」と評価される
- **具体性効果**: 抽象的な説明より具体的な事例の方が信頼される
- **選択のパラドックス回避**: サービスを3-5つに整理し、選びやすくする

### 👤 専門家③（デザイン）視点

**デザイン上の役割**:
1. 各サービスを視覚的に区別し、理解しやすくする
2. サービスごとに写真を配置し、「具体的に何ができるか」を視覚化
3. 一貫したフォーマットで情報を整理し、スキャンしやすくする

## ユーザーの状態

**このページに来る前の状態**:
- TOPや他のページから「サービス」をクリック
- 「具体的に何を頼めるのか」を知りたい
- 自分のニーズに合うサービスがあるか確認したい

**このページで感じてほしいこと**:
- 「こういうサービスを提供しているんだ」
- 「自分のニーズに合うサービスがある」
- 「対応範囲が広いから、色々相談できそう」

**このページから離れる時の状態**:
- お問い合わせへ遷移: 「このサービスについて相談したい」
- 実績紹介へ遷移: 「このサービスの実績を見たい」

---

## セクション構成

サービスページは以下の5セクションで構成する:

1. ページヒーロー（Page Hero）: ページタイトル + サービス概要
2. サービス一覧（Services List）: 各サービスの詳細（3-5サービス）
3. 対応範囲（Coverage）: 対応エリア、対応業種、対応規模
4. 料金・納期の目安（Pricing Guide）: 目安情報（任意）
5. CTA（Call to Action）: お問い合わせへの誘導

---

## セクション1: ページヒーロー（Page Hero）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
サービスページ全体の方向性を示し、ユーザーの期待値を設定する。「どんなサービスを提供しているのか」の概要を冒頭で伝える。

**👤 専門家②（心理学）**:
ページ冒頭で「何のページか」を明確にすることで、認知負荷を下げる。概要を先に見せることで、詳細を読む準備を整える。

**👤 専門家③（デザイン）**:
シンプルなデザインで、本題（サービス詳細）に早く入る。背景はニュートラルに、テキストを読みやすく。

### レイアウト

**構成**:
- 高さ: 35vh
- 背景: オフホワイト（#f8f8f8）
- ページタイトル: 中央配置、56px
- サービス概要: タイトル下に1-2行の説明

### コンテンツ要素

**ページタイトル**:
- テキスト: 「サービス」
- サイズ: 56px（PC）、40px（SP）
- 色: メインカラー

**サービス概要**:
- テキスト: 「お客様のニーズに合わせた多様なサービスを提供しています。まずはお気軽にご相談ください。」
- サイズ: 17px
- 色: サブテキストカラー
- 配置: タイトルの24px下、中央揃え、最大幅600px

### デザイン詳細

**余白**:
- 上余白: 80px
- 下余白: 60px

## セクション2: サービス一覧（Services List）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
各サービスを詳細に説明し、「何を頼めるか」を明確にする。特徴、対応範囲、強みを伝えることで、問い合わせの具体性を高める。

**👤 専門家②（心理学）**:
各サービスに写真を添えることで「具体的にイメージできる」ようにする。サービスを3-5つに整理することで、選択のパラドックスを回避する。

**👤 専門家③（デザイン）**:
各サービスを大きなブロックとして表現。写真とテキストを組み合わせ、視覚と論理の両面から理解を促す。交互レイアウトでリズムを作る。

### レイアウト

**構成**:
- 背景色: 白とオフホワイトを交互
- 各サービス: 左右分割（PC）
- 左側50%: サービス写真
- 右側50%: サービス名 + 説明 + 特徴リスト + 実績数字

**サービスの交互レイアウト**:
- 奇数番目: 左に写真、右にテキスト
- 偶数番目: 右に写真、左にテキスト

**なぜこのレイアウトか**:
- 写真で「具体的に何ができるか」を視覚化
- 交互レイアウトでスクロール時の単調さを防ぐ
- 十分なスペースで詳細を説明できる

### コンテンツ要素

**各サービスの構成**:

**サービス写真**:
- 内容: サービスを象徴する写真（作業風景、成果物等）
- サイズ: 50%幅、高さは自然な縦横比
- 処理: 角丸なし

**サービス名**:
- サイズ: 32px（PC）、24px（SP）
- 色: メインカラー
- ウェイト: Bold

**サービス説明**:
- 内容: 2-4行でサービスの概要を説明
- サイズ: 17px
- 色: サブテキストカラー
- 行間: 1.8

**特徴リスト**:
- 形式: チェックマーク付きリスト（3-5項目）
- 内容: サービスの特徴、強み
- サイズ: 16px

**実績数字（バッジ）**:
- 形式: Trust Visual の根拠数字バッジと同じ
- 内容: このサービスの実績（例: 「年間300件対応」「リピート率90%」）

**サービス内容例**:

| サービス名 | 説明 | 特徴 | 実績数字 |
|-----------|------|------|---------|
| 〇〇加工 | 高精度な〇〇加工を提供。短納期にも対応。 | 精度±0.01mm、24時間対応可、小ロット対応 | 年間500件 |
| △△製作 | オーダーメイドの△△を製作。設計から対応。 | 完全オーダー、図面不要OK、試作対応 | 累計2,000件 |
| □□サービス | □□の導入からメンテナンスまで一貫対応。 | 導入支援、定期点検、緊急対応 | 稼働率99% |

### デザイン詳細

**背景**:
- 奇数番目: 白（#FFFFFF）
- 偶数番目: オフホワイト（#f8f8f8）

**余白**:
- 各サービス: 上下100px
- 左右マージン: 各10%
- 写真とテキストの間: 60px

**特徴リストのスタイル**:
```css

.feature-list li {

  content: "✓";

```

## セクション3: 対応範囲（Coverage）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
対応エリア、対応業種、対応規模を明示することで「自分は対象か」を確認させる。対応範囲が広いことを示し、問い合わせのハードルを下げる。

**👤 専門家②（心理学）**:
「全国対応」「業種問わず」等の表現は、ユーザーに「自分も対象だ」と思わせる。逆に対象外を明示することで、ミスマッチを防ぐ。

**👤 専門家③（デザイン）**:
3つの対応範囲（エリア、業種、規模）をカード形式で並べ、一目で把握できるようにする。

### レイアウト

**構成**:
- 背景色: メインカラー（ダーク系）
- セクション見出し: 白文字、中央配置
- 対応範囲カード: 3つを横並び

**カードの配置**:
- 3カラム等幅
- カード間のギャップ: 32px
- 各カード: 白背景、上にアイコン風の見出し

### コンテンツ要素

**セクション見出し**:
- テキスト: 「対応範囲」
- サイズ: 32px
- 色: 白

**対応範囲カード（3つ）**:

| カード | 見出し | 内容 |
|--------|--------|------|
| エリア | 対応エリア | 全国対応（関東・東海・関西は特に迅速対応） |
| 業種 | 対応業種 | 製造業、建設業、食品業 他（業種問わずご相談ください） |
| 規模 | 対応規模 | 小ロット1個〜大量生産まで（試作のみも可） |

**各カードの構成**:
- 見出し: 20px、メインカラー、Bold
- 内容: 16px、サブテキストカラー
- パディング: 32px

### デザイン詳細

**背景**:
- セクション: メインカラー
- カード: 白（#FFFFFF）

**余白**:
- 上余白: 80px
- 下余白: 80px
- 左右マージン: 各15%

**カードのスタイル**:
```css
.coverage-card {
  background-color: #FFFFFF;
  padding: 32px;
  text-align: center;
}

.coverage-card h4 {
  font-size: 20px;
  font-weight: 700;
  color: var(--main-color);
  margin-bottom: 16px;
}

.coverage-card p {
  font-size: 16px;
  color: var(--sub-text-color);
  line-height: 1.6;
}
```

**レイアウト変更**:

## セクション4: 料金・納期の目安（Pricing Guide）※任意

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
料金や納期の目安を示すことで「問い合わせ前に予算感を確認したい」ユーザーのニーズに応える。ただし、詳細は個別見積もりであることを明記。

**👤 専門家②（心理学）**:
料金情報があると「隠し事がない」という印象を与える。ただし、安すぎる印象を与えないよう、「最低価格」ではなく「目安」として提示する。

**👤 専門家③（デザイン）**:
シンプルなテーブル形式で料金・納期を提示。「目安」であることを注釈で明記し、誤解を防ぐ。

### レイアウト

**構成**:
- 背景色: オフホワイト（#f8f8f8）
- セクション見出し: 「料金・納期の目安」
- 料金テーブル: サービス名、料金目安、納期目安
- 注釈: 「詳細はお見積もりにてご案内します」

### コンテンツ要素

**料金テーブル例**:

| サービス | 料金目安 | 納期目安 |
|----------|---------|---------|
| 〇〇加工 | ¥5,000〜 | 3営業日〜 |
| △△製作 | ¥30,000〜 | 2週間〜 |
| □□サービス | 月額¥50,000〜 | 即日対応可 |

**注釈**:
- テキスト: 「※ 上記は目安です。仕様、数量、納期等によって変動します。詳細はお見積もりにてご案内いたします。」
- サイズ: 14px
- 色: サブテキストカラー

### デザイン詳細

**余白**:
- 上余白: 80px
- 下余白: 80px
