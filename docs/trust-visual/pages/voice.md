# お客様の声ページ 詳細設計

## 基本情報

**URL**: `/voice`

**ファイル**: `src/app/voice/page.tsx`

## このページの目的

### 👤 専門家①（マーケティング）視点

**マーケティング目標**:
1. 実際のお客様の声で「この会社は信頼できる」という確信を与える
2. 顔写真・実名・役職付きで「本物の推薦」であることを証明する
3. 複数の声を見せることで「多くの人が満足している」ことを伝える

**KPI指標**:
- ページ完読率: 70%以上（複数の声を読んでもらう）
- お問い合わせへの遷移率: 15%以上（最も高い遷移率を目指す）
- 平均閲覧声数: 4件以上

### 👤 専門家②（心理学）視点

**心理的ゴール**:
1. 社会的証明の最強形態「顔写真付き実名推薦」で信頼を確定させる
2. 複数の業種・立場からの声で「自分も同じように満足できる」と思わせる
3. 具体的なエピソードで「リアルな体験」を伝える

**活用する心理メカニズム**:
- **社会的証明**: 複数の人が推薦 → みんなが認めている
- **権威性バイアス**: 代表取締役、部長等の肩書き → 判断力のある人が推薦
- **ストーリーテリング**: 課題→解決の流れで共感を得る
- **自己関連付け効果**: 同業種・同規模の声を見て「自分と同じ」と感じる

### 👤 専門家③（デザイン）視点

**デザイン上の役割**:
1. 顔写真を大きく表示し、「実在する人物」であることを強調
2. 引用符や背景色で「お客様の言葉」であることを明示
3. 各インタビューを1つの「ストーリー」として見せる

---

## ユーザーの状態

**このページに来る前の状態**:
- TOPや実績ページから「お客様の声」をクリック
- 「実際に使っている人はどう評価しているのか」を知りたい
- 最終的な判断材料を探している

**このページで感じてほしいこと**:
- 「この人も、あの人も満足しているんだ」
- 「自分と同じ業種の人も推薦している」
- 「具体的なエピソードがあるから、嘘ではなさそう」
- 「顔写真と実名があるから信頼できる」

**このページから離れる時の状態**:
- お問い合わせへ遷移: 「この会社に決めた。問い合わせよう」
- 会社概要へ遷移: 「どんな会社なのか最終確認したい」

---

## セクション構成

お客様の声ページは以下の4セクションで構成する:

1. ページヒーロー（Page Hero）: ページタイトル + 満足度数字
2. お客様の声一覧（Testimonials List）: 詳細インタビュー形式（5-8件）
3. 満足度まとめ（Satisfaction Summary）: 統計データのビジュアル化
4. CTA（Call to Action）: お問い合わせへの誘導

## セクション1: ページヒーロー（Page Hero）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
「お客様満足度98%」等の数字を冒頭で見せ、「満足度が高い」という事実を印象づける。これから見る声への期待感を高める。

**👤 専門家②（心理学）**:
数字を先に見せることで「98%が満足しているなら間違いない」というフレーミング効果を狙う。その後の個別の声が、この数字を裏付ける証拠として機能する。

**👤 専門家③（デザイン）**:
満足度数字を大きく表示し、Trust Visual の「Visual Impact」を体現。背景はシンプルにし、本題（お客様の声）に早く入る。

### レイアウト

**構成**:
- 高さ: 40vh
- 背景: オフホワイト（#f8f8f8）
- ページタイトル: 中央配置、56px
- 満足度数字: タイトル下に大きく表示

**なぜこのレイアウトか**:
- シンプルな構成で本題（お客様の声）に早く入る
- 満足度数字を大きく見せることで「高評価」を印象づける

### コンテンツ要素

**ページタイトル**:
- テキスト: 「お客様の声」
- サイズ: 56px（PC）、40px（SP）
- 色: メインカラー

**満足度数字**:
- 数字: 98.2%（例）
- サイズ: 96px、Black（900）、アクセントカラー
- 説明: 「お客様満足度」（20px、サブテキストカラー）
- 補足: 「2024年 お客様アンケート結果より」（14px）

### デザイン詳細

## セクション2: お客様の声一覧（Testimonials List）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
詳細なインタビュー形式で「リアルな声」を伝える。課題→導入→効果の流れで、ユーザーが自分と重ね合わせやすいストーリーを提供する。

**👤 専門家②（心理学）**:
ストーリー形式は記憶に残りやすく、感情的な共感を得やすい。「この人と同じ課題を抱えている」と感じたユーザーは、強く引き込まれる。

**👤 専門家③（デザイン）**:
各インタビューを大きなブロックとして表現。顔写真を大きく配置し、「実在する人物」感を最大化する。背景色を交互に変えてリズムを作る。

### レイアウト

**構成**:
- 背景色: 白とオフホワイトを交互
- 各インタビュー: 左右分割（PC）または縦並び（SP）
- 左側35%: 顔写真 + 会社情報
- 右側65%: インタビュー内容（課題、導入理由、効果）

**インタビューの交互レイアウト**:
- 奇数番目: 左に顔写真、右にテキスト
- 偶数番目: 右に顔写真、左にテキスト（左右反転）

**なぜこのレイアウトか**:
- 顔写真を大きく見せることで「実在する人物」感を強調
- 左右交互にすることで視覚的なリズムを作る
- 詳細なインタビュー形式で「深い信頼」を構築

**代替案を採用しない理由**:
- ❌ カード形式（小さい写真）: 顔写真のインパクトが弱まる
- ❌ 全て同じレイアウト: 単調になる
- ❌ テキストのみ: 信頼性が大幅に下がる

### コンテンツ要素

**各インタビューの構成**:

**左側（顔写真エリア）**:
- 顔写真: 縦長（3:4比率）、最低300px幅
- 会社名: 14px、サブテキストカラー
- 役職・氏名: 24px、メインカラー、Bold
- 導入時期: 12px、サブテキストカラー（例: 2020年導入）

**右側（インタビューエリア）**:

| セクション | 見出し例 | 内容 |
|-----------|---------|------|
| 課題 | 導入前の課題 | 「〇〇に困っていました」（2-3行） |
| 導入理由 | 選んだ理由 | 「〇〇が決め手でした」（2-3行） |
| 効果 | 導入後の変化 | 「〇〇が改善しました」（2-3行） |
| 一言 | メッセージ | 引用符付きの短いコメント |

**インタビュー内容例**:

```
【インタビュー1】
会社名: 株式会社〇〇製作所
役職・氏名: 代表取締役 山田太郎 様
導入時期: 2020年

【導入前の課題】
以前の取引先では納期の遅れが頻繁にあり、生産計画に支障をきたしていました。
品質にもばらつきがあり、検品に多くの時間を取られていました。

【選んだ理由】
知人からの紹介で〇〇社を知りました。見積もり対応の速さと、
品質管理体制の説明を聞いて、信頼できると感じました。

【導入後の変化】
納期通りの納品が当たり前になり、生産計画が立てやすくなりました。
不良率が大幅に下がり、検品時間も半減しました。

【一言】
「安心して任せられるパートナーです。もっと早く出会いたかったです。」
```

## セクション3: 満足度まとめ（Satisfaction Summary）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
個別の声を読んだ後、統計データで「全体的な満足度」を示す。「個人の感想」ではなく「多くの人の評価」であることを強調。

**👤 専門家②（心理学）**:
個別のストーリー（感情的）と統計データ（論理的）の両方を見せることで、感情と理性の両面から納得させる。

**👤 専門家③（デザイン）**:
数字を大きく、棒グラフやパーセンテージバーでビジュアル化。一目で「高評価」とわかるデザインに。

### レイアウト

**構成**:
- 背景色: メインカラー（ダーク系）
- セクション見出し: 白文字、中央配置
- 満足度項目: 3-4項目を横並び（PC）
- 各項目: パーセンテージ + 項目名 + バー

**なぜこのレイアウトか**:
- ダーク背景で「まとめ」「締め」の印象を与える
- 複数の項目を並べることで「総合的に高評価」を示す
- バーチャートで視覚的にわかりやすく

### コンテンツ要素

**セクション見出し**:
- テキスト: 「お客様満足度」
- サイズ: 36px、白
- 配置: 中央揃え

**満足度項目（4つ）**:

| 項目 | パーセンテージ |
|------|--------------|
| 品質満足度 | 98% |
| 納期満足度 | 96% |
| 対応満足度 | 99% |
| 再依頼意向 | 95% |

**各項目の構成**:
- パーセンテージ: 48px、白、Bold
- 項目名: 16px、白（opacity 0.8）
- バー: 幅100%、高さ8px、達成部分はアクセントカラー

### デザイン詳細

**背景**:
- 色: メインカラー（#1e3a2f または #1a2744）

**余白**:
- 上余白: 100px
- 下余白: 100px
- 左右マージン: 各15%

**満足度バーのスタイル**:
```css
.satisfaction-bar {
  width: 100%;
  height: 8px;
  background-color: rgba(255, 255, 255, 0.2);
  border-radius: 4px;
  overflow: hidden;
  margin-top: 12px;
}

.satisfaction-bar-fill {
  height: 100%;
  background-color: var(--accent-color);
  border-radius: 4px;
  transition: width 1s ease-out;
}
```

### アニメーション

**使用する**: バーのフィル + 数字のカウントアップ

```tsx
// バーのフィルアニメーション
const barFill = {
  initial: { width: 0 },
  whileInView: { width: `${percentage}%` },
  transition: { duration: 1, ease: "easeOut" },
  viewport: { once: true }
};
```

### SP（スマートフォン）での変化

**レイアウト変更**:
- 横並び → 縦並び（2×2グリッド）

**サイズ変更**:

## セクション4: CTA（Call to Action）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
お客様の声を全て読んだユーザーは「この会社は信頼できる」と確信している状態。このタイミングで問い合わせへ誘導することで、最も高いコンバージョン率を期待できる。

**👤 専門家②（心理学）**:
多くの推薦を見た後は「自分も同じように満足できるだろう」という期待が高まっている。このポジティブな感情を行動に繋げる。

**👤 専門家③（デザイン）**:
満足度まとめセクション（ダーク）からの流れで、CTAも同系色で統一。または白背景に戻して対比を作る。

### レイアウト

**構成**:
- 背景色: 白（満足度まとめセクションとの対比）
- 見出し + サブテキスト + CTAボタン

### コンテンツ要素

**見出し**:
- テキスト: 「次は、あなたの番です。」
- サイズ: 40px（PC）、28px（SP）
- 色: メインカラー

**サブテキスト**:
- テキスト: 「お客様の期待に応えること。それが私たちの使命です。まずはお気軽にご相談ください。」
- サイズ: 17px
- 色: サブテキストカラー

**CTAボタン**:
- テキスト: 「お問い合わせ」
- スタイル: アクセントカラー背景、白文字
- リンク先: `/contact`

### デザイン詳細

**背景**:
- 色: 白（#FFFFFF）

**余白**:
- 上余白: 120px
- 下余白: 140px
