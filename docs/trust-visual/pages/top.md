# TOPページ 詳細設計

## 基本情報

**URL**: `/`（ルート）

**ファイル**: `src/app/page.tsx`

---

## このページの目的

### 👤 専門家①（マーケティング）視点

**マーケティング目標**:
1. 初めてこの会社を知ったユーザーに「信頼できそう」と感じさせる（5秒以内）
2. 実績数字と取引先ロゴで「多くの企業が選んでいる」ことを伝える
3. お客様の声で「実際に満足している人がいる」ことを証明する
4. 問い合わせまたは詳細ページへの遷移を促す

**KPI指標**:
- ファーストビュー離脱率: 25%以下（数字で引き留める）
- お問い合わせページへの到達率: 10%以上
- 平均滞在時間: 90秒以上（複数の証拠を見てもらう）

### 👤 専門家②（心理学）視点

**心理的ゴール**:
1. 「知らない会社だけど、これだけの実績があるなら大丈夫そう」という安心感
2. 「有名企業も使っているなら間違いない」という信頼の借用
3. 「実際に満足している人がいる」という社会的証明の確認

**活用する心理メカニズム**:
- **社会的証明**: 取引先ロゴ、実績数字、お客様の声
- **権威性バイアス**: 有名企業のロゴ、資格バッジ
- **具体性効果**: 抽象的な表現を避け、全て数字で表現
- **ハロー効果**: 有名企業との取引実績が全体の信頼度を上げる

### 👤 専門家③（デザイン）視点

**デザイン上の役割**:
1. 「Visual Impact」を最も強く表現するページ
2. 数字を96px以上の超大型で表示し、視覚的なインパクトを与える
3. 写真を大きく使い、「実在する会社」であることを証明する
4. 情報密度は高いが、呼吸のある構成にする

## ユーザーの状態

**このページに来る前の状態**:
- Google検索、広告、紹介リンクからの流入
- この会社のことを知らない（または名前だけ聞いたことがある）
- 「本当に信頼できる会社なのか？」という疑問を持っている
- 複数社を比較検討中、または情報収集段階

**このページで感じてほしいこと**:
- 「500社以上の取引実績があるのか。ちゃんとした会社だ」
- 「大手企業も取引しているなら安心だな」
- 「お客様満足度98%か。評判がいいんだな」
- 「実際に使っている人も満足しているようだ」

**このページから離れる時の状態**:
- 選ばれる理由へ遷移: 「なぜこの会社が選ばれているのか知りたい」
- 実績紹介へ遷移: 「どんな仕事をしているのか見てみたい」
- お客様の声へ遷移: 「もっと詳しく評判を確認したい」
- お問い合わせへ遷移: 「とりあえず話を聞いてみたい」

## セクション構成

TOPページは以下の6セクションで構成する:

1. ヒーロー（Hero）: メインビジュアル + キャッチコピー + 実績数字
2. 取引先ロゴ（Client Logos）: 取引先企業のロゴ一覧
3. 選ばれる理由サマリー（Why Us Summary）: 3つの理由を簡潔に
4. 実績ギャラリー（Works Gallery）: 代表的な実績を写真で
5. お客様の声サマリー（Testimonials Summary）: 3件の声を抜粋
6. CTA（Call to Action）: お問い合わせへの誘導

## セクション1: ヒーロー（Hero）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
ファーストビューで「この会社は実績がある」と認識させる。知名度のない会社が最初にすべきことは「自己紹介」ではなく「実績証明」。数字を見せることで、会社の規模感と信頼性を即座に伝える。

**👤 専門家②（心理学）**:
人間は数字を見ると「客観的な事実」と認識する。「多くの企業に選ばれています」より「523社に選ばれています」の方が信頼される。ファーストビューに数字を配置することで、最初の3秒で信頼の土台を築く。

**👤 専門家③（デザイン）**:
数字を96px以上の超大型で表示し、視覚的な主役にする。メインビジュアル写真と組み合わせ、「実在する会社」「活動している会社」であることを視覚的に証明する。

### レイアウト

**構成**:
- 画面高さ100vh（ビューポート全体を使用）
- 左側55%: メインビジュアル写真（社員の作業風景、または代表的な成果物）
- 右側45%: キャッチコピー + サブコピー + 実績数字4つ

**メインビジュアル写真のエリア**:
- 画面左端から55%幅を占める
- 高さは100vh全体
- 写真は `object-fit: cover` で全面表示
- 写真の右端はやや暗くグラデーション処理（テキストとの境界をなじませる）

**右側テキストエリア**:
- 上部35%: キャッチコピー + サブコピー
- 下部65%: 実績数字4つ（2行×2列のグリッド）

**なぜこのレイアウトか**:
- 写真とテキストを同時に見せることで、「実在する会社」と「実績」を一度に伝える
- 左側に写真を配置するのは、人間の視線が左上から始まるため。まず「人」を見て親しみを感じてから、右の数字を見る
- 2×2グリッドの数字配置は、一覧性が高く、スキャンしやすい

**代替案を採用しない理由**:
- ❌ 全画面写真 + テキストオーバーレイ: 写真が暗くなるか、テキストが読みにくくなる
- ❌ テキストのみ（写真なし）: 「実在する会社」感が薄れる
- ❌ 数字を縦に4つ並べる: 視線の移動距離が長くなり、一覧性が下がる

### コンテンツ要素

**メインビジュアル写真**:
- 内容: 社員が作業している風景、または代表的な成果物
- 要件: 明るく、活気があり、プロフェッショナルな印象
- サイズ: 最低1920×1080px、2560×1440px推奨
- 配置理由: 「実際に仕事をしている」ことを視覚的に証明

**キャッチコピー**:
- 配置: 右側エリアの上部、左揃え
- サイズ: 48px（PC）、32px（SP）
- ウェイト: Bold（700）
- 色: ダークカラー（メインカラー）
- 行間: 1.4
- 例文: 「確かな技術と実績で、お客様の期待に応える。」
- **要件**: 句点で終わる。「私たちは」で始めない。信頼・実績を連想させる言葉を含める

**サブコピー**:
- 配置: キャッチコピーの24px下
- サイズ: 17px（PC）、15px（SP）
- ウェイト: Regular（400）
- 色: サブテキストカラー（ミディアムグレー）
- 例文: 「創業35年。お客様満足度98%の〇〇サービスを提供しています。」
- **要件**: キャッチコピーを補足する具体的な内容。数字を1つ含める

**実績数字（4つ）**:
- 配置: 右側エリアの下部、2行×2列のグリッド
- 各数字の構成:
  - 数字部分: 96px、Black（900）、メインカラー
  - 単位部分: 24px、SemiBold（600）、アクセントカラー
  - 説明部分: 14px、Regular（400）、サブテキストカラー

| 位置 | 数字例 | 単位 | 説明 |
|------|--------|------|------|
| 左上 | 523 | 社 | 取引実績 |
| 右上 | 35 | 年 | 創業からの歴史 |
| 左下 | 98.2 | % | お客様満足度 |
| 右下 | 12,000 | 件 | 累計納品数 |

**数字選定の優先順位**:
1. 取引社数（最も重要。社会的証明の核心）
2. 創業年数 or 業界経験年数（歴史＝信頼）
3. 顧客満足度 or リピート率（品質の証明）
4. 累計実績数（規模感）

### デザイン詳細

**背景**:
- 左側55%: メインビジュアル写真
- 右側45%: 薄いグレー（#f8f8f8）または白（#FFFFFF）
- 写真と背景の境界: 写真の右端20%に黒→透明のグラデーションオーバーレイ（opacity 0.3程度）

**余白**:
- 上余白: ヘッダー分（80px程度）を考慮
- 下余白: 0（100vh設計）
- 右側テキストエリアの内側余白: 上80px、右60px、下80px、左40px

**数字グリッドの詳細**:
- グリッドのギャップ: 横60px、縦48px
- 各数字ブロック内の構成:
  - 数字+単位: 同じ行に配置、単位は数字のベースラインに揃える
  - 説明: 数字の12px下

**特徴的な表現**:
- 数字96pxの超大型表示が視覚的主役
- 写真とテキストの組み合わせで「実在感」と「信頼感」を同時に演出
- 数字は4つに絞り、一覧性を確保

### アニメーション

**使用する**: 数字のカウントアップ、テキストのフェードイン

**数字カウントアップ**:
```tsx
// 各数字に対して
const countUp = {
  initial: { opacity: 0 },
  animate: { opacity: 1 },
  transition: { duration: 0.3 }
};

// 数値のカウントアップは useMotionValue + animate を使用
// 0 → 目標値 を 2秒で easeOut
// 開始タイミング: ページロード後 0.5秒
```

**テキストフェードイン**:
```tsx
// キャッチコピー・サブコピー
const textFade = {
  initial: { opacity: 0, y: 20 },
  animate: { opacity: 1, y: 0 },
  transition: { duration: 0.6, delay: 0.2 }
};
```

**写真**:
- アニメーションなし（即座に表示）
- 理由: 写真は「証拠」として即座に見えるべき

### SP（スマートフォン）での変化

**レイアウト変更**:
- 左右分割を解除
- 上部40%: メインビジュアル写真（高さ40vh）
- 下部60%: テキストエリア（キャッチコピー + サブコピー + 数字4つ）

**数字グリッドの変更**:
- 2行×2列を維持（ただし、サイズを縮小）
- 数字: 56px
- 単位: 18px
- 説明: 12px

**サイズ変更**:
- キャッチコピー: 32px
- サブコピー: 15px

**余白変更**:
- テキストエリアの内側余白: 上40px、左右5%、下40px

**スクロール誘導**:
- SPでは下に「Scroll」テキストと矢印を追加
- 下にコンテンツがあることを示す

### このセクションでの心理効果

**ユーザーが感じること**:
- 「523社も取引実績があるのか。ちゃんとした会社だ」
- 「35年も続いているなら安心だな」
- 「満足度98%なら品質は問題なさそう」
- 「写真を見ると、実際に仕事している様子がわかる」

**次のセクションへの期待**:
- 「どんな会社が取引しているんだろう」
- 「もう少し詳しく知りたい」

## セクション2: 取引先ロゴ（Client Logos）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
取引先のロゴを見せることで「この会社は信頼できる」という印象を強化する。特に有名企業のロゴは、その企業の信頼を「借用」できる。ファーストビューの数字を、具体的な企業名で裏付ける。

**👤 専門家②（心理学）**:
ハロー効果を活用する。有名企業のロゴを見ると「あの会社が取引しているなら、この会社も一流だろう」という推論が働く。ロゴは文字よりも認知が速いため、スクロール中でも即座に情報が伝わる。

**👤 専門家③（デザイン）**:
ロゴをグレースケール化することで統一感を出し、自社サイトのブランドカラーを邪魔しない。ロゴの高さを揃え、等間隔で配置することで、整然とした「きちんとした会社」の印象を与える。

### レイアウト

**構成**:
- 背景色: オフホワイト（#f8f8f8）でセクション区切り
- セクション見出し: 「お取引先企業」（中央配置、28px）
- ロゴ一覧: 8-12社のロゴを横一列に配置（PCで8社表示が理想）
- ロゴ下の補足テキスト: 「他、500社以上の企業様とお取引しています」

**ロゴの配置**:
- 各ロゴの高さ: 統一（48px推奨）
- 幅: ロゴの縦横比を維持
- ロゴ間のギャップ: 48px
- 横スクロール: 画面幅を超える場合は無限ループスクロール（オプション）

**なぜこのレイアウトか**:
- 横一列にすることで、一目で「複数の企業が取引している」ことがわかる
- グレースケール化で、ロゴ同士の色の差異を消し、統一感を出す
- 「他、500社以上」という補足で、表示しているのは一部であることを示す

**代替案を採用しない理由**:
- ❌ ロゴをカラーで表示: 色がバラバラになり、統一感が失われる
- ❌ ロゴを縦に並べる: 一覧性が下がり、インパクトが弱まる
- ❌ ロゴを2行にする: 情報量が多くなりすぎる

### コンテンツ要素

**セクション見出し**:
- テキスト: 「お取引先企業」または「CLIENTS」
- サイズ: 28px（PC）、22px（SP）
- 色: サブテキストカラー
- 配置: 中央揃え

**取引先ロゴ（8-12社）**:
- 内容: 取引先企業のロゴ画像
- 要件: 知名度の高い順に左から配置
- 画像形式: SVGまたは高解像度PNG（透過）
- カラー処理: グレースケール化（CSS filter: grayscale(100%)）
- ホバー: グレースケール解除でカラーに戻る（オプション）

**配置順の優先度**:
1. 誰もが知っている大企業（トヨタ、ソニー等）
2. 業界で有名な企業
3. 上場企業
4. 地元で有名な企業
5. その他の取引先

**補足テキスト**:
- テキスト: 「他、500社以上の企業様とお取引しています」
- サイズ: 14px
- 色: ライトグレー
- 配置: ロゴ一覧の24px下、中央揃え

### デザイン詳細

**背景**:
- 色: オフホワイト（#f8f8f8）
- 理由: ヒーローの白系から軽く変化させ、セクションの区切りを示す

**余白**:
- 上余白: 60px
- 下余白: 60px
- 左右マージン: 各10%（コンテンツ幅80%）

**ロゴのスタイル**:
```css
.client-logo {
  height: 48px;
  width: auto;
  filter: grayscale(100%);
  opacity: 0.7;
  transition: all 0.3s ease;
}

.client-logo:hover {
  filter: grayscale(0%);
  opacity: 1;
}
```

**特徴的な表現**:
- グレースケール化によるロゴの統一感
- ホバーでカラーに戻ることで「ちゃんと実在する企業」感を出す
- 余白は控えめにし、情報密度を上げる（ロゴは「証拠」なので多く見せたい）

### アニメーション

**使用する**: ロゴの横スクロール（オプション）

**横スクロールアニメーション**:
```tsx
// ロゴが画面幅を超える場合のみ使用
const infiniteScroll = {
  animate: { x: [0, -totalWidth] },
  transition: {
    duration: 30, // ゆっくり
    repeat: Infinity,
    ease: "linear"
  }
};
```

- ロゴが8社以上で画面幅を超える場合のみ適用
- 非常にゆっくり（30秒で1周）動かす
- ホバーで一時停止

**使用しない場合**:
- ロゴが画面内に収まる場合は静止表示
- 無理に動かす必要はない

### SP（スマートフォン）での変化

**レイアウト変更**:
- ロゴを2行表示（1行4社×2行）
- または横スクロール可能なエリアとして表示

**サイズ変更**:
- ロゴ高さ: 36px
- セクション見出し: 22px
- 補足テキスト: 12px

**余白変更**:
- 上余白: 48px
- 下余白: 48px

### このセクションでの心理効果

**ユーザーが感じること**:
- 「あ、〇〇社も取引しているんだ」
- 「大手企業が使っているなら信頼できそう」
- 「500社以上ということは、実績は間違いないな」

**次のセクションへの期待**:
- 「具体的にどんな仕事をしているんだろう」
- 「なぜこれだけ選ばれているのか知りたい」

## セクション3: 選ばれる理由サマリー（Why Us Summary）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
数字とロゴで「信頼できそう」と感じたユーザーに、「なぜ選ばれているのか」を論理的に説明する。ここで納得感を与えることで、詳細ページへの誘導率を上げる。

**👤 専門家②（心理学）**:
人間は「なぜ」を知りたがる。数字だけでは「すごいけど、なぜ？」という疑問が残る。3つの理由を簡潔に示すことで、認知的クロージャー（疑問が解消される満足感）を与える。

**👤 専門家③（デザイン）**:
3つの理由を横並びにし、一目で全体像を把握できるようにする。各理由に紐づく数字を小さく添えることで、「選ばれる理由」が抽象論ではなく事実に基づいていることを示す。

### レイアウト

**構成**:
- 背景色: 白（#FFFFFF）
- セクション見出し: 「選ばれる3つの理由」（中央配置、40px）
- 理由カード3つを横並び（PC）、縦並び（SP）
- 各カードに: アイコン風数字 + タイトル + 説明 + 根拠数字
- 「詳しく見る」リンク: セクション下部に中央配置

**理由カードの配置**:
- 3カラム等幅（各カード間のギャップ: 40px）
- 各カードは背景なし（ボーダーや影なし）
- 左右マージン: 各10%（コンテンツ幅80%）

**なぜこのレイアウトか**:
- 3つに絞ることで、記憶に残りやすい（マジカルナンバー3）
- 横並びにすることで、全体像を一目で把握できる
- カードにしないことで、「情報を囲い込んでいる」感を避け、オープンな印象に

**代替案を採用しない理由**:
- ❌ 5つ以上の理由: 多すぎて記憶に残らない
- ❌ カード型（影付き）: AIデザイン的になり、差別化できない
- ❌ アイコン使用: 抽象的になり、Trust Visual の「具体性」に反する

### コンテンツ要素

**セクション見出し**:
- テキスト: 「選ばれる3つの理由」
- サイズ: 40px（PC）、28px（SP）
- 色: メインカラー
- 配置: 中央揃え

**理由カード（3つ）**:

| No | タイトル例 | 説明の方向性 | 根拠数字例 |
|----|-----------|-------------|-----------|
| 01 | 「35年の経験と実績」 | 創業以来培ってきた技術力と知見 | 累計12,000件以上の納品 |
| 02 | 「徹底した品質管理」 | ISO認証に基づく品質管理体制 | 不良率0.01%以下 |
| 03 | 「迅速・柔軟な対応」 | 短納期対応と細かなカスタマイズ | 平均対応時間24時間以内 |

**各カードの内部構成**:
- 番号: 56px、アクセントカラー、Light（300）
