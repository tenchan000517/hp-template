## このページの目的

### 👤 専門家①（マーケティング）視点

**マーケティング目標**:
1. 実際の納品物・施工事例を写真で見せ、「何ができる会社か」を視覚的に伝える
2. 多様な実績を見せることで、「自分の案件も対応できそう」と感じさせる
3. 実績を見た後、お問い合わせへの遷移を促す

**KPI指標**:
- ページ平均滞在時間: 120秒以上（多くの実績を見てもらう）
- 写真クリック率: 40%以上（詳細を見たいと思わせる）
- お問い合わせへの遷移率: 12%以上

### 👤 専門家②（心理学）視点

**心理的ゴール**:
1. 写真という「証拠」で「本当に仕事をしている」ことを証明する
2. 多様な業種・規模の実績を見せ、「自分の案件も対応できる」と思わせる
3. クオリティの高い写真で「仕事の質が高い」という印象を与える

**活用する心理メカニズム**:
- **具体性効果**: 言葉より写真の方が具体的で信頼される
- **同調行動**: 「色々な業種が依頼している」→「自分も依頼していいんだ」
- **ハロー効果**: 高品質な写真 → 仕事の質も高いという推論

### 👤 専門家③（デザイン）視点

**デザイン上の役割**:
1. 写真を大きく、多く見せる「ギャラリー」形式
2. 写真の品質がダイレクトにサイトの品質を表すため、高解像度を必須とする
3. カテゴリフィルターで「自分に関連する実績」を探しやすくする

## セクション構成

実績紹介ページは以下の4セクションで構成する:

1. ページヒーロー（Page Hero）: ページタイトル + 実績サマリー数字
2. カテゴリフィルター（Category Filter）: 業種・種類での絞り込み
3. 実績ギャラリー（Works Gallery）: 写真グリッド（12-24枚）
4. CTA（Call to Action）: お問い合わせへの誘導

## セクション1: ページヒーロー（Page Hero）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
ページ冒頭で「累計12,000件以上の実績」等の数字を見せ、「たくさんの実績がある」ことを強調。これから見せるギャラリーへの期待感を高める。

**👤 専門家②（心理学）**:
数字を見せることで「これから見る実績は、12,000件のうちの一部にすぎない」という認識を持たせる。一部を見せることで「まだまだある」という印象を与える。

**👤 専門家③（デザイン）**:
数字を大きく表示し、Trust Visual の「Visual Impact」を体現。背景に実績写真のコラージュを薄く敷くことで「ギャラリーページ」であることを示唆。

### レイアウト

**構成**:
- 高さ: 50vh
- 背景: 実績写真のコラージュ（4-6枚をタイル状に配置、薄い黒オーバーレイ）
- ページタイトル: 白文字、中央配置、56px
- 実績サマリー: 数字3つを横並び（累計件数、対応業種数、年間納品数）

**なぜこのレイアウトか**:
- 背景に実績写真を使うことで「このページは写真が主役」と示唆
- 数字を見せることで「量」の多さを印象づける
- 50vhと高めにすることで、ギャラリー前の「溜め」を作る

**代替案を採用しない理由**:
- ❌ 白背景: ギャラリーページ感が薄れる
- ❌ 数字なし: 量の多さが伝わらない

### コンテンツ要素

**背景画像コラージュ**:
- 内容: 代表的な実績写真4-6枚をタイル状に配置
- 処理: 黒オーバーレイ（opacity 0.6）をかけ、テキストを読みやすく
- サイズ: 各タイルは均等に配置

**ページタイトル**:
- テキスト: 「実績紹介」
- サイズ: 56px（PC）、40px（SP）
- 色: 白（#FFFFFF）
- 配置: 中央揃え

**実績サマリー数字（3つ）**:

| 数字 | 単位 | 説明 |
|------|------|------|
| 12,000 | 件+ | 累計納品実績 |
| 50 | 業種+ | 対応業種数 |
| 800 | 件/年 | 年間納品数 |

**数字の配置**:
- タイトルの48px下
- 3つを横並び（等間隔）
- 各数字: 48px、白、Bold
- 単位: 18px、白
- 説明: 14px、白（opacity 0.8）

### デザイン詳細

**背景**:
- 構成: 写真コラージュ + 黒オーバーレイ（opacity 0.6）
- 写真: 実績の中から選んだ代表的な4-6枚

**余白**:
- 上余白: 80px（ヘッダー分）
- 下余白: 60px

### アニメーション

**使用する**: タイトルと数字のフェードイン

```tsx
const heroFade = {
  initial: { opacity: 0, y: 20 },
  animate: { opacity: 1, y: 0 },
  transition: { duration: 0.6 }
};
```

### SP（スマートフォン）での変化

**レイアウト変更**:
- 数字3つを縦並びに

**サイズ変更**:
- タイトル: 40px
- 数字: 40px
- 高さ: 40vh

## セクション2: カテゴリフィルター（Category Filter）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
ユーザーが「自分に関連する実績」を見つけやすくすることで、滞在時間を延ばし、関連性を高める。「自分の業種の実績がある」と確認できれば、問い合わせ率が上がる。

**👤 専門家②（心理学）**:
人は自分に関連する情報に注目する（カクテルパーティー効果）。業種フィルターで「自分の業種」を選べることで、「自分向けの情報がある」と感じさせる。

**👤 専門家③（デザイン）**:
フィルターボタンは控えめに配置し、ギャラリー写真を邪魔しない。選択中のカテゴリは色を変えて明示する。

**構成**:

- ボタン間のギャップ: 12px

- 横並びにすることで、全カテゴリを一目で把握できる
- 中央揃えで、左右のバランスを取る
- ボタンサイズは小さめにし、ギャラリーを邪魔しない

## セクション3: 実績ギャラリー（Works Gallery）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
写真を大量に見せることで「これだけの実績がある」ことを視覚的に証明。各写真をクリックすると詳細が見られることで、関心の高いユーザーを深くエンゲージする。

**👤 専門家②（心理学）**:
写真は言語を超えた情報伝達手段。多くの写真を見ることで、ユーザーは「この会社は本当に仕事をしている」と確信する。

**👤 専門家③（デザイン）**:
グリッドレイアウトで写真を整然と配置。写真間のギャップは狭めにし、情報密度を上げることで「たくさんある」感を演出。ホバーで詳細情報を表示し、インタラクティブ性を持たせる。

### レイアウト

**構成**:
- 背景色: オフホワイト（#f8f8f8）
- 写真グリッド: 3カラム × 4行 = 12枚（PC初期表示）
- 各写真のアスペクト比: 4:3（やや横長）
- 「もっと見る」ボタン: グリッド下部

**写真グリッドの配置**:
- 3カラム等幅（PC）
- 写真間のギャップ: 16px
- 左右マージン: 各5%（写真を大きく見せるため狭め）

**なぜこのレイアウトか**:
- 3カラムで一覧性と詳細のバランスを取る
- 4:3のアスペクト比で、横長すぎず縦長すぎない自然な見え方
- ギャップを狭くすることで「まだまだある」感を演出

**代替案を採用しない理由**:
- ❌ 2カラム: 一覧性が下がる
- ❌ 4カラム: 各写真が小さくなりすぎる
- ❌ Masonry（可変高さ）: 整然とした印象が失われる

### コンテンツ要素

**写真カード（12-24枚）**:

各カードの内部構成:
- 写真: 4:3比率、object-fit: cover
- オーバーレイ（ホバー時）: 案件名、業種、年度

**写真カードの内容例**:

| No | 業種 | 案件名 | 年度 |
|----|------|-------|------|
| 1 | 製造業 | A社様 設備導入 | 2024 |
| 2 | 建設業 | B社様 部品加工 | 2024 |
| 3 | 食品業 | C社様 ライン構築 | 2024 |
| ... | ... | ... | ... |

**オーバーレイの内容**:
- 背景: 黒の半透明（opacity 0.6）
- 業種バッジ: 左上、小さめ（12px、背景付き）
- 案件名: 中央下部、18px、白
- 年度: 案件名の下、14px、白（opacity 0.8）

**「もっと見る」ボタン**:
- テキスト: 「もっと見る」または「さらに12件表示」
- 表示条件: 表示されていない実績がある場合
- 動作: クリックで追加12件をロード（無限スクロールまたはボタン式）

### デザイン詳細

**背景**:
- 色: オフホワイト（#f8f8f8）
- 理由: 写真を際立たせつつ、長時間見ても目が疲れない

**余白**:
- 上余白: 40px（フィルターからの続き）
- 下余白: 80px
- 左右マージン: 各5%
