# TOPページ 詳細設計

## 基本情報

**URL**: `/`（ルート）

**ファイル**: `src/app/page.tsx`

---

## このページの目的

### 👤 専門家①（マーケティング）視点

**マーケティング目標**:
1. 検索/広告から流入したユーザーを「このサイトは自分向けだ」と認識させる（3秒以内）
2. 課題を言語化し、「この会社は自分の課題を理解している」と感じさせる
3. 解決策の存在を示し、サービスページまたはお問い合わせへ誘導する

**KPI指標**:
- ファーストビュー離脱率: 30%以下
- お問い合わせページへの到達率: 15%以上
- サービスページへの遷移率: 40%以上

### 👤 専門家②（心理学）視点

**心理的ゴール**:
1. 「この会社は違う」という第一印象の形成（余白と品質で伝える）
2. 「自分の課題が解決できそう」という期待の醸成
3. 「もっと知りたい」という欲求の喚起

**活用する心理メカニズム**:
- **認知的流暢性**: 余白が多く、情報が整理されていると「信頼できる」と感じる
- **損失回避**: 「この課題を放置するとどうなるか」を暗示
- **好奇心ギャップ**: 解決策を完全には見せず、「続きが気になる」状態を作る

### 👤 専門家③（デザイン）視点

**デザイン上の役割**:
1. このテンプレートの「静謐な自信」を最も強く表現するページ
2. 余白40%以上を徹底し、「高単価サービス」の品質を視覚で伝える
3. 視線の流れを縦一方向に制御し、スクロール誘導を行う

## ユーザーの状態

**このページに来る前の状態**:
- Google検索、広告、紹介リンクからの流入
- 「課題を解決したい」という動機があるが、まだ漠然としている
- 複数社を比較検討中、または探索段階
- 「この会社はどんな会社か」という基本的な問いを持っている

**このページで感じてほしいこと**:
- 「このサイトは安っぽくない。ちゃんとしている」
- 「自分の抱えている課題をわかってくれている」
- 「解決策がありそうだ。もう少し見てみよう」

**このページから離れる時の状態**:
- サービスページへ遷移: 「具体的な内容を知りたい」
- 事例ページへ遷移: 「実績を確認したい」
- お問い合わせへ遷移: 「まずは話を聞いてみたい」
- 離脱: 「自分向けではなかった」（これは問題ない。ターゲット外を除外）

## セクション構成

TOPページは以下の5セクションで構成する:

1. ヒーロー（Hero）
2. 課題提起（Problem）
3. 解決策提示（Solution）
4. 実績サマリー（Results）
5. CTA（Call to Action）

## セクション1: ヒーロー（Hero）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
ファーストビューで「自分向けのサイトだ」と認識させる必要がある。BtoBの意思決定者は時間がないため、3秒で判断する。この3秒で「もう少し見てみよう」と思わせることが、このセクションの唯一の目的。

**👤 専門家②（心理学）**:
第一印象は0.05秒で形成される。余白と品質感で「この会社は違う」という印象を刻む。情報量を絞ることで認知的負荷を下げ、「信頼できそう」という直感的判断を促す。

**👤 専門家③（デザイン）**:
画面上部60%を余白として使い、キャッチコピーのみを中央に配置する。この「何もない」空間が、このテンプレートのアイデンティティを決定づける。

### レイアウト

**構成**:
- 画面高さ100vh（ビューポート全体を使用）
- 上部40%: 完全な余白（何も配置しない）
- 中央: キャッチコピー（72px、1行または2行）
- 中央下: サブコピー（18px、1-2行）
- 下部15%: スクロール誘導（細い線 + "Scroll"テキスト）

**なぜこのレイアウトか**:
- 上部40%の余白は「贅沢さ」の表現。高級ブランドの店舗が商品を少なく陳列するのと同じ原理
- キャッチコピーを中央に配置することで、視線を確実にそこへ集中させる
- スクロール誘導を入れることで「下にコンテンツがある」ことを示し、離脱を防ぐ

**代替案を採用しない理由**:
- ❌ 左右分割（テキスト左、画像右）: BtoBサービスでは「何ができるか」より「何を解決するか」が重要。画像は不要
- ❌ 動画背景: 重く、読み込みに時間がかかる。また視線が分散する
- ❌ スライダー: 情報が時間で切り替わるとユーザーが制御できない感覚を持つ

### コンテンツ要素

**キャッチコピー**:
- 配置: 画面中央、左右マージン各20%（中央60%幅に収める）
- サイズ: 72px（PC）、40px（SP）
- ウェイト: Bold（700）
- 色: ダークネイビー（#1a1a2e）
- 行間: 1.3
- 例文: 「あなたの事業を、次のステージへ。」
- **要件**: 句点で終わる。疑問形禁止。短く、抽象的すぎない

**サブコピー**:
- 配置: キャッチコピーの40px下
- サイズ: 18px（PC）、16px（SP）
- ウェイト: Regular（400）
- 色: ミディアムグレー（#6b7280）
- 例文: 「課題整理から実装まで、一貫してサポートします」
- **要件**: キャッチコピーを補足する具体的な内容

**スクロール誘導**:
- 配置: 画面下部から60px上
- 構成: 縦線（高さ40px、幅1px、#1a1a2e）+ "Scroll"テキスト（12px、#6b7280）
- アニメーション: 縦線が上下に2px程度ゆっくり動く（無限ループ、唯一の例外）

### デザイン詳細

**背景**:
- 色: ピュアホワイト（#FFFFFF）
- 理由: 余白を最大限に活かすため。画像や色を入れると情報量が増える

**余白**:
- 上余白: なし（ヘッダーから直結）
- 下余白: 0（100vh設計のため、下はスクロールで次セクションへ）

**特徴的な表現**:
- 画面上部40%の「意図的な空白」
- 背景画像・イラスト・アイコンを一切使用しない
- テキストのみで構成する勇気

### アニメーション

**使用する**: スクロール誘導の縦線のみ

**縦線のアニメーション**:
```tsx
const scrollIndicator = {
  animate: { y: [0, 8, 0] },
  transition: {
    duration: 2,
    repeat: Infinity,
    ease: "easeInOut"
  }
};
```

**キャッチコピー・サブコピー**:
- アニメーションなし
- 理由: ページロード時に即座に表示されるべき。遅延は離脱リスク

### SP（スマートフォン）での変化

**レイアウト変更**:
- 画面高さ: 100svh（Safe Viewport Height、モバイルブラウザのUI考慮）
- 上部余白: 30%に縮小

**サイズ変更**:
- キャッチコピー: 40px
- サブコピー: 16px
- 左右マージン: 5%（画面幅90%使用）

**省略要素**: なし

### このセクションでの心理効果

**ユーザーが感じること**:
- 「シンプルで見やすい」
- 「この会社は自信がありそうだ」
- 「何をしている会社か、なんとなくわかった」

**次のセクションへの期待**:
- 「もう少し具体的に知りたい」という欲求の喚起
- スクロール誘導により「下に何かある」という期待

## セクション2: 課題提起（Problem）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
ユーザーは「自分の課題を理解してくれる会社」に相談したい。課題を言語化することで「この会社は自分のことをわかっている」と感じさせる。ここで共感を得られないと、以降のコンテンツは読まれない。

**👤 専門家②（心理学）**:
人は自分の悩みを言語化されると、「わかってくれている」と感じる。さらに、課題を放置した場合のリスクを暗示することで「損失回避」の心理を活性化し、行動への動機を高める。

**👤 専門家③（デザイン）**:
ヒーローの余白から一転、このセクションでは「密」のリズムを作る。3つの課題を縦に並べ、情報密度を上げることで、ユーザーに「読むべき内容がある」と認識させる。

### レイアウト

**構成**:
- 背景色でセクション区切り（オフホワイト #f8f8f8）
- セクション見出し（中央配置、48px）
- 見出しの60px下に課題カード3つを縦に配置
- 各課題カードは左右マージン各25%（中央50%幅）

**課題カードの内部構成**:
- 番号（32px、アクセントカラー、太字）
- 課題タイトル（24px、ダークネイビー）
- 課題説明文（17px、2-3行、ミディアムグレー）

**なぜこのレイアウトか**:
- 3つの課題を縦に並べることで「順番に読む」流れを作る
- 中央50%幅に収めることで、左右の余白を確保しつつ、読みやすい行長を維持
- 番号を入れることで「構造化されている」印象を与え、信頼感を高める

**代替案を採用しない理由**:
- ❌ 3カラム横並び: SPで崩れやすく、一覧性が高すぎて「流し読み」されやすい
- ❌ アコーディオン: クリックが必要になり、離脱リスクが上がる
- ❌ アイコン付き: 装飾が増え、Minimal表現に反する

### コンテンツ要素

**セクション見出し**:
- テキスト: 「こんな課題を抱えていませんか？」
- サイズ: 48px（PC）、32px（SP）
- 配置: 中央揃え
- 色: ダークネイビー

**課題カード（3つ）**:

| No | 課題タイトル例 | 説明文の方向性 |
|----|--------------|--------------|
| 01 | 「属人化から脱却できない」 | ノウハウが個人に依存し、組織として成長できない状態を描写 |
| 02 | 「リソースが足りない」 | 採用難、育成コスト、外注の質問題を具体的に |
| 03 | 「何から始めればいいかわからない」 | 課題は認識しているが、解決策が見えない焦りを表現 |

**各課題カードの配置**:
- 課題間の余白: 60px
- 番号と課題タイトルの間: 16px
- 課題タイトルと説明文の間: 12px

### デザイン詳細

**背景**:
- 色: オフホワイト（#f8f8f8）
- 理由: ヒーローの白から切り替えることで、新しいセクションに入ったことを視覚的に示す

**余白**:
- 上余白: 200px（ヒーローからの「断絶」を表現。新しい話題が始まる）
- 下余白: 80px（次のセクションとは「連続」。解決策へ繋げる）

**特徴的な表現**:
- 番号（01, 02, 03）をアクセントカラーで表示し、視線の起点を作る
- カードにボーダーや影をつけない。背景色と余白のみで区切る
- 課題説明文は左揃え（中央揃えにすると読みにくい）

### アニメーション

**使用する**: セクション出現時のフェードイン

**アニメーション仕様**:
```tsx
// セクション全体
const sectionFade = {
  initial: { opacity: 0 },
  whileInView: { opacity: 1 },
  transition: { duration: 0.6 },
  viewport: { once: true, margin: "-100px" }
};

// 各課題カード（stagger効果）
const cardFade = {
  initial: { opacity: 0, y: 20 },
  whileInView: { opacity: 1, y: 0 },
  transition: { duration: 0.5, delay: index * 0.1 },
  viewport: { once: true }
};
```

- 各カードを0.1秒ずつ遅らせて順番に表示
- ただし、合計でも0.8秒以内に全て表示完了

### SP（スマートフォン）での変化

**レイアウト変更**:
- 左右マージン: 5%（画面幅90%使用）

**サイズ変更**:
- セクション見出し: 32px
- 番号: 28px
- 課題タイトル: 20px
- 説明文: 16px

**余白変更**:
- 上余白: 120px
- 下余白: 60px
- 課題間: 48px

### このセクションでの心理効果

**ユーザーが感じること**:
- 「まさにこれ。自分の状況と同じだ」
- 「この会社は自分たちの課題をわかっている」
- 「このまま放置したらまずい」

**次のセクションへの期待**:
- 「では、どう解決するのか」という疑問
- 解決策を知りたいという欲求の高まり

## セクション3: 解決策提示（Solution）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
課題を提示した直後に解決策を見せることで、「この会社に頼めば解決できる」という認識を形成する。ただし、詳細はサービスページに譲り、ここでは「解決できるということ」だけを伝える。

**👤 専門家②（心理学）**:
課題提起で高まった緊張感を、解決策の提示で緩和する。このリリーフ効果により、ユーザーはポジティブな感情を抱く。ただし、全てを見せると満足して離脱するため、「もっと知りたい」を残す。

**👤 専門家③（デザイン）**:
課題セクションの「密」から、再び「疎」へ戻す。大きな余白の中に1つのメッセージを配置し、インパクトを与える。ここが視覚的なクライマックス。

### レイアウト

**構成**:
- 背景色: ピュアホワイト（#FFFFFF）に戻す
- 左側30%: 大きな見出し「解決します。」（縦書き風に1行1文字、または横書き）
- 右側50%: 解決策の説明（3つのポイント）
- 左右間のギャップ: 10%

**解決策ポイントの内部構成**:
- ポイント見出し（24px、ダークネイビー）
- ポイント説明（17px、2行程度）

**なぜこのレイアウトか**:
- 左右分割により「課題セクション」との視覚的差別化
- 左側の大きな見出しで「解決」というメッセージを強く印象づける
- 右側のポイントで「どう解決するか」の概要を示す

**代替案を採用しない理由**:
- ❌ 課題と同じ縦並び: 単調になり、視覚的なメリハリがなくなる
- ❌ 図解やフローチャート: 情報量が増え、Minimal表現に反する
- ❌ ビフォーアフター比較: 詳細すぎる。サービスページに譲る

### コンテンツ要素

**左側見出し**:
- テキスト: 「私たちが、解決します。」
- サイズ: 48px（PC）
- ウェイト: Bold
- 色: ダークネイビー

**右側ポイント（3つ）**:

| No | ポイント見出し例 | 説明の方向性 |
|----|----------------|-------------|
| 01 | 「課題の本質を見極める」 | 表面的な対処ではなく、根本原因にアプローチ |
| 02 | 「貴社に最適な形で提供」 | 標準パッケージではなく、カスタマイズ対応 |
| 03 | 「成果が出るまで伴走」 | 納品して終わりではなく、継続的サポート |

**CTAボタン**:
- 配置: ポイント3つの下、80px余白を空けて
- テキスト: 「サービス詳細を見る」
- スタイル: テキストリンク + 右矢印（→）、ボタン型ではない
- 色: ダークネイビー、ホバー時にアクセントカラー
- 理由: ここではCVではなく「サービスページへの誘導」が目的。控えめに

### デザイン詳細

**背景**:
- 色: ピュアホワイト（#FFFFFF）
- 理由: 課題セクションのオフホワイトから切り替え、新しいフェーズを示す

**余白**:
- 上余白: 160px
- 下余白: 160px
- 左右マージン: 各10%（コンテンツ幅80%）

**特徴的な表現**:
- 左右非対称レイアウト（30:10:50の比率）
- 左側見出しは縦に長く、視線を上から下へ誘導
- ポイント間の余白は48px

### アニメーション

**使用する**: セクション出現時のフェードイン + 左右からのスライド

**アニメーション仕様**:
```tsx
// 左側見出し
const leftSlide = {
  initial: { opacity: 0, x: -30 },
  whileInView: { opacity: 1, x: 0 },
  transition: { duration: 0.6 },
  viewport: { once: true }
};

// 右側コンテンツ
const rightSlide = {
  initial: { opacity: 0, x: 30 },
  whileInView: { opacity: 1, x: 0 },
  transition: { duration: 0.6, delay: 0.1 },
  viewport: { once: true }
};
```

- 左右から同時に中央へ向かって登場
- 移動距離は30px（控えめ）

### SP（スマートフォン）での変化

**レイアウト変更**:
- 左右分割を解除し、縦並びに変更
- 見出し → ポイント3つ → CTA の順

**サイズ変更**:
- 見出し: 36px
- ポイント見出し: 20px
- ポイント説明: 16px

**余白変更**:
- 上余白: 100px
- 下余白: 100px
- 左右マージン: 5%

### このセクションでの心理効果

**ユーザーが感じること**:
- 「なるほど、こういうアプローチで解決するのか」
- 「自分の課題に合っていそうだ」
- 「もう少し詳しく知りたい」

**次のセクションへの期待**:
- 「本当に成果が出るのか」という疑問
- 実績・事例を確認したいという欲求

## セクション4: 実績サマリー（Results）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
課題と解決策を示した後、「本当に成果が出るのか」という疑問に答える必要がある。具体的な数字を見せることで、信頼を確定させる。

**👤 専門家②（心理学）**:
数字は具体性バイアスを活用できる。「多くの企業で成果」より「売上32%向上」の方が信頼される。また、社会的証明として「他社も成功している」という安心感を与える。

**👤 専門家③（デザイン）**:
数字を大きく見せることで、視覚的なインパクトを与える。3つの数字を横に並べ、一目で「成果が出ている」と認識させる。

### レイアウト

**構成**:
- 背景色: ダークネイビー（#1a1a2e）で反転
- セクション見出し: 「成果の一例」（白文字、中央配置）
- 数字3つを横並び（PC）、縦並び（SP）
- 各数字の下に説明文

**数字カードの内部構成**:
- 数字（96px、白、Bold）
- 単位（24px、白）
- 説明（17px、ライトグレー #a0a0a0）

**なぜこのレイアウトか**:
- 背景色を反転させることで、このセクションの重要性を視覚的に示す
- 数字を大きくすることで、スクロール中でも目に留まる
- 3つに絞ることで、情報過多を防ぐ

**代替案を採用しない理由**:
- ❌ ロゴ一覧: 安っぽく見え、具体的な成果がわからない
- ❌ お客様の声（テキスト）: 長くなり、Minimalに反する

## セクション5: CTA（Call to Action）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
ここまで読んだユーザーは「興味がある」状態。このタイミングでCTAを出さないと、離脱されてしまう。「今すぐ問い合わせ」へ導く最後のプッシュ。

**👤 専門家②（心理学）**:
人は選択肢が多いと決断を先延ばしにする。CTAを1つに絞り、「問い合わせる」という単一のアクションに集中させる。また、「費用は発生しません」等の安心材料で障壁を下げる。

**👤 専門家③（デザイン）**:
ページ最後のセクションとして、「締め」の役割を果たす。余白を大きく取り、CTAボタンに視線を集中させる。

### レイアウト

**構成**:
- 背景色: オフホワイト（#f8f8f8）
- 中央に大きな見出し（48px）
- 見出しの下に補足テキスト（17px、1-2行）
- 補足テキストの下にCTAボタン（アクセントカラー）

**なぜこのレイアウトか**:
- 中央配置により視線を集中
- 大きな見出しで「行動してほしい」というメッセージを明確に
- 補足テキストで障壁（費用、時間）を下げる

**代替案を採用しない理由**:
- ❌ 左右分割: 視線が分散する
- ❌ 複数のCTA: 選択肢が増えると決断が遅れる
- ❌ フォーム埋め込み: お問い合わせページに誘導する方がフォーム完了率が高い

### コンテンツ要素

**見出し**:
- テキスト: 「まずは、お話しませんか。」
- サイズ: 48px（PC）、32px（SP）
- 色: ダークネイビー
- 配置: 中央揃え

**補足テキスト**:
- テキスト: 「無料でご相談いただけます。お気軽にどうぞ。」
- サイズ: 17px
- 色: ミディアムグレー（#6b7280）
- 配置: 中央揃え

**CTAボタン**:
- テキスト: 「お問い合わせ」
- サイズ: 幅280px、高さ64px
- 背景色: アクセントカラー（#c45c3e）
- 文字色: 白
- 角丸: 4px（角丸すぎない）
- ホバー: 背景色がダークネイビーに変化
- 配置: 補足テキストの48px下、中央揃え

### デザイン詳細

**背景**:
- 色: オフホワイト（#f8f8f8）
- 理由: 実績セクション（ダークネイビー）からの切り替え。白に戻しつつ、フッターとの区別

**余白**:
- 上余白: 160px
- 下余白: 200px（フッターへの余韻）
- 左右マージン: 各20%

**特徴的な表現**:
- CTAボタンはこのページで唯一の「塗りボタン」
- アクセントカラーを使用するのもこのボタンのみ
- 希少性により、ボタンへの注目度が上がる

### アニメーション

**使用する**: CTAボタンのホバーアニメーション

**アニメーション仕様**:
```tsx
const buttonHover = {
  whileHover: {
    backgroundColor: "#1a1a2e",
    transition: { duration: 0.2 }
  }
};
```

- セクション出現時のフェードインは使用しない（他セクションと統一感を持たせるなら可）
- ホバー時の色変化のみ

### SP（スマートフォン）での変化

**レイアウト変更**: なし（元々中央配置）

**サイズ変更**:
- 見出し: 32px
- 補足テキスト: 16px
- ボタン: 幅90%、高さ56px

**余白変更**:
- 上余白: 100px
- 下余白: 120px
- 左右マージン: 5%

### このセクションでの心理効果

**ユーザーが感じること**:
- 「問い合わせのハードルは低そうだ」
- 「無料なら聞いてみてもいいか」
- 「このボタンを押せばいいんだな」

**次の行動**:
- CTAクリック → お問い合わせページへ遷移
- スクロールダウン → フッター（サイトマップ）へ
