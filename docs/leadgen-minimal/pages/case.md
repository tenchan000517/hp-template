# 事例ページ 詳細設計

## 基本情報

**URL**: `/case`

**ファイル**: `src/app/case/page.tsx`

---

## このページの目的

### 👤 専門家①（マーケティング）視点

**マーケティング目標**:
1. 「本当に成果が出るのか」という疑問に対し、具体的な実績で回答する
2. 「自分と似た課題を持つ企業が成功している」という社会的証明を提供する
3. 事例を読み終えたユーザーを、お問い合わせへ確実に誘導する

**KPI指標**:
- お問い合わせページへの遷移率: 35%以上
- 事例詳細の閲覧率: 80%以上（ページに来た人が詳細まで読む）
- 平均滞在時間: 3分以上

### 👤 専門家②（心理学）視点

**心理的ゴール**:
1. 「他社も成功している」という社会的証明による安心感
2. 「自分もこうなれる」という具体的なイメージの形成
3. 「放置すると機会損失になる」という行動喚起

**活用する心理メカニズム**:
- **社会的証明**: 同業他社の成功事例は強力な説得材料
- **具体性バイアス**: 数字や固有名詞があると信頼度が上がる
- **ストーリー効果**: 課題→解決→成果の流れは記憶に残りやすい

### 👤 専門家③（デザイン）視点

**デザイン上の役割**:
1. 事例を「読み物」として構成し、最後まで読ませる
2. 数字（成果）を視覚的に強調し、インパクトを与える
3. 各事例を独立させつつ、ページ全体の統一感を維持

---

## ユーザーの状態

**このページに来る前の状態**:
- TOPまたはサービスページで興味を持っている
- 「サービス内容はわかったが、本当に成果が出るのか」疑問がある
- 問い合わせる前に「証拠」を見たい

**このページで感じてほしいこと**:
- 「具体的な成果が数字で示されている」
- 「自分と似た課題を持つ企業が成功している」
- 「この会社に頼めば自分もこうなれそう」

**このページから離れる時の状態**:
- お問い合わせへ遷移: 「自分も相談してみよう」
- サービスページへ遷移: 「サービス内容をもう一度確認したい」
- 離脱: 「自分の業界とは違う」（ターゲット外。問題なし）

## セクション2: 事例サマリー（Case Summary）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
事例の全体像を一覧で見せ、興味のある事例にすぐアクセスできるようにする。また、3件あることで「実績の厚み」を示す。

**👤 専門家②（心理学）**:
選択肢を提示することで、ユーザーに「自分で選んだ」感覚を与える。これにより事例への関与度が上がる。

**👤 専門家③（デザイン）**:
3件の事例を横並びで簡潔に表示。詳細は下のセクションで展開するため、ここでは概要のみ。

### レイアウト

**構成**:
- 背景色: オフホワイト（#f8f8f8）
- 事例カード3つを横並び（PC）、縦並び（SP）
- 各カードは業種 + 課題サマリー + 成果数字

**各カードの内部構成**:
- 業種タグ（14px、ミディアムグレー、背景なし）
- 課題サマリー（20px、2行程度、ダークネイビー）
- 成果数字（48px、アクセントカラー）+ 単位（18px）
- 成果説明（14px、ミディアムグレー）

**なぜこのレイアウトか**:
- 3カラム横並びで一覧性を確保
- 成果数字を大きくすることで、スクロール中でも目に留まる
- 業種タグで「自分と関係あるか」を即座に判断できる

**代替案を採用しない理由**:
- ❌ カードにクリック機能: 詳細は同一ページ内に配置するため、ページ遷移は不要
- ❌ 画像付きカード: 余分な要素が増え、Minimalに反する
- ❌ スライダー: 一覧性が下がる

### コンテンツ要素

**事例カード（3件）**:

| 業種 | 課題サマリー | 成果数字 | 単位 | 成果説明 |
|------|------------|----------|------|---------|
| 製造業 | 業務の属人化による生産性低下 | 45 | % | 生産性向上 |
| IT・通信 | 新規顧客獲得の頭打ち | 2.3 | 倍 | 問い合わせ数増加 |
| 小売・流通 | EC売上の伸び悩み | 180 | % | 売上成長 |

**カードの配置**:
- 3カラム等幅
- カード間のギャップ: 40px
- 左右マージン: 各10%

### デザイン詳細

**背景**:
- 色: オフホワイト（#f8f8f8）
- 理由: ページヘッダーの白から切り替え

**余白**:
- 上余白: 120px
- 下余白: 120px

**カードのスタイル**:
- 背景: なし（背景色と同一）
- ボーダー: なし
- 影: なし
- カード内パディング: 40px 0（縦のみ）

**特徴的な表現**:
- カード間を細い縦線（幅1px、#e0e0e0）で区切る（PC時のみ）
- 成果数字をアクセントカラーで強調
- カードにホバー効果なし（詳細は同一ページ内のため）

### アニメーション

**使用する**: セクション出現時のフェードイン

**アニメーション仕様**:
```tsx
const cardFade = {
  initial: { opacity: 0 },
  whileInView: { opacity: 1 },
  transition: { duration: 0.5, delay: index * 0.1 },
  viewport: { once: true }
};
```

### SP（スマートフォン）での変化

**レイアウト変更**:
- 横並び → 縦並び
- 縦線区切り → 横線区切り

**サイズ変更**:
- 成果数字: 40px
- 課題サマリー: 18px

**余白変更**:
- 上余白: 80px
- 下余白: 80px
- カード間: 60px
- 左右マージン: 5%

### このセクションでの心理効果

**ユーザーが感じること**:
- 「3件の事例があり、実績がある」
- 「数字で成果が示されている」
- 「自分の業種に近い事例がある」

**次のセクションへの期待**:
- 「詳細を知りたい」

## セクション3: 事例詳細（Case Detail）× 3件

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
事例を「ストーリー」として伝えることで、ユーザーに「自分もこうなれる」と思わせる。課題→解決→成果の流れは最も説得力がある。

**👤 専門家②（心理学）**:
人はストーリーを好み、記憶に残りやすい。また、具体的な数字と固有名詞（業種）は信頼を高める。

**👤 専門家③（デザイン）**:
各事例を独立したセクションとして配置し、読み物として構成する。見出しと本文のコントラストでスキャンしやすく。

### レイアウト

**各事例の構成**:
- 背景色を交互に変える（白 → オフホワイト → 白）
- 左右分割なし。中央1カラム（幅60%）
- 事例タイトル → 背景情報 → 課題 → 解決策 → 成果 の順

**各パートの配置**:
```
[事例タイトル] 業種名 × 課題キーワード（28px）
[背景情報] 企業規模、状況の説明（17px、3-4行）
[課題] 見出し「課題」+ 本文（17px、3-4行）
[解決策] 見出し「解決策」+ 本文（17px、3-4行）
[成果] 見出し「成果」+ 数字ハイライト + 説明
```

**なぜこのレイアウトか**:
- 中央1カラムは「読み物」として最適な行長
- 課題→解決策→成果の順序で論理的なストーリーを構成
- 見出しで各パートを区切り、スキャンも可能に

**代替案を採用しない理由**:
- ❌ 左右分割: 読み物として不自然
- ❌ タブ/アコーディオン: 全体が見えず、ストーリーが分断される
- ❌ カード化: 情報量が多すぎてカードに収まらない

### コンテンツ要素（事例1: 製造業）

**事例タイトル**:
- テキスト: 「製造業 × 業務改善」
- サイズ: 28px（PC）、24px（SP）
- 色: ダークネイビー

**背景情報**:
- テキスト: 「従業員300名の精密機器メーカー。創業40年、業界内でのシェアは高いものの、ベテラン社員への依存度が高く、ノウハウの属人化が課題となっていた。」
- サイズ: 17px
- 色: ミディアムグレー

**課題**:
- 見出し: 「課題」（20px、ダークネイビー）
- 本文: 「製造ラインの最適化は特定のベテラン社員に依存。退職リスクへの対策が急務だった。また、若手社員への技術継承が進まず、生産性が徐々に低下していた。」

**解決策**:
- 見出し: 「解決策」（20px、ダークネイビー）
- 本文: 「まず、ベテラン社員へのヒアリングを実施し、暗黙知を言語化。その上で、作業手順のマニュアル化と、若手向け研修プログラムを設計・実施した。」

**成果**:
- 見出し: 「成果」（20px、ダークネイビー）
- 数字ハイライト: 「45%」（64px、アクセントカラー）+ 「生産性向上」（18px）
- 説明: 「導入から6ヶ月で生産性が45%向上。また、若手社員の離職率が20%改善した。」

### コンテンツ要素（事例2: IT・通信）

**事例タイトル**: 「IT・通信 × 営業強化」

**背景情報**:
「従業員50名のSaaS企業。プロダクトは好評だが、新規顧客獲得が頭打ちになっていた。営業チームの拡大が必要だが、採用は難航。」

**課題**:
「営業活動が属人的で、成果にばらつきがあった。また、マーケティングとの連携が弱く、リードの質が安定しなかった。」

**解決策**:
「営業プロセスを分解し、各フェーズでのKPIを設定。マーケティング部門と連携し、リードの定義と評価基準を統一した。」

**成果**:
- 数字ハイライト: 「2.3倍」（64px）+ 「問い合わせ数」（18px）
- 説明: 「導入から3ヶ月で問い合わせ数が2.3倍に。商談化率も15%向上した。」

### コンテンツ要素（事例3: 小売・流通）

**事例タイトル**: 「小売・流通 × EC強化」

**背景情報**:
「従業員100名の食品メーカー。直販EC事業を始めたが、伸び悩んでいた。実店舗での知名度はあるが、オンラインでの認知が低い。」

**課題**:
「EC担当者が1名しかおらず、マーケティングもカスタマーサポートも兼任。戦略的な施策を打つ余裕がなかった。」

**解決策**:
「まず、EC運営の業務を整理し、外部リソースを活用できる体制を構築。その上で、SEOとコンテンツマーケティングを強化した。」

**成果**:
- 数字ハイライト: 「180%」（64px）+ 「売上成長」（18px）
- 説明: 「導入から12ヶ月でEC売上が180%成長。担当者の業務負荷も大幅に軽減した。」

### デザイン詳細

**背景**:
- 事例1: ピュアホワイト（#FFFFFF）
- 事例2: オフホワイト（#f8f8f8）
- 事例3: ピュアホワイト（#FFFFFF）
- 理由: 交互に切り替え、事例間の区切りを示す

**余白**:
- 各事例の上余白: 160px
- 各事例の下余白: 160px
- 見出しと本文の間: 16px
- パート間（課題→解決策等）: 60px
- コンテンツ幅: 60%（左右各20%マージン）

**成果の数字ハイライト**:
- 数字: 64px、アクセントカラー、Bold
- 単位/説明: 18px、ダークネイビー
- 配置: 数字を左、説明を右に横並び（または数字の下に説明）

**特徴的な表現**:
- 見出し（課題、解決策、成果）は20pxで本文と区別
- 見出しの前に細い横線（幅40px、高さ2px、ダークネイビー）を配置（オプション）
- 成果の数字はアクセントカラーで目立たせる

### アニメーション

**使用する**: 各パートの順次フェードイン

**アニメーション仕様**:
```tsx
const partFade = {
  initial: { opacity: 0, y: 20 },
  whileInView: { opacity: 1, y: 0 },
  transition: { duration: 0.5 },
  viewport: { once: true, margin: "-50px" }
};
```

### SP（スマートフォン）での変化

**レイアウト変更**:
- コンテンツ幅: 90%

**サイズ変更**:
- 事例タイトル: 24px
- 見出し: 18px
- 本文: 16px
- 成果数字: 48px

**余白変更**:
- 各事例の上下余白: 100px
- パート間: 48px
- 左右マージン: 5%

### このセクションでの心理効果

**ユーザーが感じること**:
- 「自分と似た課題が解決されている」
- 「プロセスが論理的で、再現性がありそう」
- 「この数字は具体的で信頼できる」

**次のセクションへの期待**:
- 「自分も相談してみたい」
