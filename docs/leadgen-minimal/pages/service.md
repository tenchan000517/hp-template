# サービスページ 詳細設計

## 基本情報

**URL**: `/service`

**ファイル**: `src/app/service/page.tsx`

---

## このページの目的

### 👤 専門家①（マーケティング）視点

**マーケティング目標**:
1. TOPページで興味を持ったユーザーに「具体的に何を提供するか」を伝える
2. サービスの価値を論理的に説明し、「この会社に頼みたい」と思わせる
3. 事例ページまたはお問い合わせページへ誘導する

**KPI指標**:
- お問い合わせページへの遷移率: 25%以上
- 事例ページへの遷移率: 30%以上
- 平均滞在時間: 2分以上

### 👤 専門家②（心理学）視点

**心理的ゴール**:
1. 「この会社のサービスは自分の課題に合っている」という認識の強化
2. 「なぜこの会社を選ぶべきか」という論理的根拠の提供
3. 「プロセスがわかっている」という安心感の醸成

**活用する心理メカニズム**:
- **認知負荷軽減**: 複雑なサービスをステップに分解し、理解しやすくする
- **コントラスト効果**: 「他社との違い」を明示することで選択を促す
- **プロセス可視化**: 先が見えることで不安を解消

### 👤 専門家③（デザイン）視点

**デザイン上の役割**:
1. TOPページよりも情報量が増えるが、Minimal表現を維持する
2. 「読ませる」より「見せる」レイアウトで、理解を促進する
3. 各セクションの役割を明確にし、スキャンしやすい構造を作る

## ユーザーの状態

**このページに来る前の状態**:
- TOPページで「この会社は信頼できそう」と感じている
- 「具体的に何をしてくれるのか」を知りたい
- まだ問い合わせるかどうかは決めていない

**このページで感じてほしいこと**:
- 「サービス内容が明確でわかりやすい」
- 「他社とは違うアプローチだ」
- 「このプロセスなら安心して任せられる」

**このページから離れる時の状態**:
- お問い合わせへ遷移: 「話を聞いてみたい」
- 事例ページへ遷移: 「実際の成果を確認したい」
- 離脱: 「自分の課題とは違った」（ターゲット外の除外。問題なし）

## セクション構成

サービスページは以下の5セクションで構成する:

1. ページヘッダー（Page Header）
2. サービス概要（Service Overview）
3. 選ばれる理由（Why Choose Us）
4. プロセス（Process）
5. CTA（Call to Action）

## セクション1: ページヘッダー（Page Header）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
ナビゲーションから遷移してきたユーザーに「正しいページに来た」と確認させる。また、このページで何を得られるかを0.5秒で理解させる。

**👤 専門家②（心理学）**:
新しいページに遷移した際、ユーザーは一瞬「ここはどこか」という認知的確認を行う。ページヘッダーがこの確認を助け、コンテンツへの集中を促す。

**👤 専門家③（デザイン）**:
TOPページのヒーローとは異なり、簡潔に。ページタイトルと短い説明のみで構成し、すぐに本題へ入る。

### レイアウト

**構成**:
- 高さ: 40vh（ビューポートの40%）
- ページタイトル（48px）中央配置
- タイトル下に説明文（18px）1行

**なぜこのレイアウトか**:
- TOPページのヒーロー（100vh）よりも小さく、下層ページであることを示す
- ただし、十分な余白を確保し、Minimal表現を維持
- 説明文1行で「このページの目的」を端的に伝える

**代替案を採用しない理由**:
- ❌ 100vhのヒーロー: 下層ページには大げさ。スクロール量が増える
- ❌ 背景画像付き: 情報量が増え、Minimalに反する
- ❌ パンくずリスト: このテンプレートではナビゲーションがフラットなため不要

### コンテンツ要素

**ページタイトル**:
- テキスト: 「サービス」
- サイズ: 48px（PC）、36px（SP）
- ウェイト: Bold
- 色: ダークネイビー
- 配置: 中央揃え、垂直方向は中央よりやや上

**説明文**:
- テキスト: 「私たちが提供する価値と、その実現方法をご紹介します。」
- サイズ: 18px（PC）、16px（SP）
- 色: ミディアムグレー
- 配置: タイトルの24px下、中央揃え

### デザイン詳細

**背景**:
- 色: ピュアホワイト（#FFFFFF）
- 理由: シンプルに徹する

**余白**:
- 上余白: ヘッダーから120px
- 下余白: 80px

**特徴的な表現**:
- 装飾なし。テキストのみ
- ヘッダーからの距離を十分に取り、呼吸を持たせる

### アニメーション

**使用しない**

理由: ページロード時に即座に表示されるべき。ヘッダーセクションのアニメーションはユーザーを待たせる印象を与える。

### SP（スマートフォン）での変化

**レイアウト変更**: なし

**サイズ変更**:
- タイトル: 36px
- 説明文: 16px
- 高さ: 35vh

**余白変更**:
- 上余白: 80px
- 下余白: 60px

### このセクションでの心理効果

**ユーザーが感じること**:
- 「サービスページに来た」という確認
- 「シンプルで見やすそうだ」

**次のセクションへの期待**:
- 「具体的なサービス内容を知りたい」

## セクション2: サービス概要（Service Overview）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
提供するサービスの全体像を示す。BtoBでは「何をしてくれるか」が明確でないと信頼されない。ただし、詳細に入りすぎず、価値提案に焦点を当てる。

**👤 専門家②（心理学）**:
人は「理解できるもの」を好む。サービスを3-4つのカテゴリに分類することで、認知的に処理しやすくなる。マジカルナンバー4±1の法則。

**👤 専門家③（デザイン）**:
情報量が増えるセクションだが、カードUIを避ける。背景色の切り替えと余白で区切り、Minimal表現を維持。

### レイアウト

**構成**:
- 背景色: オフホワイト（#f8f8f8）
- セクション見出し（36px）左揃え
- サービスカテゴリ3つを縦に配置
- 各カテゴリは左右2分割（40:60の比率）

**各カテゴリの内部構成**:
- 左側（40%）: カテゴリ番号（32px） + カテゴリ名（28px）
- 右側（60%）: 説明文（17px、3-4行） + サービス項目リスト

**なぜこのレイアウトか**:
- 左右分割により「番号/名前」と「詳細」を視覚的に分離
- 縦スタックにより順番に読ませる
- 左側の番号が視線の起点となり、スキャンしやすい

**代替案を採用しない理由**:
- ❌ 3カラム横並び: SPで崩れやすく、一覧性が高すぎて流し読みされる
- ❌ タブ切り替え: クリックが必要で、全体像が見えにくい
- ❌ アコーディオン: 同上

### コンテンツ要素

**セクション見出し**:
- テキスト: 「サービス内容」
- サイズ: 36px（PC）、28px（SP）
- 色: ダークネイビー
- 配置: 左揃え、左マージン10%

**サービスカテゴリ（3つ）**:

| No | カテゴリ名 | 説明の方向性 | 項目例 |
|----|----------|-------------|-------|
| 01 | 戦略策定 | 現状分析から戦略立案まで | 現状診断、ゴール設定、ロードマップ作成 |
| 02 | 実行支援 | 戦略を具体的なアクションに | プロジェクト管理、リソース提供、進捗管理 |
| 03 | 定着化 | 成果を持続させる仕組みづくり | マニュアル整備、研修、効果測定 |

**サービス項目リストのスタイル**:
- ビュレットなし（・や●を使わない）
- 項目間は16px余白
- 項目の前に「—」（emダッシュ）を付ける
- 色: ミディアムグレー

### デザイン詳細

**背景**:
- 色: オフホワイト（#f8f8f8）
- 理由: ページヘッダーの白から切り替え、セクションの区切りを示す

**余白**:
- 上余白: 160px
- 下余白: 160px
- カテゴリ間: 120px
- 左右マージン: 各10%

**カテゴリ間の区切り**:
- 細い水平線（幅100%、高さ1px、#e0e0e0）
- または区切りなし（余白のみで分離）

**特徴的な表現**:
- 左40%:右60%の非対称レイアウト
- 番号（01, 02, 03）を大きく表示し、構造を視覚化
- カードにせず、背景色と余白のみで区切る

### アニメーション

**使用する**: セクション出現時のフェードイン

**アニメーション仕様**:
```tsx
// 各カテゴリのstagger効果
const categoryFade = {
  initial: { opacity: 0, y: 20 },
  whileInView: { opacity: 1, y: 0 },
  transition: { duration: 0.5, delay: index * 0.15 },
  viewport: { once: true, margin: "-50px" }
};
```

### SP（スマートフォン）での変化

**レイアウト変更**:
- 左右分割を解除し、縦並びに
- 番号 → カテゴリ名 → 説明文 → 項目リスト の順

**サイズ変更**:
- セクション見出し: 28px
- カテゴリ名: 24px
- 説明文: 16px

**余白変更**:
- 上余白: 100px
- 下余白: 100px
- カテゴリ間: 80px
- 左右マージン: 5%

### このセクションでの心理効果

**ユーザーが感じること**:
- 「サービスの全体像が理解できた」
- 「段階的に進めてくれるんだな」
- 「自分に必要なサービスがありそう」

**次のセクションへの期待**:
- 「なぜこの会社を選ぶべきか」への疑問

## セクション3: 選ばれる理由（Why Choose Us）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
競合と比較検討しているユーザーに「この会社を選ぶ理由」を提示する。サービス内容だけでは差別化できない。「なぜ私たちか」を論理的に説明する。

**👤 専門家②（心理学）**:
人は選択する際、「選んだ理由」を言語化したい。このセクションがその言語を提供する。上司への報告や稟議書にそのまま使える表現を意識する。

**👤 専門家③（デザイン）**:
サービス概要セクションからデザインを変え、メリハリをつける。大きな数字やキーワードを使い、スキャンで要点を把握できるように。

### レイアウト

**構成**:
- 背景色: ピュアホワイト（#FFFFFF）
- セクション見出し（36px）中央配置
- 選ばれる理由3つを横並び（PC）、縦並び（SP）
- 各理由はキーワード（28px）+ 説明文（17px）

**各理由の内部構成**:
- アイコンなし
- キーワード（28px、Bold、ダークネイビー）
- 説明文（17px、3-4行、ミディアムグレー）

**なぜこのレイアウトか**:
- 3カラム横並びにより「比較」しやすい
- サービス概要の縦並びとは異なり、視覚的変化を与える
- キーワードを大きくすることで、スクロール中でも目に留まる

**代替案を採用しない理由**:
- ❌ アイコン付き: 装飾が増え、Minimalに反する
- ❌ カードUI: 角丸・影がAIデザインっぽくなる
- ❌ 番号付き: サービス概要と被る

### コンテンツ要素

**セクション見出し**:
- テキスト: 「選ばれる理由」
- サイズ: 36px（PC）、28px（SP）
- 配置: 中央揃え

**選ばれる理由（3つ）**:

| キーワード | 説明の方向性 |
|-----------|-------------|
| 本質への集中 | 表面的な対処ではなく、課題の根本原因にアプローチする姿勢 |
| 柔軟な対応 | 標準パッケージではなく、各社の状況に合わせたカスタマイズ |
| 成果へのコミット | 納品して終わりではなく、成果が出るまで責任を持つ |

**各カラムの配置**:
- 3カラム等幅（PC）
- カラム間のギャップ: 60px
- 左右マージン: 各10%

### デザイン詳細

**背景**:
- 色: ピュアホワイト（#FFFFFF）
- 理由: サービス概要のオフホワイトから切り替え

**余白**:
- 上余白: 200px（強調のため大きく）
- 下余白: 160px
- セクション見出しと理由群の間: 80px

**特徴的な表現**:
- キーワードを28pxと大きく表示
- アイコン・装飾を一切使わない
- 文字のみで構成する勇気

**カラム間の区切り**:
- 区切り線なし
- 余白のみで分離

### アニメーション

**使用する**: セクション出現時のフェードイン

**アニメーション仕様**:
```tsx
const reasonFade = {
  initial: { opacity: 0 },
  whileInView: { opacity: 1 },
  transition: { duration: 0.6, delay: index * 0.1 },
  viewport: { once: true }
};
```

- y方向の移動なし（opacityのみ）
- 控えめなstagger効果

### SP（スマートフォン）での変化

**レイアウト変更**:
- 横並び → 縦並び
- 各理由を縦にスタック

**サイズ変更**:
- セクション見出し: 28px
- キーワード: 24px
- 説明文: 16px

**余白変更**:
- 上余白: 120px
- 下余白: 100px
- 理由間: 60px
- 左右マージン: 5%

### このセクションでの心理効果

**ユーザーが感じること**:
- 「他社との違いが明確だ」
- 「この理由なら上司にも説明できる」
- 「信頼できる姿勢だ」

**次のセクションへの期待**:
- 「実際にどう進めるのか」への疑問

## セクション4: プロセス（Process）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
BtoBでは「どう進めるか」の不透明さが問い合わせの障壁になる。プロセスを可視化することで「依頼しても大丈夫」という安心感を与える。

**👤 専門家②（心理学）**:
人は「先が見える」と安心する。プロセスを段階的に示すことで、不確実性を軽減し、意思決定を促進する。

**👤 専門家③（デザイン）**:
タイムライン形式で縦に流れるレイアウト。左に番号、右に内容を配置し、スクロールで「進行」を体感させる。

### レイアウト

**構成**:
- 背景色: オフホワイト（#f8f8f8）
- セクション見出し（36px）左揃え
- プロセス5ステップをタイムライン形式で縦に配置
- 左側に番号、右側に内容（左25%:右65%、間10%）

**各ステップの内部構成**:
- 左側: ステップ番号（48px、薄い色）
- 右側: ステップ名（24px） + 説明文（17px、2-3行）

**タイムラインの縦線**:
- 左側番号の中央から下方向へ伸びる縦線
- 幅1px、色#e0e0e0
- 最後のステップの下で終了

**なぜこのレイアウトか**:
- タイムライン形式は「順序」を直感的に伝える
- 左右分割により「番号」と「内容」を視覚的に分離
- 縦線が「繋がり」「流れ」を表現

**代替案を採用しない理由**:
- ❌ 横並び: 5ステップは横に収まらない
- ❌ フローチャート: 矢印やアイコンが増え、Minimalに反する
- ❌ アコーディオン: 全体像が見えにくい

### コンテンツ要素

**セクション見出し**:
- テキスト: 「ご依頼の流れ」
- サイズ: 36px（PC）、28px（SP）
- 配置: 左揃え、左マージン10%

**プロセス（5ステップ）**:

| Step | ステップ名 | 説明の方向性 |
|------|----------|-------------|
| 01 | お問い合わせ | フォームから連絡。1営業日以内に返信 |
| 02 | ヒアリング | オンラインで課題をお聞きする。60分程度 |
| 03 | ご提案 | 課題に合わせた具体的な提案書を提出 |
| 04 | ご契約 | 条件合意後、契約締結。着手金の請求 |
| 05 | プロジェクト開始 | キックオフミーティングから開始 |

**各ステップの配置**:
- ステップ間の余白: 80px
- 番号と内容の間: タイムライン縦線で繋ぐ

### デザイン詳細

**背景**:
- 色: オフホワイト（#f8f8f8）
- 理由: 選ばれる理由の白から切り替え

**余白**:
- 上余白: 160px
- 下余白: 160px
- 左右マージン: 各10%

**タイムライン縦線のスタイル**:
- 位置: 左側番号エリア（25%幅）の中央
- 幅: 1px
- 色: #e0e0e0
- 長さ: ステップ1の中央からステップ5の中央まで

**番号のスタイル**:
- サイズ: 48px
- 色: ライトグレー（#c0c0c0）透かし的に
- フォント: Bold

**特徴的な表現**:
- 番号を大きく薄く表示し、装飾ではなく構造として機能
- タイムラインの縦線で「流れ」を可視化
- 各ステップに矢印やアイコンを使わない

### アニメーション

**使用する**: 各ステップの順次フェードイン

**アニメーション仕様**:
```tsx
const stepFade = {
  initial: { opacity: 0, x: -20 },
  whileInView: { opacity: 1, x: 0 },
  transition: { duration: 0.5, delay: index * 0.1 },
  viewport: { once: true, margin: "-30px" }
};
```

- 左からスライドインすることで「進行」を表現
- 控えめな移動距離（20px）

### SP（スマートフォン）での変化

**レイアウト変更**:
- 左右分割を維持するが比率を調整（20%:75%、間5%）
- タイムライン縦線は維持

**サイズ変更**:
- セクション見出し: 28px
- ステップ番号: 36px
- ステップ名: 20px
- 説明文: 15px

**余白変更**:
- 上余白: 100px
- 下余白: 100px
- ステップ間: 60px
- 左右マージン: 5%

### このセクションでの心理効果

**ユーザーが感じること**:
- 「プロセスが明確でわかりやすい」
- 「問い合わせ後の流れがイメージできた」
- 「安心して依頼できそうだ」

**次のセクションへの期待**:
- 「問い合わせてみようか」という気持ち

## セクション5: CTA（Call to Action）

### なぜこのセクションが必要か

**👤 専門家①（マーケティング）**:
プロセスを理解したユーザーは「依頼への障壁」が下がっている。このタイミングでCTAを出し、問い合わせへ誘導する。

**👤 専門家②（心理学）**:
プロセス説明で安心感を得たユーザーに、行動を促す。「次の一歩」を明確にすることで、決断を促進する。

**👤 専門家③（デザイン）**:
TOPページのCTAと同様のデザイン。一貫性を持たせることで、CTAの認識を強化。

### レイアウト

**構成**:
- 背景色: ダークネイビー（#1a1a2e）
- 見出し（36px、白）中央配置
- 補足テキスト（17px、ライトグレー）
- CTAボタン（アクセントカラー）

**なぜこのレイアウトか**:
- 背景色反転で「ここが終着点」を示す
- TOPページのCTAと一貫したデザインで認識を強化
- 中央配置で視線を集中

**代替案を採用しない理由**:
- ❌ 左右分割: CTAセクションはシンプルに
- ❌ フォーム埋め込み: お問い合わせページに誘導する方が完了率が高い

### コンテンツ要素

**見出し**:
- テキスト: 「ご相談から、始めませんか。」
- サイズ: 36px（PC）、28px（SP）
- 色: 白

**補足テキスト**:
- テキスト: 「まずはお気軽にお問い合わせください。」
- サイズ: 17px
- 色: ライトグレー（#a0a0a0）

**CTAボタン**:
- テキスト: 「お問い合わせ」
- スタイル: TOPページと同一
- 配置: 中央

**事例ページへのリンク**:
- テキスト: 「事例を見る」
- スタイル: 白テキスト + アンダーライン、ホバーで色変化
- 配置: CTAボタンの32px下

### デザイン詳細

**背景**:
- 色: ダークネイビー（#1a1a2e）
- 理由: セクション終了を示し、アクションを促す

**余白**:
- 上余白: 120px
- 下余白: 120px
- 左右マージン: 各20%

**特徴的な表現**:
- 背景反転でCTAセクションを強調
- 事例ページへのリンクを追加し、「まだ迷っている」ユーザーにも選択肢を

### アニメーション

**使用しない**: CTAセクションは即座に表示

### SP（スマートフォン）での変化

**サイズ変更**:
- 見出し: 28px
- 補足テキスト: 16px
- ボタン: 幅90%

**余白変更**:
- 上余白: 80px
- 下余白: 80px
- 左右マージン: 5%

### このセクションでの心理効果

**ユーザーが感じること**:
- 「問い合わせるなら今だ」
- 「まだ迷うなら事例を見てもいい」

**次の行動**:
- CTAクリック → お問い合わせページ
- 事例リンククリック → 事例ページ
