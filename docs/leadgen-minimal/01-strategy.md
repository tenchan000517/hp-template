# LeadGen Minimal テンプレート 全体戦略

## 1-1. このテンプレートの独自価値

### 👤 専門家①（WEBマーケティング30年の権威）の視点

#### Standard テンプレートとの決定的な違い

Standard テンプレートは「情報網羅型」であり、会社概要、事業内容、採用情報など企業の全体像を伝えることを目的とする。一方、LeadGen Minimal は**「問い合わせ獲得」という単一目標に全てを最適化**している。

| 観点 | Standard | LeadGen Minimal |
|------|----------|-----------------|
| 目的 | 企業認知・情報提供 | 問い合わせ獲得 |
| ページ数 | 7-10ページ | 5ページ（最小構成） |
| 情報量 | 網羅的 | 厳選（CVに必要なもののみ） |
| ナビゲーション | 詳細な階層構造 | フラット（3クリック以内にCV） |
| CTA配置 | 各ページに1箇所 | 各セクション末尾に配置 |
| ファーストビュー | 会社紹介 | 課題提起→解決への期待 |

#### このテンプレートでしか実現できないコンバージョン戦略

**「削ぎ落としによる集中」戦略**

高単価BtoBサービスにおいて、情報過多は意思決定を妨げる。LeadGen Minimalでは以下を徹底する:

1. **選択肢の排除**: ナビゲーションは「TOP」「サービス」「事例」「お問い合わせ」の4項目のみ。ユーザーに「どこを見るべきか」を迷わせない
2. **段階的説得**: TOP→サービス→事例→お問い合わせ という一本道の設計。各ページで「次に何を見るべきか」を明示
3. **信頼の積み上げ**: 余白と情報精度で「この会社は自信がある」という印象を与える

#### 「3クリック以内にCV」の実現方法

**クリック1**: TOPページ到達（検索/広告から）
**クリック2**: ヘッダーの「お問い合わせ」ボタン、またはセクション末尾のCTA
**クリック3**: フォーム送信

これを実現するための設計:
- ヘッダーは常時固定表示。「お問い合わせ」ボタンは最右に配置し、他のナビとは異なる色（アクセントカラー）で目立たせる
- TOPページの各セクション末尾に「まずは相談する」「詳しく聞く」等のCTAを配置
- 事例ページからは「同じ課題を相談する」というコンテキスト付きCTAでフォームへ誘導

### 👤 専門家②（日本の行動心理学の権威）の視点

#### このテンプレートが活用する心理的メカニズム

**1. 認知的流暢性（Cognitive Fluency）**

情報が少なく、余白が多いデザインは「処理しやすい」と脳が判断する。処理しやすいものは「信頼できる」「好ましい」と評価される傾向がある。LeadGen Minimalの余白設計は、この認知的流暢性を最大化する。

**2. 権威性バイアス（Authority Bias）**

日本人は特に「専門家」「実績」に弱い。事例ページで具体的な数字（売上30%向上、コスト削減額など）を示すことで、権威性を担保する。ただし、過剰な実績アピールは逆効果となるため、「3つの事例」に厳選し、それぞれを深く見せる。

**3. 損失回避（Loss Aversion）**

日本のBtoB意思決定者は「失敗したくない」という心理が強い。「導入しないことのリスク」を暗示する表現をTOPページで使用:
- ❌「業績向上のために」（獲得フレーム）
- ✅「競合に遅れを取らないために」（損失フレーム）

**4. 社会的証明（Social Proof）**

「他社も導入している」という安心感。ただし、ロゴの羅列は安っぽくなるため使用しない。代わりに、事例ページで「業界トップ企業A社」のような表現で権威性と社会的証明を同時に満たす。

#### 高単価サービスへの信頼をどう構築するか

**信頼構築の3段階**:

| 段階 | ページ | 心理的目標 | 手法 |
|------|--------|-----------|------|
| 第1段階 | TOP | 「この会社は違う」 | 余白、トーン、課題への共感 |
| 第2段階 | サービス | 「解決できそう」 | 論理的な説明、プロセスの可視化 |
| 第3段階 | 事例 | 「実際に成果が出ている」 | 具体的な数字、ビフォーアフター |

重要なのは、**各段階で「次を見たい」と思わせること**。第1段階で信頼の種を蒔き、第2段階で育て、第3段階で収穫する。

#### 「余白」が心理に与える影響

**1. 高級感の演出**

人間は「スペースにはコストがかかる」と無意識に認識している。高級ブランドの店舗が商品を少なく陳列するのと同じ原理。余白が多い＝「余裕がある」＝「高品質」という連想が働く。

**2. 情報の重要度向上**

余白に囲まれた情報は「厳選されたもの」と認識される。逆に情報を詰め込むと、全てが等価値に見え、何も印象に残らない。

**3. 呼吸のリズム**

日本人は特に「間（ま）」を重視する文化を持つ。適切な余白は、ユーザーに心理的な「呼吸」の時間を与え、情報を咀嚼する余裕を生む。

### 👤 専門家③（人間最強のWEBデザイナー）の視点

#### このテンプレートのビジュアルアイデンティティ

**「静謐な自信」**

LeadGen Minimalのビジュアルアイデンティティは「静謐な自信」である。派手な装飾や過剰な動きで注目を集めるのではなく、**「何も足さない、何も引かない」状態で成立する美しさ**を追求する。

具体的な表現:
- **余白**: 画面の40%以上を余白とする。コンテンツは画面中央に集中
- **タイポグラフィ**: 見出しは極端に大きく（72-96px）、本文は控えめに（16-18px）。この対比で視線を誘導
- **カラー**: モノトーン+1色。アクセントカラーはCTAにのみ使用し、希少性を持たせる
- **グリッド**: 12カラムグリッドを基本とするが、あえて左右非対称の配置を取り入れ、「計算された不均衡」を演出

#### 「Minimal」表現をどう具体化するか

**Minimalは「少ない」ではなく「本質のみ」**

よくある誤解: Minimal = 要素が少ない、シンプル、白い
正しい理解: Minimal = 不要なものがない、全てに意味がある

LeadGen Minimalでの具体化:

| 要素 | 非Minimal的アプローチ | Minimal的アプローチ（本テンプレート） |
|------|----------------------|-------------------------------------|
| 見出し | 装飾的なアイコン付き | テキストのみ。サイズと余白で階層を表現 |
| カード | 角丸+影+ボーダー | 背景色の微差のみ。または区切り線1本 |
| ボタン | グラデーション+影+アイコン | 単色塗り+テキスト。ホバーで色反転 |
| セクション区切り | 斜め線、波形、SVG装飾 | 余白の増減のみ。または細い水平線1本 |
| 写真 | 丸型クロップ、フレーム付き | 矩形のまま。余白で呼吸を持たせる |

#### AIデザインとの差別化ポイント

AIが生成しがちなデザインの特徴:
1. **全要素にカード化**: 角丸12px、影付きのカードで全てを囲む
2. **紫系グラデーション**: 青紫〜ピンクのグラデーションを多用
3. **均一な余白**: 全セクションで同じ上下マージン
4. **過剰なアニメーション**: 全要素にフェードイン
5. **アイコンの乱用**: 文字情報の横に必ずアイコン

**LeadGen Minimalでの対抗策**:

1. **カードを使わない**: 背景色のわずかな変化（#FFFFFF → #FAFAFA）で領域を分ける
2. **寒色系モノトーン**: ダークネイビー（#1a1a2e）+ オフホワイト（#f8f8f8）を基調に
3. **不均一な余白**: セクションごとに余白を変え、リズムを作る（詳細は1-4で後述）
4. **アニメーションは2種類のみ**: フェードイン（opacity）と微スライド（translateY 20px）のみ
5. **アイコン禁止**: テキストと数字のみで情報を伝える

## 1-2. カラー戦略

### メインカラーの推奨系統と心理効果

**推奨: ダークネイビー系（#1a1a2e 〜 #2d3a4a）**

| 色 | HEX | 心理効果 | 使用場所 |
|----|-----|---------|---------|
| ダークネイビー | #1a1a2e | 信頼、専門性、落ち着き | テキスト、ヘッダー背景 |
| ミディアムネイビー | #2d3a4a | 権威、安定 | 見出し、フッター |
| ライトグレー | #6b7280 | 補助、控えめ | サブテキスト、キャプション |

**なぜダークネイビーか**:
- 黒（#000000）よりも柔らかく、長時間見ても疲れにくい
- 青系は「信頼」「専門性」を連想させ、BtoBに適している
- 日本の意思決定者（40-60代）に親和性が高い

### アクセントカラーの使いどころ

**推奨: 深いテラコッタ（#c45c3e）または 落ち着いたティール（#2a7d7d）**

アクセントカラーは**CTAボタンにのみ使用**する。これにより:
- CTAの視認性が極限まで高まる
- 「このボタンは重要」という認識を自然に形成
- 色の希少性が高級感を演出

**使用禁止箇所**:
- 見出し
- アイコン
- 背景色
- リンクテキスト（リンクは下線で表現）

### 背景色のバリエーション

| 色 | HEX | 使用場所 | 目的 |
|----|-----|---------|------|
| ピュアホワイト | #FFFFFF | メインコンテンツエリア | 清潔感、集中 |
| オフホワイト | #f8f8f8 | 交互セクションの背景 | 領域分け（カードの代替） |
| ウォームグレー | #f0f0ec | 引用、補足情報エリア | 情報の階層化 |
| ダークネイビー | #1a1a2e | ヘッダー、フッター、CTAセクション | 対比、締め |

**重要**: 背景色の切り替えでセクションを区切る。ボーダーや装飾線は原則使用しない。

### 避けるべき色の組み合わせ

1. **紫系全般**: AIデザインの象徴。安っぽく見える
2. **ビビッドな青（#0066FF等）**: 企業サイトの典型で差別化できない
3. **グリーン系**: エコ、健康のイメージが強く、BtoBサービスに合わない
4. **黒+赤**: 警告色の組み合わせ。高級感ではなく緊張感を生む
5. **グラデーション全般**: Minimal表現に反する

## 1-3. タイポグラフィ戦略

### 見出しのサイズ階層

| 階層 | PC サイズ | SP サイズ | ウェイト | 用途 |
|------|----------|----------|---------|------|
| H1 | 72px | 40px | Bold (700) | ページタイトル（1ページ1回のみ） |
| H2 | 48px | 32px | Bold (700) | セクション見出し |
| H3 | 28px | 22px | SemiBold (600) | サブ見出し |
| 本文 | 17px | 16px | Regular (400) | 通常テキスト |
| キャプション | 14px | 13px | Regular (400) | 補足、注釈 |

**なぜこのサイズか**:
- H1とH2の差を大きくすること（72px vs 48px）で、ページ内の階層を明確化
- H1を大きくすることで「何についてのページか」を0.5秒で理解させる
- 本文17pxは、日本語の可読性を保ちながら、画面内の情報密度を最適化

### 本文とのコントラスト

**コントラスト比 4.5:1 を厳守**

| 組み合わせ | 背景色 | 文字色 | コントラスト比 |
|-----------|--------|--------|--------------|
| 通常 | #FFFFFF | #1a1a2e | 15.2:1 ✅ |
| サブ | #FFFFFF | #6b7280 | 5.3:1 ✅ |
| 反転 | #1a1a2e | #FFFFFF | 15.2:1 ✅ |
| 薄背景 | #f8f8f8 | #1a1a2e | 14.1:1 ✅ |

### 日本語特有の考慮点

**字間（letter-spacing）**:
- 見出し: 0.05em（やや広め。漢字の可読性向上）
- 本文: 0（日本語はデフォルトで適切）

**行間（line-height）**:
- 見出し: 1.3（タイトにして塊感を出す）
- 本文: 1.8（日本語は横幅が広いため、縦に余裕を持たせる）

**フォントファミリー**:
```css
font-family: "Noto Sans JP", "Hiragino Kaku Gothic ProN", "Hiragino Sans", Meiryo, sans-serif;
```

**なぜNoto Sans JPか**:
- Googleフォントとして無料で利用可能
- ウェイトのバリエーションが豊富（100-900）
- 可読性が高く、モダンな印象
- 游ゴシックは環境依存で表示が安定しない

### 「Minimal」表現におけるタイポグラフィの役割

Minimalデザインでは、装飾要素を排除する分、**タイポグラフィ自体がデザイン要素**となる。

**具体的な表現**:
1. **見出しのサイズで領域を支配する**: 72pxの見出しは、それ自体が視覚的インパクトを持つ
2. **余白との対話**: 大きな見出しの周囲に十分な余白を配置し、「呼吸する」レイアウトに
3. **ウェイトの対比**: 見出し（Bold）と本文（Regular）の対比で、視線誘導を行う
4. **装飾を排除**: 見出しへの下線、背景色、アイコンは使用しない

## 1-4. 余白とリズム戦略

### セクション間の余白の考え方

**基本原則: 余白率40%以上**

画面を見た時、コンテンツよりも余白が多いと感じる状態を目指す。

**セクション間の余白設計**:

| パターン | 上余白 | 下余白 | 使用場面 |
|---------|--------|--------|---------|
| 標準 | 160px | 160px | 通常のセクション間 |
| 強調 | 200px | 120px | 重要セクションの前（呼吸を持たせる） |
| 連続 | 80px | 80px | 関連性の高いセクション間 |
| 断絶 | 240px | 160px | 文脈の切り替え時 |

**なぜ固定値か**:
- %指定は画面サイズで変動し、意図した余白感を維持できない
- ただしSPでは上記の60%の値を使用

### 「Minimal」表現における余白の意味

**余白は「無」ではなく「間（ま）」**

日本の伝統美学における「間」の概念をデジタルに翻訳する。

1. **余白は情報である**: 「ここにはあえて何も置かない」という判断の表明
2. **余白は時間である**: ユーザーに情報を咀嚼する時間を与える
3. **余白は価値である**: 「スペースを贅沢に使える」という自信の表れ

### 密と疎のリズム設計

**モノトーンでは退屈になるため、リズムをつける**

```
[疎] ヒーロー（余白大）
  ↓
[密] 課題提起（情報量多め）
  ↓
[疎] 解決策提示（余白大、インパクト重視）
  ↓
[密] サービス詳細（情報量多め）
  ↓
[疎] CTA（余白大、集中させる）
```

このリズムにより:
- ユーザーの集中力を維持
- 重要な情報（解決策、CTA）に余白を与えて目立たせる
- スクロール時の体験に変化を与える

### 均一な余白を避ける具体的な方法

**1. セクションごとに余白を変える**

```css
/* ❌ AIデザインでよくある均一な余白 */
.section { padding: 80px 0; }

/* ✅ LeadGen Minimalのリズム */
.hero { padding: 0 0 120px; }
.problem { padding: 200px 0 80px; }
.solution { padding: 160px 0 160px; }
.cta { padding: 120px 0 200px; }
```

**2. コンテンツ内の余白も非対称に**

```css
/* ❌ 均一 */
.card { padding: 40px; }

/* ✅ 非対称（上を多めに取り、高級感を出す） */
.card { padding: 60px 40px 40px; }
```

**3. 見出しと本文の間の余白**

見出しの下は「32px」、本文の下は「24px」と、要素の重要度で余白を変える。

## 1-5. アニメーション戦略

### どこにアニメーションを使い、どこに使わないか

**使用する場面**:
1. スクロールによるセクション出現時（初回のみ）
2. CTAボタンのホバー時
3. ヘッダーのスクロール時変化

**使用しない場面**:
1. ページ遷移（即座に表示。読み込み中のローダーのみ許可）
2. フォーム入力時
3. ナビゲーション開閉
4. 画像のズームや回転
5. 装飾的なループアニメーション

### 「Minimal」表現におけるアニメーションの役割

**アニメーションは「動き」ではなく「気づき」のために使う**

派手なアニメーションは視線を奪い、コンテンツへの集中を妨げる。LeadGen Minimalでは、アニメーションを最小限に抑え、**「このコンテンツが表示されました」という静かな合図**として使用する。

### 過剰にならない基準

**3秒ルール**: ページを開いて3秒以内に全てのアニメーションが完了していること
**1回ルール**: 同じ要素が2回以上アニメーションしないこと
**脇役ルール**: アニメーション自体が注目されないこと

### 具体的なアニメーションパターン（2種類に限定）

**パターン1: フェードイン + マイクロスライド**

```tsx
// 使用場面: セクションの出現時
const fadeInUp = {
  initial: { opacity: 0, y: 20 },
  whileInView: { opacity: 1, y: 0 },
  transition: { duration: 0.6, ease: "easeOut" },
  viewport: { once: true, margin: "-100px" }
};
```

- 移動距離は20pxのみ（大きいと派手になりすぎる）
- 0.6秒で完了（これ以上長いと遅く感じる）
- `once: true` で再トリガーを防止
- `margin: "-100px"` で画面中央付近で発火

**パターン2: ホバーによる色反転**

```tsx
// 使用場面: CTAボタン
const buttonHover = {
  initial: { backgroundColor: "#c45c3e", color: "#FFFFFF" },
  whileHover: { backgroundColor: "#1a1a2e", color: "#FFFFFF" },
  transition: { duration: 0.2 }
};
```

- 0.2秒で即座に反応（インタラクティブ感を出す）
- スケール変化やシャドウ追加は行わない（Minimalに反する）

**使用禁止のアニメーション**:
- パララックス
- 3D回転
- バウンス
- スプリング（跳ね返り）
- 連続的なパルス
- テキストの1文字ずつ出現

## 設計原則まとめ

### このテンプレートの3つの柱

1. **削ぎ落とし**: 必要なものだけを残す。迷ったら削る
2. **余白の贅沢**: スペースをコストと考えず、価値として扱う
3. **静謐な自信**: 派手さで勝負しない。品質で語る

### 実装時のチェックリスト

- [ ] カードコンポーネントを使用していないか？（背景色の変化で代替）
- [ ] 紫系の色を使用していないか？
- [ ] 全セクションの余白が均一になっていないか？
- [ ] アニメーションが2種類を超えていないか？
- [ ] アイコンを装飾的に使用していないか？
- [ ] 3クリック以内にお問い合わせフォームに到達できるか？
